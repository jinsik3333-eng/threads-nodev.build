Claude Code가 뭔가요?

코딩 몰라도 됩니다. 터미널이 뭔지 몰라도 됩니다. 이 가이드는 그런 분들을 위해 만들었습니다.

일단 — 지금 쓰고 있는 Claude랑 뭐가 다른가요?

claude.ai 또는 Claude Desktop을 써봤다면, 채팅창에 질문 타이핑 → Claude 답변 형태로 쓰고 있을 겁니다. Claude Code는 거기서 한 발짝 더 나아간 버전입니다.

📱 claude.ai / Desktop
채팅창에서 질문하고 답변을 받는 형태.

Claude가 텍스트로 "이렇게 하세요"라고 알려주면, 내가 직접 그걸 실행해야 합니다.
⚡ Claude Code
내 컴퓨터 폴더에 직접 접근해서 파일을 읽고, 만들고, 수정합니다.

"이렇게 하세요"가 아니라 Claude가 직접 실행합니다.
저도 처음엔 "claude.ai로도 충분한데 왜 Claude Code를 써야 하지?"라고 생각했어요. 그런데 써보고 나서 달라졌습니다. 폴더 안에 파일이 여러 개 있을 때, 일일이 복붙하지 않아도 Claude가 알아서 다 읽고 작업해줍니다. 반복 업무를 맡기면 정말 맡아서 합니다.

그래서 누가 쓰면 좋은가요?

📁 파일 여러 개를 다룰 때

보고서 10개 정리, 엑셀 파일 분석, 폴더 안 이미지 이름 일괄 변경 등. 파일을 하나하나 복붙하지 않아도 됩니다.

🔁 반복 작업을 자동화할 때

매주 같은 형식으로 뭔가를 만들어야 한다면, 한 번 세팅해두면 Claude가 매번 알아서 합니다.

📝 코딩이 필요한 작업

코딩을 전혀 몰라도 됩니다. "이런 기능을 만들어줘"라고 말하면 Claude가 코드를 직접 작성하고 실행합니다.

🧠 AI에게 맥락을 계속 유지하게 할 때

매번 "나는 마케터야, 이런 프로젝트야" 설명 없이도 Claude가 내 폴더와 작업 방식을 기억합니다.

터미널 처음 만나기

무섭게 생긴 검은 창이지만, 그냥 "타이핑으로 명령하는 앱"입니다. 부수거나 망가뜨릴 수 없어요.

터미널이 뭔가요?

컴퓨터에 원래 설치되어 있는 앱입니다. 카카오톡처럼 메시지를 보내는 대신, 컴퓨터에 명령을 타이핑하는 곳입니다.

처음 터미널을 열면 "내가 뭔가 잘못하면 컴퓨터가 망가지는 거 아닌가?"라는 생각이 들 수 있어요. 걱정 안 하셔도 됩니다. 타이핑만으로는 아무것도 망가지지 않습니다. 명령어를 입력하고 Enter를 누르기 전까지는 아무 일도 일어나지 않아요.

터미널 여는 방법

  • 키보드에서 Command(⌘) + Space를 동시에 누릅니다. 화면 중앙에 검색창이 뜹니다.
  • 검색창에 terminal이라고 입력하고 Enter를 누릅니다.
  • 흰색이나 검은색 창이 하나 뜨고, 마지막 줄에 % 또는 $ 기호가 있으면 정상입니다. 이걸 프롬프트라고 부릅니다. "여기에 명령어를 입력하세요"라는 표시입니다.
  • 키보드에서 Windows 키를 누릅니다.
  • powershell이라고 입력하고, "Windows PowerShell"을 클릭합니다. (또는 "터미널"로 검색해도 됩니다.)
  • 파란색(또는 검은색) 창에 PS C:\Users\이름> 같은 텍스트가 보이면 됩니다. 이게 프롬프트입니다.

기본 개념 딱 하나만

터미널에서 가장 중요한 개념은 "지금 어느 폴더에 있느냐"입니다. Claude Code는 내가 터미널에서 현재 위치한 폴더를 기준으로 작업합니다.

입력해보세요
pwd
# 이런 식으로 현재 폴더 경로가 출력됩니다
/Users/이름  (Mac)
C:\Users\이름  (Windows)
pwd = "Print Working Directory"
"지금 내가 어디 있는지 보여줘"라는 명령어입니다. 이것만 알아도 됩니다. 터미널에서 뭔가 입력하고 Enter를 눌러도 아무것도 망가지지 않습니다.

Claude Code 설치하기

명령어를 복붙해서 Enter만 누르면 됩니다. 하나씩 따라오세요.

Node.js가 뭔가요? (왜 필요한가요?)

Claude Code를 실행하기 위한 엔진입니다. 앱을 실행하려면 런타임이 필요한 것처럼, Claude Code를 실행하려면 Node.js가 필요합니다. 일반 앱 설치처럼 클릭으로 설치할 수 있습니다.

단계별 설치

  • 터미널에 아래 명령어를 입력하고 Enter를 누르세요.
    node --version
    # 버전이 나오면 (예: v20.11.0) → 이미 설치됨. 2번 건너뛰고 3번으로 이동
    # "command not found" 오류가 나오면 → 아래 2번 진행
  • nodejs.org 접속 → LTS 버전 다운로드 → 설치 파일 실행 (일반 앱 설치와 동일).
    설치 후 터미널을 완전히 닫고 다시 열어야 합니다.
    Windows 주의
    설치 후 반드시 PowerShell을 닫고 다시 여세요. PATH 설정이 새 창에서만 반영됩니다.
    그래도 안 된다면: PowerShell에 Set-ExecutionPolicy RemoteSigned -Scope CurrentUser 입력 → Y 선택.
  • 아래 명령어를 터미널에 복붙하고 Enter를 누르세요.
    npm install -g @anthropic-ai/claude-code
    # 여러 줄의 텍스트가 스크롤됩니다 → 정상입니다, 기다리세요
    # 1~2분 소요됩니다
    # 마지막에 오류 없이 프롬프트($, %)로 돌아오면 완료
    이 명령어가 하는 일: npm이라는 도구(Node.js를 설치하면 같이 설치됨)가 인터넷에서 Claude Code 패키지를 받아서 컴퓨터에 설치합니다. 복잡해 보이지만 "앱 설치"와 같습니다.
  • claude --version
    # 버전 번호가 나오면 설치 완료! (예: 1.x.x)
    # "command not found"가 나오면 아래 FAQ 확인
    ⚠️ "command not found" 오류가 나요 ▶

    Mac: 터미널을 완전히 종료 후 다시 열어보세요. 그래도 안 되면 아래 명령어를 실행하세요.

    export PATH="$HOME/.npm-global/bin:$PATH"

    Windows: PowerShell을 닫고 다시 여세요. 또는 시스템 환경 변수에서 PATH를 확인하세요.

    그래도 안 된다면: 이 가이드 하단의 다음 편 연락처로 문의해주세요.

막혔을 때 해결법

어디선가 막혔다면 여기서 해결하세요. 설치 과정에서 자주 나오는 오류 3가지를 정리했습니다.

자주 막히는 상황 3가지

❓ node --version 입력했는데 "command not found"가 나와요 ▶

원인: Node.js가 설치되지 않았거나, 설치 후 터미널을 재시작하지 않았습니다.

해결:

1. nodejs.org에서 LTS 버전을 다운로드해서 설치하세요.

2. 설치 완료 후 터미널을 완전히 닫고 다시 여세요.

3. 다시 node --version을 입력하세요.

Windows 추가 조치
그래도 안 된다면 PowerShell에 아래를 입력하고 Y를 누르세요.
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
❓ npm install 중간에 오류가 나고 멈췄어요 ▶

원인 1 — 권한 문제 (Mac에서 "EACCES" 오류):
sudo 없이 해결하는 방법입니다. 아래 순서대로 하세요.

npm 글로벌 경로 변경 (한 번만 하면 됩니다)
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

이후 다시 npm install -g @anthropic-ai/claude-code를 실행하세요.

원인 2 — 인터넷 연결 불안정:
와이파이를 끊었다가 다시 연결한 후 같은 명령어를 다시 입력하세요. npm은 중단된 지점부터 자동으로 이어받습니다.

❓ claude 입력했는데 "command not found"가 나요 ▶

원인: npm 글로벌 경로가 PATH에 없어서 터미널이 claude 명령어를 찾지 못합니다.

Mac 해결:

export PATH="$HOME/.npm-global/bin:$PATH"
# 이 줄을 입력하고 Enter → 그 다음 claude 재입력

영구적으로 해결하려면 위 FAQ(npm install 오류)에서 ~/.zshrc에 경로를 추가하는 방법을 따르세요.

Windows 해결:
PowerShell을 완전히 닫고 다시 여세요. 그래도 안 되면 Windows 설정 → 시스템 → 정보 → 고급 시스템 설정 → 환경 변수에서 PATH를 확인하세요.

그래도 해결이 안 된다면
이 가이드 제작자 쓰레드 계정에 DM을 남겨주세요. 오류 메시지를 스크린샷으로 찍어서 보내주시면 답변드립니다.
