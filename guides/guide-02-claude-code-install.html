Claude Code가 뭔가요?

코딩 몰라도 됩니다. 터미널이 뭔지 몰라도 됩니다. 저도 그랬거든요.

저는 Claude Desktop을 먼저 썼어요. 쓰다 보니 "파일을 직접 다루게 할 수 없을까?"라는 생각이 들었고, 그때 Claude Code를 알게 됐습니다. 처음 설치하고 GUI로 열었는데, 솔직히 막막했어요. 입력창은 있는데 뭘 어떻게 해야 할지 모르겠더라고요. 그냥 닫았다가 다시 열기를 몇 번 반복했습니다. 근데 조금씩 써보면서 달라졌어요. 크롤링 도구를 하나 만들었는데, 예전에 몇 시간씩 걸리던 작업이 버튼 하나로 되는 걸 보고 "아, 이거 진짜구나" 싶었습니다. 그 이후로 업무 방식이 바뀌었어요.

한 줄 정의

"내 컴퓨터 폴더에 직접 접근해서, 수십 개 파일을 한 번에 읽고, 정리하고, 결과물을 만들어주는 AI"

claude.ai와 어떻게 다른가요?

이미 claude.ai 또는 Claude Desktop을 써봤다면, Claude Code는 거기서 한 발짝 더 나아간 버전입니다.

기능📱 claude.ai / Desktop⚡ Claude Code
파일 접근직접 올려야 함폴더 전체 자동 읽기
작업 범위대화 1회수십~수백 파일 동시 처리
결과 저장복사→붙여넣기자동 저장
반복 작업매번 수동자동화 가능
기억대화 내에서만CLAUDE.md로 영구 기억
저도 처음엔 "claude.ai로도 충분한데 왜 Claude Code를 써야 하지?"라고 생각했어요. 그런데 써보고 나서 달라졌습니다. 폴더 안에 파일이 여러 개 있을 때, 일일이 복붙하지 않아도 Claude가 알아서 다 읽고 작업해줍니다.

실제로 어떤 일을 시킬 수 있나요?

제가 직접 만들어서 쓰고 있거나, 주변 직장인들이 바로 써먹은 것들입니다.

🕷️ 크롤링 도구 만들기

경쟁사 가격 모니터링, 채용공고 수집, 뉴스 자동 수집. "이 사이트에서 이 데이터 긁어오는 도구 만들어줘"라고 하면 됩니다. 저도 이걸로 몇 시간짜리 반복 작업을 없앴습니다.

📊 분석 도구 만들기

CSV·엑셀 파일을 받아서 특정 조건으로 분석해주는 도구를 직접 만들 수 있습니다. 파이썬 몰라도 됩니다. "이 형식의 파일을 넣으면 요약해주는 도구 만들어줘"가 통합니다.

📁 문서·파일 일괄 처리

회의록 30개 읽기 → 이번 달 결정사항만 추출 → 파일로 저장. 파일명 일괄 변경, 폴더 정리도 됩니다. 일일이 열어볼 필요가 없어요.

🔁 반복 업무 자동화

미팅 녹취 → 요약·액션아이템 자동 정리. 주간·월간 보고서 템플릿 고정 후 자동 생성. 한 번 만들어두면 계속 씁니다.

🌐 웹 리서치 자동화

여러 페이지를 동시에 읽고 비교표를 만들어줍니다. 경쟁사 분석, PDF 요약 일괄 처리. "이 5개 링크 다 읽고 표로 정리해줘"가 됩니다.

✍️ 콘텐츠 제작

블로그, 뉴스레터, SNS 초안 작성 → 파일로 자동 저장. 제안서, 보고서 템플릿도 만들어두면 매번 처음부터 쓸 필요가 없습니다.

시작 전 준비물 확인

아래 항목이 있으면 바로 시작할 수 있습니다.

  • 스마트폰으로는 설치할 수 없습니다. PC/노트북이 필요합니다.
  • Claude Code 설치 시 파일을 다운로드합니다. 와이파이 환경 권장.
  • 설치 후 처음 실행할 때 계정이 필요합니다. claude.ai에서 미리 만들어두면 편합니다. Step 4에서 안내합니다.
요금제 한 줄 정리
Claude Code는 Claude Pro ($20/월)에서 쓸 수 있습니다. 집중 작업을 1시간만 해도 한도에 닿는 경우가 있어서, 본격적으로 쓸 생각이라면 Claude Max ($100/월)가 더 편합니다. 처음엔 Pro로 시작해보세요.

터미널 처음 만나기

무섭게 생긴 검은 창이지만, 그냥 "타이핑으로 명령하는 앱"입니다. 부수거나 망가뜨릴 수 없어요.

터미널이 뭔가요?

컴퓨터에 원래 설치되어 있는 앱입니다. 카카오톡처럼 메시지를 보내는 대신, 컴퓨터에 명령을 타이핑하는 곳입니다.

처음 터미널을 봤을 때 저도 무서웠어요. 검은 창에 커서만 깜빡이는데, 잘못 건드리면 컴퓨터가 어떻게 되는 거 아닌가 싶었거든요. 근데 해보니까 그냥 채팅창이랑 비슷해요. 타이핑하고 엔터 누르는 것뿐이고, 엔터 누르기 전까지는 아무 일도 안 생깁니다.

터미널 여는 방법

  • 키보드에서 Command(⌘) + Space를 동시에 누릅니다. 화면 중앙에 검색창이 뜹니다.
  • 검색창에 terminal이라고 입력하고 Enter를 누릅니다.
  • 흰색이나 검은색 창이 하나 뜨고, 마지막 줄에 % 또는 $ 기호가 있으면 정상입니다. 이걸 프롬프트라고 부릅니다. "여기에 명령어를 입력하세요"라는 표시입니다.
  • 키보드에서 Windows 키를 누릅니다.
  • powershell이라고 입력하고, "Windows PowerShell"을 클릭합니다.
  • 파란색(또는 검은색) 창에 PS C:\Users\이름> 같은 텍스트가 보이면 됩니다. 이게 프롬프트입니다.

기본 개념 딱 하나만

터미널에서 가장 중요한 개념은 "지금 어느 폴더에 있느냐"입니다. Claude Code는 내가 터미널에서 현재 위치한 폴더를 기준으로 작업합니다.

입력해보세요
pwd
# 이런 식으로 현재 폴더 경로가 출력됩니다
/Users/이름  (Mac)
C:\Users\이름  (Windows)
pwd = "Print Working Directory"
"지금 내가 어디 있는지 보여줘"라는 명령어입니다. 터미널에서 뭔가 입력하고 Enter를 눌러도 아무것도 망가지지 않습니다.

폴더 이동하는 법

Claude Code를 실행하면 현재 폴더의 파일을 모두 다룰 수 있습니다. 그래서 작업 시작 전에 원하는 폴더로 이동하는 습관이 중요합니다.

폴더 이동 방법
cd ~/Desktop/내폴더이름
# cd = "Change Directory" (폴더 이동)
# Mac: ~/Desktop 은 바탕화면
# Windows: cd C:\Users\이름\Desktop\내폴더이름

# 팁: 폴더를 터미널 창에 드래그하면 경로가 자동 입력됩니다

파일을 지정해서 말하는 법 — @ 기호

파일 이름 앞에 @를 붙이면 "이 파일을 기준으로 작업해줘"라는 의미가 됩니다.

@ 기호 활용 예시
@회의록.txt 에서 액션아이템만 뽑아줘
@매출현황.csv 이 파일 분석해서 월별 추이 알려줘
@docs/ 이 폴더 문서 모두 읽고 목차 만들어줘
# 파일 경로를 일일이 설명하지 않아도 됩니다
# 폴더 이름에 / 를 붙이면 폴더 전체를 읽습니다

큰 작업 전 — Plan Mode로 먼저 확인

# 마음에 안 들면 → 방향 수정 후 다시 실행

Cursor·VS Code에서 쓰기

터미널 없이 GUI 화면에서 Claude Code를 쓸 수 있습니다. 비개발자에게 훨씬 편한 방법입니다.

터미널이 낯설게 느껴진다면 — 당연합니다. 저도 처음엔 까만 화면이 무서웠어요. Cursor를 쓰기 시작하면서 눈으로 파일을 보면서 Claude와 대화할 수 있게 됐고, 그게 훨씬 자연스러웠습니다. 터미널을 잘 다루게 된 것도 사실 GUI로 먼저 익숙해진 덕분이었어요.

터미널 vs GUI — 뭐가 다른가요?

⌨️ 터미널 방식
까만 화면에 명령어 입력

파일이 어디 있는지 직접 경로 알아야 함

결과가 텍스트로만 표시됨
🖥️ GUI 방식 (Cursor/VS Code)
왼쪽에 파일 목록, 오른쪽에 내용

클릭으로 파일 열기, Claude 창도 나란히

수정된 내용을 색상으로 바로 확인

Cursor — 비개발자 추천 1순위

Cursor는 Claude Code가 내장된 편집기입니다. VS Code와 거의 똑같이 생겼지만, AI 기능이 더 잘 통합되어 있습니다.

🎯 왜 비개발자에게 좋은가

파일 목록을 눈으로 보면서 클릭으로 열고, Claude 채팅창이 오른쪽에 나란히 뜹니다. "이 파일을 Claude에게 보여주기"가 드래그 한 번으로 됩니다.

💰 요금제

무료 플랜으로도 Claude 모델 사용 가능. 단, Claude Code API를 직접 연결하면 Claude의 전체 기능을 쓸 수 있습니다.

Cursor 설치 & Claude Code 연결

  • cursor.com 접속 → "Download" 클릭 → 설치 파일 실행. 일반 앱 설치와 동일합니다.
  • Cursor 실행 → File → Open Folder → 작업할 폴더 선택. 왼쪽에 파일 목록이 뜹니다.
  • 단축키 Cmd+L (Mac) / Ctrl+L (Windows) → 오른쪽에 채팅창이 열립니다. 여기서 바로 대화할 수 있습니다.
  • 왼쪽 파일 목록에서 파일 클릭 → 채팅창에 @파일명 또는 파일을 채팅창으로 드래그. Claude가 그 파일을 바로 읽습니다.
터미널 내장 기능
Cursor에도 터미널이 내장되어 있습니다. Ctrl + ` (백틱)를 누르면 화면 아래에 터미널이 열립니다. 나중에 터미널이 필요할 때 별도 앱 없이 쓸 수 있습니다.

VS Code — 이미 설치되어 있다면

VS Code가 이미 있다면 Claude Code 확장 프로그램을 설치해서 바로 쓸 수 있습니다.

  • VS Code 왼쪽 사이드바 → 🧩 확장 아이콘 클릭 → 검색창에 "Claude Code" 입력 → Anthropic이 만든 공식 확장 설치
  • 왼쪽 사이드바에 Claude 아이콘이 생깁니다. 클릭하면 채팅 패널이 열립니다. Cursor와 거의 동일한 방식으로 사용합니다.

GUI에서 자주 쓰는 방식

📂 파일 지정

왼쪽 파일 목록에서 파일을 클릭해서 열고, 채팅창에 @파일명 입력하거나 파일을 채팅창으로 드래그합니다.

✏️ 수정 확인

Claude가 파일을 수정하면 변경된 줄이 색상으로 표시됩니다. 초록: 추가된 내용, 빨강: 삭제된 내용. 눈으로 바로 확인 가능합니다.

↩️ 되돌리기

수정 결과가 맘에 안 들면 Cmd+Z / Ctrl+Z로 바로 되돌릴 수 있습니다. 파일 단위로 이전 상태 복원도 가능합니다.

🔍 여러 파일 한꺼번에

채팅창에서 @파일A @파일B처럼 여러 파일을 동시에 지정할 수 있습니다. 두 파일을 비교하거나 합칠 때 유용합니다.

Cursor와 Claude Code는 별개입니다
Cursor는 편집기(화면), Claude Code는 AI 엔진(두뇌)입니다. Cursor 안에서 Claude Code를 쓰는 것이지, 둘 중 하나를 선택하는 게 아닙니다. 터미널에서 설치한 Claude Code가 Cursor 안에서도 그대로 동작합니다.

잘 쓰는 법

설치만 하면 50%입니다. 이 두 가지를 알면 생산성이 진짜 달라집니다.

1. CLAUDE.md — Claude의 기억장치

💡 .md가 뭔가요?
Markdown(마크다운)이라는 문서 형식입니다. 제목은 #, 목록은 -처럼 간단한 기호로 구조를 표현하는데, 컴퓨터가 읽기 쉽고 사람도 텍스트 그대로 충분히 읽을 수 있습니다.

더 예쁘게 보고 싶다면? Cursor·VS Code 같은 편집기에서 파일을 열고 미리보기를 켜면 제목·굵기·목록이 깔끔하게 렌더링됩니다.

Notion·Obsidian과 호환됩니다. 두 서비스 모두 Markdown 기반으로 만들어져서 CLAUDE.md를 그대로 붙여넣으면 서식이 살아납니다. Obsidian은 특히 .md 파일을 폴더째로 열 수 있어서 프로젝트 메모와 함께 관리하기 좋습니다.

매번 "나는 이런 업무 담당하고, 이런 식으로 써줘"를 입력하지 않아도 됩니다. 폴더 안에 CLAUDE.md라는 파일을 만들어두면 Claude가 세션이 시작될 때마다 자동으로 읽습니다.

처음엔 CLAUDE.md 없이 매번 설명하면서 썼어요. 근데 세션이 끊기면 또 처음부터 설명해야 하는 게 너무 번거롭더라고요. CLAUDE.md를 만들고 나서부터는 "나 이런 업무 담당하고 있어, 이런 식으로 해줘"를 한 번만 써두면 됐습니다. 특히 분석 도구를 만들 때, 매번 데이터 구조 설명 안 해도 되니까 시간이 확 줄었어요.
CLAUDE.md 없을 때
매 세션마다:
"나는 마케터야. 보고서는 간결하게. 한국어로. 표 형식 선호. 고객사 실명 쓰지 마."

→ 매번 반복 설명 필요
CLAUDE.md 있을 때
처음 한 번만 작성해두면
세션 시작할 때 Claude가 자동으로 읽음.

→ 바로 본론부터 시작 가능

CLAUDE.md 만드는 두 가지 방법

⚡ /init — 자동으로 만들기

현재 폴더를 Claude가 스캔하고 CLAUDE.md 초안을 자동 생성합니다. 처음 프로젝트를 시작할 때 가장 빠른 방법입니다.

/init
폴더 구조를 분석해서 CLAUDE.md 초안을 만들어줍니다.
초안을 보고 내 역할·규칙을 추가하면 완성.

✍️ 직접 요청하기

원하는 내용을 직접 말해서 만들 수도 있습니다. 처음부터 내 업무에 맞게 구체적으로 만들고 싶을 때.

이 폴더는 영업 데이터 분석용이야.
원본 파일 수정 금지, 한국어로 답변.
이 규칙으로 CLAUDE.md 만들어줘.
업무별 CLAUDE.md 템플릿은 4편에서 자세히 다룹니다. 리서치·콘텐츠·회의록·보고서용 템플릿을 복붙해서 쓸 수 있습니다. 지금은 일단 기본 구조로 시작하고, 4편에서 내 업무에 맞게 고도화하세요.

2. /compact — 대화가 느려지면 이것부터

대화가 길어질수록 Claude의 "기억 용량"이 차오릅니다. 느려지거나 이상한 답변이 나오기 시작하면 /compact를 입력하세요.

/compact 사용 예시
/compact
/compact 리서치 결과 중심으로
/compact 코드 변경사항 위주로
# 대화를 요약해서 용량을 줄입니다
# 맥락은 유지되면서 속도가 빨라집니다
# 방향을 지정하면 해당 내용을 중심으로 요약됩니다

💬 같은 주제 계속 — /compact

대화를 요약해서 용량을 줄입니다. 맥락은 유지되면서 빨라집니다.

🔄 새 주제로 넘어갈 때 — /clear

이전 대화를 모두 지웁니다. 완전히 새로 시작할 때 사용하세요.

3. 잘 말하는 법 — 프롬프트 기초

Claude Code에 어떻게 말하느냐가 결과의 80%를 결정합니다.

파일 정리해줘
reports/ 폴더 안의 .docx 파일들을 날짜순으로 정렬해서 목록으로 만들어줘. 2024년 것만.
보고서 써줘
@매출데이터.csv 읽고 이번 달 매출 요약 보고서를 만들어줘. A4 1장 분량, 표 1개 포함, 한국어로.
다 고쳐줘
@제안서.docx 에서 2페이지 도입부 문단만 더 설득력 있게 다듬어줘. 나머지는 건드리지 마.
좋은 요청의 공식
① 어떤 파일 + ② 어떤 작업 + ③ 어떤 형태로 = 원하는 결과
예: "@회의록.txt 에서 / 액션아이템만 추출해서 / 담당자별로 정리해줘"

4. /statusline — 지금 상태 한눈에 보기

터미널 하단에 현재 모델, 토큰 사용량, 비용 등을 상시 표시하는 상태바입니다. 켜두면 대화가 얼마나 쌓였는지, 비용이 어느 정도인지를 실시간으로 확인할 수 있습니다.

켜기 / 끄기
/statusline
# 입력할 때마다 표시 ↔ 숨기기가 전환됩니다
# 하단에 표시 예시: claude-sonnet-4-5 | ↑ 12k ↓ 3k | $0.04 | 32%

📊 상태바에서 볼 수 있는 것

현재 사용 중인 모델 이름 / 입출력 토큰 수 / 예상 비용 / 컨텍스트 사용률(%). 비용이 갑자기 올라가거나 컨텍스트가 꽉 차가고 있을 때 바로 알 수 있습니다.

막혔을 때 해결법

설치와 사용 중 자주 나오는 상황들을 정리했습니다. 해당하는 항목을 펼쳐보세요.

설치 & 실행 문제

❓ node --version 입력했는데 "command not found"가 나와요 ▶

원인: Node.js가 설치되지 않았거나, 설치 후 터미널을 재시작하지 않았습니다.

1. nodejs.org에서 LTS 버전을 다운로드해서 설치하세요.

2. 설치 완료 후 터미널을 완전히 닫고 다시 여세요.

3. 다시 node --version을 입력하세요.

Windows 추가 조치
그래도 안 된다면 PowerShell에 아래를 입력하고 Y를 누르세요.
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
❓ npm install 중간에 오류가 나고 멈췄어요 (EACCES 권한 오류) ▶

원인: npm이 파일을 쓸 권한이 없습니다. sudo 없이 해결하는 방법입니다.

터미널에서 순서대로 실행 (Mac)
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

이후 다시 npm install -g @anthropic-ai/claude-code를 실행하세요.

❓ claude 입력했는데 "command not found"가 나요 ▶

원인: npm 글로벌 경로가 터미널 PATH에 등록되지 않았습니다.

Mac 즉시 해결 (임시):

export PATH="$HOME/.npm-global/bin:$PATH"
# 입력 후 다시 claude를 입력하세요

Windows 해결: PowerShell을 완전히 닫고 다시 여세요.

영구 해결은 위 "EACCES 오류" 항목의 방법을 따르세요.

❓ 로그인이 안 됩니다 / 브라우저가 안 열려요 ▶

1. claude 실행 후 팝업이 안 열리면 c를 누르면 URL이 표시됩니다 → 그 URL을 직접 브라우저에 붙여넣기하세요.

2. 그래도 안 되면 로그아웃 후 재시도하세요.

/logout
# 로그아웃 후 터미널을 닫고 다시 claude를 실행하세요

3. 최후의 수단: 아래 파일을 삭제 후 다시 시도하세요.

rm ~/.claude.json

사용 중 문제

❓ 너무 느려요 / 답변이 이상해졌어요 ▶

대화가 너무 길어져서 "기억 용량"이 찬 것입니다. 순서대로 해보세요.

1. /compact 입력 → 가장 효과적

2. 그래도 느리면 /clear로 대화 초기화 후 다시 시작

3. claude doctor 실행해서 상태 점검

❓ 원하는 대로 안 고쳐줘요 / 다른 부분을 건드려요 ▶

요청이 너무 모호하거나 범위가 넓을 때 발생합니다.

코드 개선해줘
auth.js 42번 줄 TypeError 오류만 수정해줘. 나머지 코드는 건드리지 마.

반복되는 규칙은 /memory 또는 CLAUDE.md에 저장해두세요.

같은 실수가 반복된다면 /clear로 대화를 초기화하세요.

❓ 파일을 수정했는데 실수인 것 같아요 ▶

1. 즉시 되감기: Esc Esc (두 번 빠르게) → 직전 상태로 복구

2. 여러 단계 전으로: /rewind 명령어 → 시점 선택

3. Git 사용 중이라면:

git diff            # 변경 내용 확인
git checkout -- 파일경로  # 파일 되돌리기
❓ 한글이 깨져서 나와요 (Mac/Linux) ▶

터미널 인코딩 설정 문제입니다.

~/.zshrc 파일에 추가 (터미널에 그대로 복붙)
echo 'export LANG=ko_KR.UTF-8' >> ~/.zshrc && echo 'export LC_ALL=ko_KR.UTF-8' >> ~/.zshrc && source ~/.zshrc
❓ 권한 확인 팝업이 너무 자주 떠요 ▶

반복 허용하는 작업은 자동 승인으로 바꿀 수 있습니다.

/permissions
# 도구별로 Allow / Ask / Deny 설정 가능
# 또는 Shift+Tab → Auto-Accept Mode로 자동 진행
