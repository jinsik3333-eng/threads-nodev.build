나만의 워크스페이스

폴더 구조를 한 번 잡아두면, 매번 설명 없이도 Claude가 바로 맥락을 파악합니다.

도구가 있는데 왜 매번 "나는 마케터고, 이 프로젝트는 분기 보고서용이야"라고 설명해야 하나 싶었어요. 워크스페이스를 세팅하면 폴더를 여는 것만으로 Claude가 상황을 파악하고 시작합니다. 한 번 해두면 계속 씁니다.

워크스페이스란?

업무 단위로 묶은 폴더입니다. 폴더 안에 CLAUDE.md(규칙), input/(원본 자료), output/(결과물)을 넣어두면 — Claude가 폴더를 열자마자 "이 폴더는 무슨 작업용이고, 어떤 규칙으로 해야 하는지"를 파악합니다.

❌ 세팅 전
"나는 영업팀 소속이고, 이 파일은 3분기 실적 데이터야. 원본은 건드리지 말고, 결과는 엑셀로 저장해줘..."

매번 새 세션마다 동일한 설명 반복
✅ 세팅 후
폴더 열기 → claude 실행

Claude가 CLAUDE.md를 읽고 바로 파악. "3분기 영업 분석 작업이군요. 어떻게 도와드릴까요?"

이 가이드의 성공 기준

  • ✓ 업무별 폴더 3개 이상 세팅
  • ✓ 각 폴더에 맞는 CLAUDE.md 작성
  • ✓ 월 사용 비용 파악 및 절약 습관 3개 적용

업무별 폴더 만들기

핵심은 하나: 업무 폴더 안에 CLAUDE.md를 넣어두는 것입니다. 구조는 자유롭게 잡으면 됩니다.

핵심 원칙

폴더 구조를 "반드시 이렇게 해야 한다"는 정답은 없습니다. 다만 실제로 써보면서 잘 작동하는 패턴이 있습니다.

📋 CLAUDE.md — 필수

폴더 안에 CLAUDE.md만 있어도 효과가 큽니다. 이 폴더가 어떤 작업용인지, Claude가 어떻게 행동해야 하는지를 적습니다.

📁 원본 파일 구분 — 선택

원본은 건드리지 않고 결과물만 별도로 모아두고 싶다면 폴더를 나눕니다. 편하면 그냥 한 폴더에 둬도 됩니다.

실제로 어떻게 쓰나요?

예를 들어 영업 데이터 분석 작업을 위한 폴더라면 이런 식입니다. 폴더 이름, 파일 구조는 자신에게 맞게 바꾸세요.

예시 — 이게 정답은 아닙니다
sales-analysis/ ← 업무 단위로 폴더 하나
├── CLAUDE.md ← Claude에게 줄 규칙 (필수)
├── 원본데이터.xlsx ← 원본 자료
└── 분석결과_0115.xlsx ← 작업 결과물
폴더 구조보다 CLAUDE.md 내용이 더 중요합니다. input/, output/ 폴더를 만드는 것보다 CLAUDE.md에 "원본 파일은 수정하지 마"라고 한 줄 쓰는 게 더 효과적인 경우가 많습니다.

폴더와 CLAUDE.md 만들기

Claude에게 부탁하기 (추천) ▶

Claude Code 터미널을 켜고 원하는 대로 말하면 됩니다.

프롬프트 예시
내 바탕화면에 "sales" 폴더 만들어줘. 그 안에 CLAUDE.md도 만들고, "영업 데이터 분석 전용. 원본 파일 수정 금지. 결과는 날짜_이름 형식으로 저장"이라고 써줘.
Desktop/sales/ 폴더와 CLAUDE.md 파일을 만들었습니다.
터미널 명령어로 직접 ▶
Mac 터미널
mkdir -p ~/Desktop/sales && touch ~/Desktop/sales/CLAUDE.md
Windows PowerShell
New-Item -Path "$env:USERPROFILE\Desktop\sales" -ItemType Directory -Force
New-Item "$env:USERPROFILE\Desktop\sales\CLAUDE.md" -ItemType File
폴더 이름은 영어로 짧게
한글 경로는 일부 도구에서 문제가 생길 수 있습니다. 영어로 짧게, 하이픈으로 연결하세요. sales, reports, meetings처럼 단순할수록 좋습니다.

폴더 안 CLAUDE.md 채우기

이 폴더에서만 적용할 규칙을 씁니다. 글로벌 CLAUDE.md(4편)와 다른 점은 — 여기서는 이 작업에 특화된 내용만 씁니다.

글로벌 CLAUDE.md vs 폴더 CLAUDE.md

🌍 글로벌 (4편에서 다뤘습니다)
위치: ~/.claude/CLAUDE.md

모든 폴더에 항상 적용.
"항상 한국어로", "나는 마케터야"처럼 나 자신에 관한 것.
📁 이 폴더 전용 (지금 다루는 것)
위치: 폴더 안 CLAUDE.md

이 폴더를 열 때만 적용.
"원본 파일 수정 금지", "이 폴더의 목적은..."처럼 이 작업에 관한 것.

폴더 CLAUDE.md에 뭘 써야 하나요?

이 폴더가 어떤 작업용인지, Claude가 이 폴더에서 어떻게 행동해야 하는지를 씁니다. 자신의 작업에 맞게 자유롭게 수정하세요.

📊 데이터 분석 / 보고서 작업 예시 ▶
# 이 폴더: 영업 데이터 분석 ## 이 폴더의 목적 월별 영업 실적 분석 및 임원 보고용 자료 작성. ## 파일 규칙 - 원본 파일 수정 금지 — 읽기만 - 결과물 파일명: 날짜_이름 형식 (예: 0115_분석결과) - 금액 단위: 원, 천 단위 콤마 ## 분석 방식 - 비교 기준: 전월 대비 + 전년 동기 대비 - 이상치 발견 시 별도로 메모 - 결론 먼저, 세부 내용은 뒤에
📝 문서 작업 / 보고서 작성 예시 ▶
# 이 폴더: 기획서·보고서 작업 ## 이 폴더의 목적 제안서, 보고서, 회의록 작성. ## 문체 - 외부 문서: 존댓말, 간결하게 - 내부 메모: 요점 중심 - 이메일 초안: 2~3문단 이내 ## 작업 규칙 - 원본 수정 전에 백업 파일 먼저 - 마지막에 "검토 필요 항목" 목록 추가
🎨 콘텐츠 제작 예시 ▶
# 이 폴더: SNS·블로그 콘텐츠 ## 브랜드 톤 - 친근하고 실용적인 말투 - 전문 용어 최소화 - 이모지 적당히 (과도하게 금지) ## 채널별 분량 - 블로그: 소제목 3개 이상, 1,500자 내외 - 인스타: 첫 줄에 훅, 150자 이내 - 뉴스레터: 핵심 1개 + 행동 유도

Claude에게 CLAUDE.md 만들어달라고 부탁하기

이렇게 부탁하면 됩니다
이 폴더는 영업 데이터 분석 전용으로 쓸 거야. 원본 파일 수정 금지, 결과물은 날짜_이름 형식으로 저장, 금액은 원 단위로 써줘. 이 규칙으로 CLAUDE.md 만들어줘.
CLAUDE.md를 작성했습니다. 내용을 확인해보세요...
짧게 쓸수록 좋습니다
CLAUDE.md는 매 메시지마다 읽힙니다. 길수록 토큰이 더 나옵니다. 실제로 매번 필요한 규칙만 5~10줄 안에 담는 게 제일 좋습니다.

유용한 기능 모음

알면 작업 속도가 달라지는 기능들입니다. 하나씩 써보세요.

이미지 입력 — 화면 캡처로 작업 지시하기

텍스트로 설명하기 어려운 것은 화면을 캡처해서 붙여넣으면 됩니다. Claude가 이미지를 읽고 분석합니다.

📸 화면 캡처 후 붙여넣기

Mac: Cmd+Shift+4 → 영역 선택 후 Cmd+V
Windows: Win+Shift+S → 영역 선택 후 Ctrl+V

💡 활용 예시

엑셀 표 캡처 후 "이 표에서 증감률 계산해줘", 오류 화면 캡처 후 "이게 왜 안 되는 건지 설명해줘"

이미지 활용 예시
[화면 캡처 이미지 붙여넣기] 이 대시보드에서 빨간 숫자들만 표로 정리해줘.
이미지를 분석했습니다. 빨간 수치 항목들을 정리하면...

음성 입력 — 말로 지시하기

긴 요청을 타이핑하기 귀찮을 때 운영체제의 받아쓰기 기능을 활용하세요.

🎤 Mac 받아쓰기

시스템 설정 → 키보드 → 받아쓰기 활성화
단축키(보통 Fn Fn)를 누르고 말하면 텍스트로 입력됩니다.

🎤 Windows 받아쓰기

Win + H를 누르면 음성 입력이 시작됩니다. Claude Code 입력창에서 바로 씁니다.

체크포인트 — 실수했을 때 되돌리기

Claude Code는 작업 중 자동으로 스냅샷을 저장합니다. 잘못된 방향으로 작업이 진행됐을 때 이전 상태로 돌아갈 수 있습니다.

단축키
Esc (두 번 연속)
현재 작업을 중단하고 직전 체크포인트 복원 옵션이 나타납니다
대용량 작업 전에는 Plan Mode를 먼저
폴더 전체를 수정하거나 파일을 여러 개 건드리는 작업 전에는 Shift+Tab 2회로 Plan Mode에 진입해서 계획을 먼저 확인하세요. 체크포인트로도 복구가 어려운 상황을 예방합니다.

자동 압축 — Claude가 알아서 하는 것

대화가 길어지면 Claude Code가 자동으로 이전 내용을 요약해서 압축합니다. 이 과정은 자동이라 별도 설정이 필요하지 않습니다. 직접 더 빠르게 압축하고 싶으면 /compact를 씁니다.

비용 관리

Claude는 메시지 수가 아니라 토큰 수로 과금됩니다. 핵심 원리 하나만 이해하면 비용을 크게 줄일 수 있습니다.

처음에 비용 청구서 보고 놀랐어요. 알고 보니 한 대화창에서 계속 작업하면서 맥락이 엄청 쌓인 게 원인이었습니다. Claude는 매 메시지마다 대화 처음부터 끝까지 다시 읽거든요. 대화가 길수록 기하급수적으로 비싸집니다.

핵심 원리

💸 비용이 오르는 구조
100번째 메시지 = 1번째부터 100번째까지 전체를 다시 읽음

대화가 길수록 → 매 응답이 더 비싸짐

큰 파일 여러 개를 계속 열어두면 → 매번 전체 파일 비용 발생
💚 비용을 낮추는 구조
작업마다 /clear로 새로 시작

길어진 대화는 /compact로 압축

꼭 필요한 파일만 @로 지정해서 열기

즉시 적용 가능한 절약 팁

초급

1. 파일 지정해서 요청하기

"코드 개선해줘" ❌ → "@auth.py 의 login 함수만 봐줘" ✅
범위가 좁을수록 Claude가 읽는 토큰이 줄어듭니다.

초급

2. 새 작업은 /clear로 시작

전혀 다른 주제로 넘어갈 때는 반드시 /clear. 이전 대화가 남아있으면 불필요한 맥락이 매번 비용을 냅니다. (자세한 사용법은 4편 참고)

초급

3. 대화가 길어졌다면 /compact

같은 세션에서 계속 작업해야 한다면 주기적으로 /compact. 대화를 요약·압축해서 토큰을 줄입니다. (자세한 사용법은 4편 참고)

초급

4. 질문 묶어서 한 번에

질문 3개를 따로 보내지 말고 한 번에 묶어서. 컨텍스트 로딩 비용이 3번 → 1번으로 줄어듭니다.

초급

5. 메시지 편집 활용

새 메시지를 보내는 대신 이전 메시지 수정 버튼으로 내용을 바꾸면 대화가 더 짧게 유지됩니다.

중급

6. Esc로 조기 차단

Claude가 엉뚱한 방향으로 가면 즉시 Esc. 잘못된 방향의 전체 응답이 완료될 때까지 기다리면 그만큼 비용이 더 나옵니다.

중급

7. Plan Mode 먼저 — 삽질 방지

Shift+Tab 2회로 계획 먼저 확인. 잘못된 방향을 실행하고 되돌리는 것보다 계획 단계에서 수정하는 게 훨씬 쌉니다.

중급

8. 세션 15~20개 메시지 후 새로 시작

같은 대화를 100개 메시지까지 이어가면 초반보다 몇 배 비싸집니다. 작업 단위로 끊어서 새 세션으로 이어가는 게 경제적입니다.

중급

9. CLAUDE.md 500줄 이하 유지

CLAUDE.md는 매 메시지마다 읽힙니다. 길수록 비용이 올라갑니다. 실제로 매번 필요한 규칙만 남겨두세요.

비용 확인 방법

명령어 누가 쓰나요 알 수 있는 것
/cost API 직접 사용자 이번 세션의 토큰 수와 예상 비용
/stats Pro·Max 구독자 사용 패턴, 누적 통계
Console 대시보드 API 사용자 월별 누적 비용, 지출 한도 설정
Claude Max 구독자라면
월 정액제이므로 API처럼 토큰별 과금은 없습니다. 하지만 사용량 한도가 있으니 /stats로 주기적으로 확인하고, /compact로 효율을 높이는 습관은 여전히 유용합니다.
