Skills — 반복 작업을 /명령어로

매번 "회의록을 이런 형식으로 정리해줘, 참석자 목록을 맨 위에 넣고, 액션아이템은 담당자별로…" 같은 설명을 반복하고 있나요? Skills를 사용하면 그 모든 설명을 레시피로 저장해두고 /meeting 한 마디로 꺼내 쓸 수 있습니다.

반복 설명의 비용

Claude와 일하면서 가장 피곤한 게 이겁니다:

월요일 "주간 보고서 써줘. 형식은 먼저 이번 주 핵심 성과를 3줄로, 그 다음 진행 중인 업무 목록, 마지막에 이슈 사항이야. 어조는 간결하게." 다음 주 월요일 "주간 보고서 써줘. 형식은 먼저 이번 주 핵심 성과를 3줄로, 그 다음 진행 중인 업무 목록…"

같은 작업인데 매번 같은 설명을 반복합니다. Claude는 이전 세션을 기억하지 못하기 때문입니다.

Skills가 해결하는 것

Skills는 이 반복 설명을 한 번만 작성해두는 시스템입니다. 파일로 저장되어 있어서 Claude가 매번 꺼내 씁니다.

Skills 없을 때

매번 긴 설명 타이핑. 형식이 조금씩 다르게 나옴. 지시가 길수록 토큰 소비도 큼.

Skills 있을 때

/weekly-report 한 줄. 항상 동일한 형식. 팀원과도 같은 결과물을 공유 가능.

어떤 작업이 Skills에 맞나

📋 보고서/문서

주간 보고, 회의록, 업무 일지 등 형식이 고정된 반복 문서

📱 콘텐츠 제작

SNS 게시물, 뉴스레터, 블로그 포스트 등 브랜드 톤이 필요한 작업

🌐 번역/변환

파일을 특정 형식으로 번역하거나 변환하는 작업

Skills란?

Skills는 Claude Code에 새 능력을 추가하는 재사용 가능한 워크플로우입니다. SKILL.md라는 파일을 만들면 Claude가 그것을 도구로 인식하고 /스킬이름으로 실행할 수 있습니다.

레시피 비유

요리 레시피와 똑같습니다:

요리 레시피

재료 목록, 순서, 조리법을 한 번 기록해두면 매번 "어떻게 만들지?" 고민 없이 그대로 따라할 수 있습니다.

SKILL.md

작업 절차, 형식, 규칙을 한 번 기록해두면 Claude가 매번 그대로 실행합니다. /weekly-report 한 번으로.

Skills 파일 구조

Skills는 폴더 하나와 그 안의 SKILL.md 파일로 구성됩니다:

폴더 구조
~/.claude/skills/ ← 내 모든 프로젝트에서 사용 (개인용) └── weekly-report/ └── SKILL.md ← 작업 절차를 담은 파일 .claude/skills/ ← 현재 프로젝트에서만 사용 └── meeting/ └── SKILL.md

SKILL.md 파일 구성

SKILL.md는 두 부분으로 나뉩니다:

📋 앞부분 (Frontmatter)

---로 감싼 설정 영역. 스킬 이름, 설명, 실행 방식을 정의합니다.

📝 본문

Claude에게 줄 지시사항. "이렇게 저렇게 해줘"를 적는 곳입니다. 자연어로 쓰면 됩니다.

SKILL.md 기본 구조
--- name: weekly-report ← 스킬 이름 (폴더명이 기본값) description: 주간 보고서를 작성합니다. 보고서가 필요할 때 사용. --- 여기에 Claude에게 줄 지시사항을 자유롭게 씁니다. $ARGUMENTS 는 실행할 때 추가로 전달하는 내용입니다. 예: /weekly-report 이번 주 완료 작업: A, B, C → $ARGUMENTS = "이번 주 완료 작업: A, B, C"
description이 핵심입니다. Claude는 이 설명을 보고 "언제 이 스킬을 쓸지" 판단합니다. "주간 보고서 써줘"라고 하면 description을 보고 자동으로 weekly-report 스킬을 실행합니다.
Step 2 / 5

Skills 만드는 방법

3단계면 됩니다. 회의록 정리 스킬을 예시로 만들어보겠습니다.

1

폴더 만들기

스킬 이름으로 폴더를 만듭니다. 개인용이면 ~/.claude/skills/ 아래에 만드세요:

터미널
mkdir -p ~/.claude/skills/meeting
2

SKILL.md 작성

~/.claude/skills/meeting/SKILL.md 파일을 만들고 아래 내용을 붙여넣으세요:

~/.claude/skills/meeting/SKILL.md
--- name: meeting description: 회의록을 정리합니다. 회의 내용을 메모해서 넘기면 구조화된 회의록으로 만들어줍니다. --- $ARGUMENTS 내용을 아래 형식으로 회의록을 작성해주세요: ## 회의 기본 정보 - 일시: (내용에서 파악, 없으면 "미확인") - 참석자: (내용에서 파악) ## 핵심 논의 내용 (3~5줄 요약) ## 결정 사항 (결정된 것들을 목록으로) ## 액션 아이템 | 담당자 | 할 일 | 기한 | |--------|--------|------| (담당자-할 일-기한을 표로 정리, 기한이 없으면 "미정")
3

실행해보기

Claude Code에서 바로 사용하세요:

Claude Code에서
/meeting 오늘 마케팅팀 회의. 참석: 김팀장, 이대리, 박사원. Q2 캠페인 예산 조정 논의. 김팀장이 예산 20% 삭감 결정. 이대리가 5/10까지 수정안 제출하기로 함.
Claude 출력 (예시)
## 회의 기본 정보 - 일시: 미확인 - 참석자: 김팀장, 이대리, 박사원 ## 핵심 논의 내용 Q2 캠페인 예산 조정을 논의했습니다. 전반적인 예산 감축이 필요하다는 방향에 합의했으며... ## 결정 사항 - Q2 캠페인 예산 20% 삭감 ## 액션 아이템 | 담당자 | 할 일 | 기한 | |--------|--------|------| | 이대리 | Q2 예산 수정안 제출 | 5/10 |
이게 전부입니다. 이제 회의가 끝날 때마다 메모를 붙여넣고 /meeting만 입력하면 됩니다.

자연어로도 실행됩니다

스킬 이름을 몰라도 됩니다. description이 잘 쓰여 있으면 Claude가 자동으로 맞는 스킬을 씁니다:

자연어 실행
"오늘 회의 내용 정리해줘. [회의 메모 붙여넣기]"
→ description을 보고 자동으로 /meeting 스킬 실행

SKILL.md 직접 작성이 어렵다면

Claude에게 바로 만들어달라고 해도 됩니다:

Claude에게
"~/.claude/skills/meeting/ 폴더에 SKILL.md 만들어줘. 회의 메모를 받아서 구조화된 회의록으로 만들어주는 스킬이야."
→ 폴더 생성 + SKILL.md 파일 자동 작성
공식 /skill-creator 플러그인도 있습니다. 인터뷰 방식으로 스킬을 만들어주는 공식 플러그인입니다. Claude Code에서 /skill-creator를 입력하면 됩니다. 별도 설치가 필요할 수 있습니다.
Step 3 / 5

실용 예시 4가지

비개발자 직장인이 자주 쓰는 스킬 예시입니다. 복사해서 그대로 사용하거나 내 업무에 맞게 수정하세요.

📊 주간 보고서 자동화 ▼

만들 위치: ~/.claude/skills/weekly-report/SKILL.md

--- name: weekly-report description: 주간 업무 보고서를 작성합니다. 이번 주 업무 내용을 정리할 때 사용. --- 아래 정보를 바탕으로 주간 보고서를 작성해주세요: $ARGUMENTS 출력 형식: ## 이번 주 핵심 성과 (3줄 이내) ## 진행 중인 업무 (업무명 / 진행률 / 다음 단계) ## 이슈 및 리스크 (있으면 작성, 없으면 "없음") ## 다음 주 계획 어조는 간결하고 사실 중심으로.

사용법:

/weekly-report 이번 주 완료: A 기획안 작성, B 고객 미팅. 진행: C 프로젝트 50% 완료. 이슈: 예산 승인 지연.
📱 SNS 콘텐츠 자동화 ▼

만들 위치: ~/.claude/skills/sns-post/SKILL.md

--- name: sns-post description: SNS 게시물을 작성합니다. 인스타그램, 링크드인, 트위터 포스팅이 필요할 때 사용. --- $ARGUMENTS 내용을 아래 세 가지 SNS용으로 변환해주세요: ### 인스타그램 - 친근하고 공감 가는 어조 - 300자 이내 - 관련 해시태그 5~8개 ### 링크드인 - 전문적이고 인사이트 있는 어조 - 400자 이내 - 독자가 배울 점 포함 ### 트위터/X - 핵심만 간결하게 - 140자 이내 각 버전을 구분선(---)으로 나눠 출력해주세요.

사용법:

/sns-post 우리 팀이 6개월 만에 신제품 출시에 성공했습니다. 어려운 상황에서도 팀원들이 끝까지 해냈습니다.
🌐 문서 번역 ▼

만들 위치: ~/.claude/skills/translate/SKILL.md

--- name: translate description: 파일이나 텍스트를 번역합니다. 번역이 필요할 때 사용. --- $ARGUMENTS를 번역해주세요. 규칙: - 제품명, 고유명사, 코드, 명령어는 원문 유지 - 마크다운 표, 목록, 링크 구조 보존 - 번역 결과만 출력 (설명 없이)

사용법:

/translate 영어로: 안녕하세요, 회의 참석 부탁드립니다.
📧 이메일 초안 작성 ▼

만들 위치: ~/.claude/skills/email-draft/SKILL.md

--- name: email-draft description: 이메일 초안을 작성합니다. 이메일 작성이 필요할 때 사용. --- 아래 목적으로 이메일 초안을 작성해주세요: $ARGUMENTS 규칙: - 수신자: 고객/파트너사면 정중하고 전문적인 어조 - 수신자: 내부 팀원이면 간결하고 직접적인 어조 - 제목 포함해서 출력 - 마지막에 "이 초안을 수정하거나 어조를 바꾸고 싶다면 말씀해주세요" 추가

사용법:

/email-draft 고객사 A에 5/20 미팅 제안. 목적은 Q3 협업 방향 논의.
SKILL.md는 내 말투로 써도 됩니다. "이렇게 해줘", "저 형식으로 출력해줘"처럼 자연어로 쓰면 됩니다. Claude가 이해하는 모든 지시가 스킬 본문이 될 수 있습니다.
Step 4 / 5

자주 묻는 것

Skills를 사용하면서 마주치는 질문들입니다.

스킬을 만들었는데 /스킬이름이 인식이 안 돼요 ▼

체크할 것:

1. 폴더 구조: ~/.claude/skills/스킬이름/SKILL.md 경로가 정확한지 확인하세요. SKILL.md는 대문자입니다.

2. Frontmatter: ---로 시작하고 끝나는 부분이 있는지, name과 description이 있는지 확인하세요.

3. Claude Code 재시작: 새로 만든 스킬은 Claude Code를 재시작해야 인식됩니다.

스킬 내용을 수정하고 싶어요 ▼

SKILL.md 파일을 직접 열어서 수정하면 됩니다. 텍스트 파일이라 메모장, VS Code 어디서든 편집 가능합니다.

또는 Claude에게 "meeting 스킬의 출력 형식을 수정해줘. 액션아이템에 우선순위 열도 추가해줘"라고 말하면 Claude가 파일을 직접 수정해줍니다.

기존 CLAUDE.md의 커스텀 명령어와 뭐가 달라요? ▼

이전에 .claude/commands/ 폴더에 만들던 커스텀 슬래시 커맨드가 Skills로 통합되었습니다. 기존 방식으로 만든 파일은 그대로 동작합니다.

새로 만들 때는 Skills 방식(.claude/skills/)을 권장합니다. SKILL.md에는 frontmatter로 더 세밀한 설정이 가능하고, Claude가 상황에 맞게 자동으로 선택해 실행할 수 있습니다.

스킬이 너무 많아지면 관리가 어렵지 않나요? ▼

맞습니다. 스킬이 많아지면 어떤 게 있는지 헷갈릴 수 있습니다.

관리 팁: /를 입력하면 전체 목록이 나옵니다. 스킬 description이 명확해야 목록에서 바로 알아볼 수 있습니다. 자주 안 쓰는 스킬은 폴더를 _archive/로 이동하거나 삭제해도 됩니다.

팀원과 스킬을 공유하려면 어떻게 하나요? ▼

공유하고 싶은 스킬을 ~/.claude/skills/ 대신 프로젝트 폴더 안의 .claude/skills/에 만드세요.

그러면 이 폴더를 Git으로 공유할 때 스킬도 함께 공유됩니다. 팀원이 같은 프로젝트를 열면 동일한 스킬을 쓸 수 있습니다.
