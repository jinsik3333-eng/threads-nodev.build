Hooks — 내 작업 지키는 안전장치

AI에게 파일 작업을 맡길 때 불안했던 적 있나요? Hooks는 Claude가 특정 명령을 실행하기 전에 자동으로 한 번 멈추게 하는 안전 규칙입니다. 위험한 실수를 줄이는 데 도움이 됩니다.

Hooks는 위험한 행동 전 안전벨트입니다

파일 삭제나 민감한 명령처럼 위험한 행동을 실행하기 전에 한 번 멈추게 합니다.

AI에게 일 맡기는 게 왜 무서운가

Claude Code는 강력합니다. 파일을 읽고 만들고 수정하고 삭제할 수 있습니다. 그런데 이런 일이 실제로 일어납니다:

실제 사고 사례
😱
"Downloads 폴더 정리해줘" 했더니 필요한 파일까지 영구 삭제 — 휴지통에도 없음
😱
"이 버전으로 되돌려줘" 했더니 그동안 작업한 내용이 전부 사라짐
😱
데이터 정리 작업 중 표 전체 삭제 — 복구 불가

Claude가 일부러 그러는 게 아닙니다. AI는 "하라는 대로" 실행하는 데 최선을 다합니다. 그래서 사람이 막아줄 안전장치가 필요합니다.

Hooks가 하는 일

Hooks는 Claude가 특정 행동을 하기 직전에 자동으로 확인합니다. "이 명령어는 위험하니 막아라"고 지정해두면 Claude가 실행 시도하는 순간 자동으로 차단됩니다.

🛡️ Hook이 없을 때

Claude가 위험한 명령어를 실행할지 말지 스스로 판단합니다. AI의 "판단"에 내 소중한 파일이 달려 있습니다.

✅ Hook이 있을 때

정해둔 위험 명령은 규칙으로 먼저 막습니다. AI 판단에만 맡기는 것보다 훨씬 안전하지만, 중요한 작업 전 백업은 여전히 필요합니다.

이 가이드에서 할 일

1. Safety Hook을 예시로 이해

위험한 명령을 실행하기 전에 멈추는 규칙입니다. 코드를 외우는 게 아니라 “이런 안전장치를 만들 수 있구나”를 이해하면 됩니다.

2. 메신저/알림 Hook은 Claude에게 만들게 하기

작업이 끝났을 때 Telegram, Slack, Discord, macOS 알림 등으로 알려주는 Hook은 직접 코딩하지 말고 Claude Code에게 시키는 방식으로 만듭니다.

이 편의 기준은 “내가 코드를 이해했나?”가 아닙니다.
“Claude에게 안전장치와 알림을 만들어달라고 말할 수 있나?”가 목표입니다.
섹션 1 / 6

Hook은 Claude에게 시켜서 만들면 됩니다

복잡한 설정을 먼저 외우지 마세요. 비개발자에게 가장 쉬운 방식은 “내가 원하는 안전장치/알림을 설명하고, Claude Code가 파일을 만들게 하는 것”입니다.

먼저 이렇게 말하세요

아래 문장을 Claude Code에 붙여넣으면 됩니다. Hook 용어를 몰라도, 원하는 결과를 말하면 Claude가 필요한 파일과 설정을 만들어줍니다.

Claude Code에 붙여넣기
내 Claude Code에 Hook을 설정하고 싶어. 나는 코딩을 잘 모르니까 필요한 파일과 settings.json 설정은 네가 만들어줘. 목표는 두 가지야. 1. 위험한 명령어는 실행 전에 막기 - rm, git reset --hard, git push --force, DROP TABLE 같은 명령은 차단 - 차단되면 왜 위험한지 한국어로 설명 2. 작업이 끝나면 알림 보내기 - 우선 macOS 알림으로 알려주기 - 나중에 Telegram/Slack/Discord 웹훅으로 바꾸기 쉽게 구조화 먼저 어떤 파일을 만들지 계획을 보여주고, 내가 승인하면 파일을 만들고 테스트 방법까지 알려줘.
메신저 알림을 붙이고 싶다면
웹훅 URL이나 토큰은 채팅에 그대로 노출하지 말고 환경변수로 저장하게 시키세요. 예: TELEGRAM_BOT_TOKEN, SLACK_WEBHOOK_URL.

이벤트 → 반응 구조

Claude Code는 다양한 동작을 할 때 "이벤트"를 발생시킵니다. Hook은 그 이벤트에 반응하는 자동 처리기입니다.

Claude가 행동 시도파일 삭제 명령어 실행
→
Hook 자동 실행위험 여부 확인
→
결과 판정허용(0) / 차단(2)
→
차단 or 실행메시지와 함께 전달

주요 이벤트 종류

이벤트언제 발생주요 용도
PreToolUse필수 Claude가 도구 실행하기 직전 위험한 명령어 차단
Stop필수 Claude가 응답 완료 시 작업 완료 알림
Notification Claude가 사용자 확인 필요 시 팝업/소리 알림
PostToolUse 도구 실행 직후 자동 포매팅, 로그
SessionStart 세션 시작 시 업무 컨텍스트 자동 주입

이 중에서 PreToolUse(차단)와 Stop(완료 알림) 두 개만 설정하면 대부분의 필요를 커버합니다.

Hook의 허용/차단 로직

✅ exit 0 — 허용

Hook 스크립트가 0으로 종료하면 Claude가 그대로 진행합니다.

🚫 exit 2 — 차단

Hook이 2로 종료하면 Claude가 멈추고 stderr 메시지를 받아 대안을 찾습니다.

설정 파일 위치

Hook은 settings.json이라는 파일에 등록합니다. 두 가지 위치가 있습니다:

🌐 전체 적용 (권장)

~/.claude/settings.json
내 컴퓨터의 모든 프로젝트에 적용됩니다. Safety Hook처럼 어디서나 필요한 규칙은 여기에 두세요.

📁 프로젝트 전용

.claude/settings.json
현재 폴더(프로젝트)에만 적용됩니다. 이 프로젝트만의 특수 규칙이 필요할 때 사용합니다.

처음엔 전체 적용(~/.claude/settings.json)만 써도 충분합니다. 이 파일 하나로 Safety Hook + 완료 알림 모두 설정합니다.

Safety Hook 예시 보기

여기부터는 “Claude가 만들어줄 수 있는 안전장치의 예시”입니다. 처음부터 코드를 이해하려고 하지 말고, 어떤 위험을 막는지와 테스트 방법만 보면 됩니다.

이 코드는 예시입니다. 직접 작성하기보다 Step 2의 프롬프트로 Claude에게 만들게 하는 쪽을 추천합니다. 아래는 결과물이 어떤 모양인지 확인하는 용도입니다.

설치는 3단계로만 보면 됩니다

Python 코드를 이해하려고 하지 않아도 됩니다. 파일 만들기 → 실행 권한 → 설정 연결 순서로 따라가면 됩니다.

3단계로 완성하기

1

스크립트 파일 만들기

파일을 직접 열어서 붙여넣는 대신, 아래 명령어를 터미널에 그대로 붙여넣으면 ~/.claude/hooks/block_dangerous.py 파일이 만들어집니다:

터미널에 그대로 붙여넣기 — block_dangerous.py 생성
mkdir -p ~/.claude/hooks cat > ~/.claude/hooks/block_dangerous.py <<'PY' #!/usr/bin/env python3 import json, re, sys # ✏️ 차단할 명령어 목록 — 필요하면 추가/제거 가능 BLOCKED = [ (r"\brm\s+", "rm 대신 trash를 쓰세요 (brew install trash)"), (r"git\s+reset\s+--hard", "git reset --hard는 작업을 영구 삭제합니다"), (r"git\s+push.*--force", "git push --force는 히스토리를 덮어씁니다"), (r"git\s+push.*\s-f\b", "git push -f는 히스토리를 덮어씁니다"), (r"git\s+clean\s+-.*f", "git clean -f는 파일을 영구 삭제합니다"), (r"DROP\s+(DATABASE|TABLE)", "DROP은 데이터를 영구 삭제합니다"), ] data = json.load(sys.stdin) if data.get("tool_name") != "Bash": sys.exit(0) cmd = data.get("tool_input", {}).get("command", "") for pattern, reason in BLOCKED: if re.search(pattern, cmd, re.IGNORECASE): print(f"🚫 차단됨: {reason}", file=sys.stderr) sys.exit(2) sys.exit(0) PY
이 화면에서는 코드를 이해하지 않아도 됩니다.
복사 → 터미널에 붙여넣기 → Enter 순서로 진행하면 파일 생성까지 한 번에 끝납니다.
2

실행 권한 부여

만든 파일에 실행 권한을 줍니다:

터미널
chmod +x ~/.claude/hooks/block_dangerous.py
3

settings.json에 등록

~/.claude/settings.json 파일을 열고(없으면 새로 만들고) 아래 내용을 붙여넣습니다:

~/.claude/settings.json (Safety Hook만)
{ "hooks": { "PreToolUse": [ { "matcher": "Bash", "hooks": [ { "type": "command", "command": "python3 ~/.claude/hooks/block_dangerous.py" } ] } ] } }

동작 확인

설정 후 Claude에게 이렇게 물어보세요:

Claude에게
"빈 테스트 폴더에서 sample-delete-test.txt를 rm 명령어로 삭제해줘"
Claude 응답 (예시)
🚫 차단됨: rm 대신 trash를 쓰세요 (brew install trash) rm 명령어가 차단되었습니다. trash 명령어를 대신 사용할게요: trash test.txt
차단 메시지가 뜨면 성공입니다. Claude가 rm 대신 휴지통(trash)으로 이동시킵니다. 실수해도 복구할 수 있습니다.

차단 패턴 추가하기

파일의 BLOCKED 목록에 원하는 패턴을 추가할 수 있습니다:

추가 예시 — block_dangerous.py의 BLOCKED 목록에 추가
# 프로덕션 서버 접근 차단 (r"ssh.*prod", "프로덕션 서버 직접 접근이 금지됩니다"), # 특정 폴더 삭제 차단 (r"rm.*invoices/", "invoices 폴더는 삭제할 수 없습니다"),

메신저/알림 Hook 만들기

Claude가 오래 걸리는 작업을 처리할 때 계속 쳐다보고 있을 필요가 없습니다. “끝나면 알려줘”를 Hook으로 만드는 편입니다.

먼저 Claude에게 이렇게 시키세요

코드를 직접 짜기보다, 원하는 알림 방식을 말하고 Claude가 만들게 하세요.

Claude Code에 붙여넣기
Claude Code 작업이 끝나면 나에게 알림을 보내는 Hook을 만들어줘. 나는 코딩을 잘 모르니까 필요한 파일과 settings.json 수정은 네가 해줘. 1차 목표: macOS 알림 또는 소리로 알려주기 2차 목표: 나중에 Telegram/Slack/Discord 웹훅으로 바꿀 수 있게 구조화하기 토큰이나 웹훅 URL 같은 비밀값은 코드에 직접 쓰지 말고 환경변수로 읽게 만들어줘. 먼저 계획을 보여주고, 내가 승인하면 설정 파일을 만들어줘.

Claude가 만들어줄 수 있는 가장 간단한 예시

아래는 “작업 완료 시 시스템 소리” 예시입니다. 처음에는 이런 설정이 생긴다고만 이해하면 됩니다:

~/.claude/settings.json (완료 알림만)
{ "hooks": { "Stop": [ { "matcher": "", "hooks": [ { "type": "command", "command": "afplay /System/Library/Sounds/Glass.aiff" } ] } ], "Notification": [ { "matcher": "", "hooks": [ { "type": "command", "command": "afplay /System/Library/Sounds/Frog.aiff" } ] } ] } }
Stop과 Notification에 다른 소리를 쓰는 이유: "Glass" 소리 = 작업 완료. "Frog" 소리 = 내 확인이 필요함. 소리만으로 상황을 구분할 수 있습니다.

사용 가능한 소리 목록

터미널 — 소리 파일 목록 확인
ls /System/Library/Sounds/
Basso.aiff Blow.aiff Bottle.aiff Frog.aiff Funk.aiff Glass.aiff Hero.aiff Morse.aiff Ping.aiff Pop.aiff Purr.aiff Sosumi.aiff Submarine.aiff Tink.aiff

원하는 소리로 Glass.aiff나 Frog.aiff 부분을 바꾸면 됩니다.

팝업 알림 예시

메신저 연동 전에는 화면 팝업만으로도 충분합니다. Claude에게 “소리 대신 팝업으로 바꿔줘”라고 말하면 됩니다:

팝업 알림 방식
{ "type": "command", "command": "osascript -e 'display notification \"작업이 완료되었습니다\" with title \"Claude Code\"'" }
완료 알림은 Claude가 오래 걸리는 작업을 맡길 때 진가를 발휘합니다. 수십 개 문서 정리, 긴 분석 작업 등 5분 이상 걸리는 일을 시킬 때 설정해두면 다른 업무를 병행할 수 있습니다.
섹션 4 / 6

최종 설정은 Claude에게 합치라고 시키세요

Safety Hook과 알림 Hook을 직접 합치려면 헷갈립니다. 비개발자는 Claude에게 “두 설정을 충돌 없이 합쳐줘”라고 시키는 방식이 더 안전합니다.

먼저 이렇게 말하세요

Claude Code에 붙여넣기
지금까지 만든 Safety Hook과 완료 알림 Hook을 하나의 ~/.claude/settings.json으로 합쳐줘. 기존 설정이 있으면 덮어쓰기 전에 백업 파일을 만들어줘. 조건: - 위험 명령 차단은 유지 - 작업 완료 알림도 유지 - JSON 문법 오류가 없게 검증 - 마지막에 /hooks로 확인하는 방법과 테스트 방법을 알려줘 내가 코드를 몰라도 따라할 수 있게 한 단계씩 진행해줘.

아래 코드는 참고용입니다. 직접 편집보다 위 프롬프트로 Claude에게 합치게 하는 것을 추천합니다:

~/.claude/settings.json 예시
{ "hooks": { "PreToolUse": [ { "matcher": "Bash", "hooks": [ { "type": "command", "command": "python3 ~/.claude/hooks/block_dangerous.py" } ] } ], "Stop": [ { "matcher": "", "hooks": [ { "type": "command", "command": "afplay /System/Library/Sounds/Glass.aiff" } ] } ], "Notification": [ { "matcher": "", "hooks": [ { "type": "command", "command": "afplay /System/Library/Sounds/Frog.aiff" } ] } ] } }

설정 체크리스트

  • mkdir -p ~/.claude/hooks 실행했음

  • chmod +x ~/.claude/hooks/block_dangerous.py 실행했음

1. 파일 경로 확인: ~/.claude/hooks/block_dangerous.py 경로가 정확한지 확인하세요. 터미널에서 ls ~/.claude/hooks/로 파일이 있는지 확인합니다.

2. 실행 권한 확인: chmod +x ~/.claude/hooks/block_dangerous.py를 다시 실행해보세요.

3. Claude Code 재시작: settings.json을 수정한 뒤엔 Claude Code를 완전히 종료하고 다시 시작해야 반영됩니다.

그래도 안 되면 Claude Code 안에서 /hooks를 입력해 등록 상태를 확인하세요.

Hook 때문에 정상적인 작업도 차단됩니다 ▼

block_dangerous.py 파일을 열고 BLOCKED 목록에서 문제가 되는 패턴을 제거하거나 수정하세요.

예를 들어 rm 명령어를 자주 써야 하는 작업이라면 해당 패턴을 주석 처리하면 됩니다:

# (r"\brm\s+", "rm 대신 trash를 쓰세요"), ← 앞에 # 붙이면 비활성화
소리 알림이 너무 시끄럽습니다 ▼

소리 대신 팝업으로 바꾸거나, 소리 볼륨을 낮추거나, Stop Hook 전체를 지워도 됩니다.

팝업으로 바꾸려면 afplay 명령어를 아래로 교체하세요:

"command": "osascript -e 'display notification \"작업 완료\" with title \"Claude Code\"'"
Hook이 뭔가 잘못 실행되는지 확인하고 싶어요 ▼

Claude Code를 디버그 모드로 실행하면 Hook 실행 로그를 볼 수 있습니다:

claude --debug

또는 Claude Code 안에서 Ctrl+O를 눌러 도구 실행 로그를 확인하세요.

이 두 가지 외에 유용한 Hook이 더 있나요? ▼

더 있습니다. 예를 들어:

민감 폴더 보호: customers/, invoices/ 같은 폴더를 수정 자체가 불가능하도록 막을 수 있습니다.

세션 시작 시 컨텍스트 주입: Claude가 새 세션을 시작할 때마다 업무 규칙 파일을 자동으로 읽게 할 수 있습니다.

프롬프트 기반 Hook: 단순 차단이 아니라 "판단이 필요한 상황"에 Claude에게 확인 질문을 던지게 할 수도 있습니다.

하지만 처음에는 Safety Hook + 완료 알림 두 개로 충분합니다. 익숙해지면 cc101.axwith.com의 Hooks 섹션을 참고하세요.

Cursor나 VS Code에서도 Hook이 동작하나요? ▼

기본적으로는 동작하지만 2026년 4월 현재 일부 이벤트(PostToolUse 등)에서 버그 보고가 있습니다.

이 가이드에서 설정한 PreToolUse(Safety Hook)와 Stop(완료 알림)은 GUI 환경에서도 정상 동작합니다. 가장 중요한 두 Hook이 안정적으로 지원됩니다.
