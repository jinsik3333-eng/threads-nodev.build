WHAT THIS KIT DOES

아이디어 하나를
30일 콘텐츠 일정표로 바꿉니다

“콘텐츠 아이디어 주세요”가 아니라, 실제로 30일 동안 올릴 수 있게 결과물의 형태를 지정합니다.

언제 쓰나바꿔 넣을 것복붙 프롬프트예시 입력결과물 기준검수 체크

언제 쓰는 프롬프트인가요?

이 프롬프트는 “오늘 뭐 올리지?”가 반복될 때 씁니다. 핵심은 Claude에게 단순 아이디어가 아니라 30일치 콘텐츠 운영안을 만들게 하는 것입니다.

계정이 멈췄을 때올릴 게 없는 게 아니라, 쪼개는 구조가 없는 상태일 수 있습니다.
주제는 있는데 막힐 때한 주제를 각도, 포맷, CTA로 나누면 생각보다 오래 씁니다.
외주/팀 없이 운영할 때혼자 운영할수록 매번 새로 생각하지 않는 시스템이 필요합니다.
주의: Claude가 준 30개를 그대로 다 올리는 게 목표는 아닙니다. 먼저 30개 재료를 뽑고, 그중 7~10개만 골라 다듬는 방식이 더 좋습니다.

먼저 이 5가지만 바꿔 넣으세요

아래 값을 정하지 않으면 Claude가 너무 일반적인 콘텐츠를 만듭니다. 완벽할 필요는 없고, 한 줄씩만 채우면 됩니다.

TOPIC
주제

예: 비개발자가 Claude Code로 자기 일을 도구화하는 법

AUDIENCE
독자

예: 코딩은 모르지만 AI로 업무를 줄이고 싶은 1인 사업자/마케터

PLATFORM
플랫폼

예: Instagram carousel, Threads, Reels

TONE
톤

예: 과장 없이 실무적, 하지만 첫 장은 결과 중심으로 짧게

GOAL
목표

예: 저장, 프로필 링크 클릭, 무료 HTML 키트 사용

RESOURCE
연결 리소스

예: 콘텐츠 30일치 프롬프트 키트 / 업무 지시서 템플릿

복붙용 메인 프롬프트

아래 프롬프트를 복사한 뒤, 대괄호 안의 값만 바꿔서 Claude에 붙여넣으세요.

MAIN PROMPT
너는 내 콘텐츠 전략가이자 편집장이다.

아래 주제 하나를 30일치 인스타그램/스레드 콘텐츠로 쪼개줘.
목표는 "그럴듯한 아이디어 목록"이 아니라, 실제로 업로드할 수 있는 콘텐츠 캘린더를 만드는 것이다.

[주제]
- 여기에 주제를 적는다.

[독자]
- 여기에 독자를 적는다.

[플랫폼]
- 예: Instagram carousel, Threads, Reels

[톤]
- 예: 과장 없이 실무적. 번역투 금지. 첫 문장은 짧고 결과 중심.

[목표]
- 예: 저장 유도, 프로필 링크 클릭, 복사 가능한 무료 HTML 프롬프트 키트 사용

다음 형식으로 정리해줘.

1. 먼저 이 주제를 5개의 큰 각도로 나눠줘.
   - 문제 공감
   - 오해 교정
   - 실전 방법
   - 예시/케이스
   - 체크리스트/템플릿

2. 30일치 콘텐츠 캘린더를 표로 만들어줘.
   각 행에는 아래 항목을 포함해줘.
   - Day
   - 콘텐츠 제목
   - 포맷: 캐러셀 / 짧은 글 / 릴스 / 체크리스트 / 케이스
   - 첫 장 후킹 문장
   - 핵심 메시지
   - CTA
   - 연결할 리소스 또는 프롬프트

3. 그중 가장 먼저 만들 7개를 골라줘.
   고른 이유도 한 줄씩 적어줘.

4. 첫 번째 콘텐츠는 바로 제작할 수 있게 더 자세히 풀어줘.
   - 6장 캐러셀 구조
   - 각 장 제목
   - 각 장에서 보여줄 요소
   - 캡션 초안
   - 댓글/저장/프로필 링크 CTA 후보

5. 마지막으로 전체 결과를 검수해줘.
   아래 기준에 맞지 않는 것은 고쳐줘.
   - 너무 흔한 AI 팁처럼 보이지 않는가?
   - 독자가 바로 실행할 수 있는가?
   - 자료 미끼처럼 보이지 않는가?
   - 내 계정의 관점이 분명한가?
   - 첫 장이 1초 안에 이해되는가?

예시 입력값

처음에는 아래 예시를 그대로 넣고 결과를 확인해보세요. 이후 주제와 독자만 바꾸면 됩니다.

EXAMPLE INPUT
[주제]
비개발자가 Claude를 이용해 콘텐츠 제작 시스템을 만드는 법

[독자]
코딩은 모르지만 인스타그램/스레드를 꾸준히 운영하고 싶은 1인 사업자, 마케터, 크리에이터

[플랫폼]
Instagram carousel 중심. 일부는 Threads 짧은 글과 Reels로 재활용.

[톤]
과장하지 않고 실무적. 하지만 첫 장은 결과가 바로 보이게 짧고 선명하게. 어색한 번역투 금지.

[목표]
저장 유도와 프로필 링크 클릭. PDF 자료집이 아니라 복붙 가능한 HTML 프롬프트 키트로 연결.

Claude가 만들어야 하는 결과물

좋은 답변은 아이디어 몇 개가 아니라, 바로 고르고 실행할 수 있는 운영안입니다.

1. 5개 콘텐츠 각도

문제, 오해, 방법, 케이스, 체크리스트처럼 주제를 오래 쓸 수 있게 나눕니다.

2. 30일 캘린더

Day, 제목, 포맷, 첫 장 후킹, CTA, 연결 리소스까지 있어야 합니다.

3. 우선 제작 7개

처음부터 30개를 다 만들지 말고, 먼저 만들 7개를 골라야 합니다.

4. 첫 콘텐츠 상세 구조

6장 캐러셀, 캡션, CTA까지 이어져야 실제 제작으로 넘어갈 수 있습니다.

결과물 검수 체크리스트

Claude가 답을 줬다면, 그대로 쓰기 전에 아래 기준으로 한 번 걸러주세요.

첫 장 제목이 1초 안에 이해되는가?
“AI 팁 모음”처럼 흔하지 않고, 내 계정의 관점이 들어갔는가?
각 게시글에 연결할 리소스나 다음 행동이 있는가?
30개 중 실제로 이번 주에 만들 7개가 분명한가?
자료를 받게 하는 것보다, 바로 써보게 하는 구조인가?
어색한 번역투나 과한 광고 문구가 없는가?

다음에 할 일

이 프롬프트로 30개가 나왔다면, 전부 만들려고 하지 말고 첫 7개만 고르세요. 그리고 첫 번째 게시글은 카드뉴스로, 같은 내용은 15초 영상으로 재활용할 수 있습니다.
