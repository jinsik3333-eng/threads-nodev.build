CLAUDE COWORK GUIDE · 같이 일하는 법

Claude에게
일을 맡기는 순서

“알아서 해줘”가 아니라, 어떤 일을 언제 실행하고 어떤 결과물로 확인할지까지 맡기는 법입니다. Scheduled, Live Artifacts, Dispatch도 실제 프롬프트로 연습해보세요.

1. Cowork 홈

새 작업을 만들고 프로젝트 맥락을 고르는 시작 화면입니다.

2. Projects

작업 폴더와 지시사항을 묶어 반복 업무의 맥락을 보관합니다.

3. Scheduled tasks

반복 보고, 정리, 점검 업무를 정해진 시간에 실행하도록 예약합니다.

4. Live artifacts

계속 열어보고 갱신하는 체크리스트, 요약, 대시보드를 만듭니다.

개인 정보나 실제 대화가 들어가지 않은 가이드용 예시 이미지입니다. 실제 Claude Desktop/Cowork 화면은 업데이트와 사용 환경에 따라 달라질 수 있습니다.

1. 질문하지 말고 업무를 맡기세요

“좋은 콘텐츠 만들어줘”는 질문에 가깝습니다. “누구에게 보여줄지, 어떤 자료를 쓸지, 어떤 형식으로 받을지, 무엇을 피할지”를 알려주면 업무가 됩니다.

2. 업무 지시서 6요소

역할
어떤 관점으로 볼지
목표
최종 결과물이 무엇인지
자료
참고할 원문, 메모, 파일
제약
길이, 톤, 금지 표현
출력 형식
표, 목록, 초안, 카드 구성
검수 기준
좋은 결과인지 판단할 기준

3. Scheduled: 정해진 시간에 반복 업무 맡기기

스케줄은 “한 번 잘 된 업무”를 반복시키는 기능입니다. 처음부터 자동화하지 말고, 아래 프롬프트를 수동으로 한 번 성공시킨 뒤 예약하세요.

1
수동으로 먼저 실행원하는 결과가 나오는지 한 번 확인합니다.
2
예약 만들기Claude Code CLI에서는 /schedule을 실행합니다. 웹/데스크톱에서는 예약 작업 만들기 메뉴가 보이면 거기서 시간과 반복 주기를 고릅니다.
3
확인 기준 넣기자동 실행 결과가 맞는지 판단할 체크리스트를 함께 적습니다.
매주 월요일 오전에 쓸 “팀 회의 준비 메모”를 만들어줘. 자료: [지난주 업무 메모 붙여넣기] 출력 형식: 1. 이번 주 회의에서 확인할 안건 5개 2. 결정이 필요한 항목 3. 내가 미리 준비할 자료 4. 회의 전에 팀에 보낼 짧은 메시지 검수 기준: - 추측하지 말 것 - 자료에 없는 내용은 “확인 필요”로 표시 - 10분 안에 읽을 수 있게 짧게
/schedule 작업 이름: 팀 회의 준비 메모 반복: 매주 월요일 오전 9시 할 일: 지난주 업무 메모를 바탕으로 이번 주 회의 안건, 결정 필요 항목, 준비 자료, 팀 공유 메시지를 만들어줘. 결과 확인 기준: 추측 금지, 확인 필요 표시, 10분 안에 읽히는 분량.

Claude Code 공식 문서 기준으로 예약 작업은 웹/데스크톱 또는 CLI의 /schedule 흐름에서 만들 수 있습니다. 실제 메뉴명은 환경에 따라 다를 수 있습니다.

4. Live Artifacts: 결과물을 보면서 고치기

Live Artifacts는 “답변을 읽는 것”보다 “결과물을 열어보고 수정하는 것”에 가깝습니다. 문서, 체크리스트, 표, 작은 화면 초안처럼 눈으로 확인해야 하는 작업에 쓰세요.

업무 체크리스트
아래 내용을 Live Artifact로 볼 수 있는 한 페이지 체크리스트로 만들어줘. 체크박스, 우선순위, 완료 기준을 넣어줘. 자료: [업무 절차 붙여넣기]
간단한 랜딩 페이지
아래 아이디어를 1페이지 HTML 아티팩트로 만들어줘. 모바일에서 먼저 보이게 하고, 버튼은 하나만 넣어줘. 아이디어: [서비스 설명 붙여넣기]
보고서 초안
아래 메모를 주간 보고서 아티팩트로 만들어줘. 섹션은 성과, 문제, 다음 액션, 확인 필요로 나눠줘. 메모: [메모 붙여넣기]
비교 대시보드
A안과 B안을 비교하는 간단한 대시보드 아티팩트를 만들어줘. 점수표, 장단점, 추천안을 한 화면에서 보이게 해줘. 자료: [비교 내용 붙여넣기]
핵심은 “만들어줘”에서 끝내지 않는 것입니다. 아티팩트를 본 뒤 “2번 항목을 더 짧게”, “표를 추가해줘”, “모바일 기준으로 다시”처럼 고치세요.

5. Dispatch: 지금 던지고 나중에 이어받기

Dispatch는 긴 작업을 지금 화면에서 붙잡고 있지 않고, 일단 맡긴 뒤 나중에 이어받는 감각으로 이해하면 쉽습니다. 이동 중에는 짧게 맡기고, 데스크톱이나 터미널에서 결과를 이어 확인하는 식입니다.

지금은 자세히 작업할 시간이 없어서, 이 일을 나중에 이어서 할 수 있게 정리해줘. 목표: [무엇을 끝내야 하는지] 현재 메모: [짧은 메모 붙여넣기] 필요한 자료: [나중에 찾아야 할 자료] 완료 기준: [끝났다고 볼 조건] 출력은 아래 형식으로 해줘. 1. 작업 요약 2. 지금 부족한 정보 3. 다음에 바로 실행할 첫 문장 4. Claude Code나 데스크톱에서 이어서 시킬 프롬프트
이전에 정리한 작업 요약을 이어서 진행해줘. 먼저 부족한 정보를 다시 확인하고, 그다음 바로 실행 가능한 3단계 작업 계획을 만들어줘. 내가 승인하기 전에는 실제 파일 수정이나 발행은 하지 마.

6. 업무 지시서 복붙 프롬프트

너는 내 업무 파트너야. 아래 일을 초안 → 검수 → 수정 순서로 같이 진행해줘.

역할: [예: 콘텐츠 기획자 / 회의록 정리 담당 / 고객문의 분류 담당]
목표: [최종 결과물]
자료: [붙여넣을 메모 또는 파일 설명]
제약: [말투, 길이, 금지할 표현, 꼭 포함할 내용]
출력 형식: [표 / bullet / 5장 구성 / 이메일 초안]
검수 기준: [좋은 결과물이라고 판단할 기준 3개]

먼저 초안을 만들고, 마지막에 “확인하면 좋은 점 3개”를 따로 적어줘.

7. 무엇을 맡겨야 할지 모를 때는 이 질문부터

아래 4개는 설명을 읽는 용도가 아니라, 바로 복사해서 Claude에게 붙여넣는 시작 질문입니다.

반복 업무 찾기

내가 매주 반복하는 업무를 보고, Claude에게 맡기기 좋은 순서로 5개만 골라줘. 위험한 업무는 제외해줘.

Scheduled 후보 만들기

이 업무를 매주 자동으로 돌린다면 입력 자료, 실행 시간, 결과 확인 기준이 뭐가 필요할지 표로 정리해줘.

Live Artifact로 확인하기

아래 내용을 바탕으로 한 페이지짜리 체크리스트 초안을 만들어줘. 내가 바로 고칠 수 있게 항목을 짧게 나눠줘. [내용 붙여넣기]

Dispatch처럼 짧게 맡기기

지금은 메모만 남길게. 나중에 작업할 수 있게 목표, 필요한 자료, 다음 행동 3개로 정리해줘. [메모 붙여넣기]

8. 같이 일하는 3단계

초안
일단 첫 결과물을 받습니다.
검수
좋은 결과인지 기준으로 확인합니다.
수정
부족한 기준을 한 줄 더 붙여 다시 시킵니다.

9. 결과가 별로일 때 바로 쓰는 문장

수정 지시도 고민하지 말고 필요한 문장만 복사해서 이어 붙이면 됩니다.

너무 길어. 핵심만 5줄로 줄여줘.
비개발자도 이해하게 쉬운 말로 바꿔줘.
표로 바꾸고, 실행할 일만 따로 빼줘.
추측한 내용은 지우고, 원문에 있는 내용만 남겨줘.
이 결과물을 검수 기준에 맞춰 다시 평가해줘.

10. 프로젝트/폴더가 필요한 순간

같은 업무를 반복하거나, 파일과 지시사항을 계속 이어가야 한다면 프로젝트나 작업 폴더가 필요합니다. Cowork 프로젝트는 관련 작업을 파일, 맥락, 지시사항, 메모리와 함께 묶는 작업공간으로 이해하면 쉽습니다.

  • 매번 같은 설명을 반복한다.
  • 참고 파일이 여러 개다.
  • 결과물을 다음 작업에서 다시 써야 한다.
  • Claude Code로 실제 파일을 만들거나 고치고 싶다.

11. Claude Code로 넘어가도 되는 기준

  • 만들고 싶은 결과물을 한 문장으로 설명할 수 있다.
  • 작업 폴더가 있다.
  • 성공 기준을 3개 이상 말할 수 있다.
  • Claude의 초안을 보고 무엇을 고칠지 말할 수 있다.

오늘 할 일

오늘 업무 하나를 골라 위 프롬프트에 넣어보세요. 완벽한 요청이 아니라, 초안 → 검수 → 수정까지 한 번 돌아보는 게 목표입니다.
