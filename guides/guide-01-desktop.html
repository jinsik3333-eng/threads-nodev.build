Claude Code 전에 왜 Desktop을 먼저 보나요?

이 시리즈의 주인공은 Claude Code입니다. 1편은 그 전에 계정 로그인과 Claude 화면 감각만 5분 안에 맞추는 준비 편입니다. CLI나 MCP가 낯선 초보자라면, Desktop 커넥터로 먼저 시작하는 편이 훨씬 쉽습니다.

Claude Desktop 첫 화면은 이렇게 생겼습니다

앱을 열었을 때 가운데 큰 입력창이 보이면 정상입니다. 작은 버튼 이름은 몰라도 괜찮고, 이 입력창에 내 업무를 자연어로 쓰면 됩니다.

초보자에게 Desktop이 쉬운 이유
Claude Desktop은 최근 커넥터(Connectors) 기능으로 Google Drive, Notion 같은 다른 프로그램과 더 쉽게 연결해 쓸 수 있습니다. 물론 CLI나 MCP를 통해 Claude Code와 더 깊게 연결할 수도 있지만, 처음 시작하는 분에게는 Desktop의 커넥터 방식이 훨씬 쉽고 빠릅니다.
저도 GPT, Gemini 다 써봤는데 Claude를 쓰기 시작하면서 뭔가 다르다고 느꼈습니다. 다른 AI들이 빠르게 답을 내놓는다면, Claude는 한 번 더 생각하는 느낌이랄까요. 비개발자인 저한테는 그게 더 맞았어요. 그 시작이 Claude Desktop이었습니다.

claude.ai랑 뭐가 다른가요?

이미 브라우저에서 claude.ai를 써봤다면, Claude Desktop은 그걸 앱으로 만든 것입니다. 기능은 거의 같은데, 쓸 때 느낌이 다릅니다.

🌐 claude.ai (브라우저)
크롬이나 사파리를 열고 → 탭 찾고 → 로그인 확인하고 → 질문.

파일을 쓰려면 직접 복붙해야 하고, 결과물도 직접 저장해야 합니다.
💻 Claude Desktop (앱)
앱으로 바로 열 수 있고, 커넥터로 Notion·Google Drive 같은 서비스와 쉽게 연결할 수 있습니다.

Claude Code처럼 깊게 쓰기 전, 초보자는 Desktop에서 “다른 앱 자료를 Claude가 참고한다”는 감각을 먼저 익히기 좋습니다.
처음엔 "그냥 브라우저로 쓰면 되지 않나?" 싶었는데, 앱으로 쓰기 시작하면서 달라졌어요. 브라우저 탭 찾아가는 게 생각보다 귀찮거든요. 뭔가 막히면 그냥 단축키 하나로 바로 물어볼 수 있고, 작업 흐름이 안 끊깁니다. 저한테는 그게 생각보다 컸어요.

어떤 분께 추천하나요?

⚡ 빠른 질문이 많은 분

이메일 다듬기, 단어 선택, 번역 등 짧은 질문을 자주 한다면 단축키로 바로 뜨는 앱이 훨씬 빠릅니다.

📋 여러 앱을 오가며 일하는 분

엑셀, 파워포인트, 이메일을 동시에 쓰는 분. 브라우저 탭 전환 없이 Claude를 쓸 수 있습니다.

🔌 다른 앱과 쉽게 연결하고 싶은 분

Notion, Google Drive 같은 업무 앱을 커넥터로 연결하면 복붙을 줄이고 Claude가 자료를 참고하게 만들 수 있습니다.

⚙️ Claude Code·Cowork를 앱으로 쓰고 싶은 분

터미널 없이도 Claude Code와 Claude Cowork를 Desktop 앱 안에서 바로 실행할 수 있습니다. 터미널이 익숙하지 않은 분께 특히 유용합니다.

시작 전 준비물 확인

아래 항목이 있으면 바로 시작할 수 있습니다.

  • 스마트폰용 앱은 별도입니다. 이 가이드는 PC/노트북 기준입니다.
  • 설치 파일 다운로드에 필요합니다. 설치 후에는 오프라인에서도 앱은 열립니다.
  • Anthropic 계정이 없으면 앱 설치 후 가입합니다. Step 3에서 안내합니다.
Claude.ai를 이미 쓰고 있다면
기존 계정을 그대로 씁니다. Claude Desktop 전용 계정을 따로 만들 필요 없습니다. 요금제도 동일하게 적용됩니다.

첫 대화 해보기

그냥 말하듯이 입력하면 됩니다. 처음에 이것부터 해보세요.

뭘 해볼 수 있을까요?

Claude Desktop은 파일 작업보다 대화 기반 작업에 특화되어 있습니다. 저는 처음에 이메일 다듬기부터 시작했어요. 가장 효과가 바로 보이거든요. 아래 중 하나를 입력해보세요.

예시 1 — 이메일 다듬기
다음 이메일을 더 예의 바르고 자연스럽게 다듬어줘.
"안녕하세요. 지난번 건 어떻게 됐나요? 빨리 답변 주세요."
안녕하세요.

지난번 말씀드렸던 건에 대해 여쭤보고 싶어서 연락드립니다. 바쁘신 중에 번거로우시겠지만, 가능하신 시간에 답변 주시면 감사하겠습니다.
예시 2 — 보고서 요약
아래 내용을 임원 보고용으로 3줄 요약해줘. 핵심만.
[보고서 내용 붙여넣기]
1. [핵심 결과 한 줄]
2. [주요 리스크 한 줄]
3. [권장 조치 한 줄]

알아두면 편한 기능들

⌨️ 단축키로 빠르게 열기

Mac: Cmd+Shift+Space (또는 앱 설정에서 변경 가능)
Windows: 시스템 트레이에서 아이콘 클릭
어떤 앱에서든 바로 Claude를 불러올 수 있습니다.

🗂️ 프로젝트 기능

"새 프로젝트" 를 만들면 그 안에서의 대화는 맥락이 유지됩니다. 업무별로 프로젝트를 나눠두면 매번 설명하지 않아도 됩니다.

📎 파일 첨부

PDF, Word, 이미지 파일을 드래그하거나 클립 아이콘으로 첨부하면 Claude가 내용을 읽고 분석합니다.

✍️ 스타일 지정

설정에서 응답 언어, 말투, 길이를 미리 지정해둘 수 있습니다. 매번 "한국어로 답해줘"라고 하지 않아도 됩니다.

이것만 주의하세요
회사 기밀, 고객 개인정보, 비밀번호는 입력하지 마세요. Claude와의 대화는 Anthropic 서버로 전송됩니다. 중요한 내부 정보는 사내 가이드라인을 먼저 확인하세요.
🎓 여기까지 왔다면
Claude 계정 준비와 Desktop 첫 대화를 완료했습니다. 이제 2편에서 Claude Code를 설치하면, 대화가 “내 컴퓨터 폴더에서 실제 결과물을 만드는 작업”으로 넘어갑니다.

막혔을 때 해결법

설치나 실행 중에 막히는 경우가 드물지만, 자주 나오는 상황 3가지를 정리했습니다.

자주 막히는 상황 3가지

❓ "열 수 없습니다. 개발자를 확인할 수 없습니다" (Mac) ▶

원인: Mac의 보안 기능(Gatekeeper)이 App Store 외부에서 받은 앱을 막는 것입니다. 정상입니다.

해결:

방법 1 — 우클릭으로 열기

Claude 앱 아이콘을 오른쪽 클릭(또는 Ctrl+클릭) → 열기 → 경고 창에서 다시 열기 클릭.

방법 2 — 시스템 설정에서 허용

시스템 설정 → 개인 정보 보호 및 보안 → "보안" 항목에서 "그래도 열기" 클릭.

한 번 허용하면 다음부터는 이 과정 없이 바로 실행됩니다.
❓ 로그인 후 브라우저에서 앱으로 돌아오지 않아요 ▶

원인: 브라우저가 앱으로 연결하는 딥링크를 막거나, 팝업이 차단된 경우입니다.

해결:

1. 브라우저에서 로그인을 완료합니다.

2. "Open Claude" 버튼이 보이면 클릭합니다. 팝업 차단 알림이 뜨면 허용합니다.

3. 그래도 안 된다면: Claude Desktop 앱을 직접 클릭해서 포커스를 가져오면 자동으로 로그인 상태가 됩니다.

앱을 완전히 종료 후 다시 실행해도 대부분 해결됩니다.

❓ 앱이 응답하지 않거나 빈 화면만 나와요 ▶

원인: 앱 캐시 문제이거나 인터넷 연결이 끊긴 경우입니다.

해결 순서:

1. 앱을 완전히 종료합니다.
Mac: Cmd+Q 또는 독 아이콘 우클릭 → 종료
Windows: 시스템 트레이 아이콘 우클릭 → 종료

2. 인터넷 연결을 확인합니다. (브라우저로 다른 사이트 접속 가능한지 확인)

3. 앱을 다시 실행합니다.

그래도 반복된다면 앱을 삭제 후 claude.ai/download에서 최신 버전을 다시 받으세요.

그래도 해결이 안 된다면
이 가이드 제작자 쓰레드 계정에 DM을 남겨주세요. 오류 화면을 스크린샷으로 찍어서 보내주시면 답변드립니다.

여기까지 왔다면 — 완료입니다 ✅
