# 일반앱 리워드 허브형 전환 설계

작성일: 2026-05-10
대상: 게임앱을 제외한 일반앱 전체
상태: 출시 전 공통 UX 기준안

## 결론

일반앱은 출시 전부터 **리워드 허브형 단일 홈 구조**를 기본값으로 잡는다.
앱별 기능은 다르더라도, 보상·광고·알림·크로스앱 이동 구조는 공통 컴포넌트로 통일한다.

핵심 목적은 광고를 억지로 끼워 넣는 것이 아니라, 사용자가 한 화면에서 계속 선택할 수 있게 만드는 것이다.

```text
기능 사용하기
→ 보상 받기
→ 광고 보고 추가 보상 받기
→ 다른 앱도 해보기
→ 알림 켜고 다시 방문하기
```

## 벤치마크에서 확인한 패턴

### 1. 알림 설정 모달

잘되는 리워드형 앱은 초기에 알림 권한을 단순히 요구하지 않고, 보상 맥락으로 설명한다.

예시:

- 복권 도착 알림
- 포인트 받을 기회 알림
- 오늘 퀘스트 리셋 알림
- 출석 보상 알림

권장 문구:

```text
알림 설정

보상 도착 알림
새로운 포인트 기회가 생기면 알려드려요.

오늘 퀘스트 알림
받을 수 있는 보상이 있으면 알려드려요.

[나중에] [알림 켜기]
```

### 2. 한 페이지 안의 리워드 허브

뎁스를 깊게 나누기보다, 한 화면에서 아래 행동을 모두 보여준다.

- 내 포인트/별/복권 수
- 오늘 받을 수 있는 보상
- 광고 보고 보상 받기
- 퀘스트 진행률
- 출석 상태
- 다른 앱으로 이동
- 교환/뽑기 CTA

### 3. 광고 보기 CTA 상시 노출

영상 광고는 숨기지 않고 항상 선택지로 둔다.

권장 CTA:

```text
광고 보고 5P 받기
복권 한 장 더 받기
오늘 보상 받기
결과 한 번 더 보기
```

주의:

- 강제 광고처럼 보이면 이탈 위험이 커진다.
- “보상형 선택지”로 보여야 한다.
- 광고 시청 가능 횟수와 쿨다운은 명확히 표시한다.

### 4. 크로스앱 이동 카드

한 앱에서 끝나지 않도록 다른 미니앱으로 이동할 수 있는 카드를 둔다.

예시:

```text
초성 퀴즈 풀고 포인트 받기
피부 테스트하고 포인트 받기
오늘 운세 보고 포인트 받기
계산 결과 보고 포인트 받기
```

## 일반앱 공통 화면 구조

```text
[상단 헤더]
- 앱 이름
- 알림 버튼
- 내 포인트 / 별 / 복권 수

[메인 기능 카드]
- 앱별 핵심 기능
- 오늘의 결과 / 테스트 / 계산 / 퀴즈 등

[보상 카드]
- 현재 보상 상태
- 광고 보고 포인트 받기
- 복권 뽑기 또는 추가 기회 받기

[퀘스트 카드]
- 오늘 퀘스트 진행률
- 기능 1회 사용하기
- 광고 1회 보기
- 다른 앱 1개 방문하기

[크로스앱 이동 카드]
- 다른 일반앱 3개 추천
- 각 앱별 보상 문구

[하단 고정 CTA]
- 광고 보고 5P 받기
- 또는 오늘 보상 받기
```

## 앱별로 달라지는 영역

공통 허브는 유지하고, 아래만 앱마다 바꾼다.

| 앱 유형 | 메인 기능 | 보상 연결 |
| --- | --- | --- |
| 퀴즈 앱 | 문제 풀기 | N문제 완료 후 포인트 |
| 테스트 앱 | 결과 보기 | 결과 확인 후 포인트 |
| 운세 앱 | 오늘 운세 보기 | 광고 보고 추가 운세 |
| 계산기 앱 | 계산 결과 보기 | 광고 보고 상세 결과 |
| 정보형 앱 | 카드/팁 보기 | 오늘 팁 확인 후 포인트 |

## 출시 전 적용 순서

1. 공통 리워드 정책 확정
2. 일반앱 공통 홈 템플릿 구현
3. 알림 모달 구현
4. 보상형 광고 CTA 구현
5. 크로스앱 이동 카드 구현
6. 대표 일반앱 1개에 적용
7. 나머지 일반앱에 메인 기능만 바꿔 적용

## MVP 범위

첫 배포에는 아래만 있어도 충분하다.

- 포인트/복권 상태 표시
- 메인 기능 카드
- 광고 보고 5P 받기 버튼
- 오늘 퀘스트 3개
- 다른 앱 이동 3개
- 알림 설정 모달
- 하단 고정 광고 CTA

## 주의사항

- 광고 버튼은 항상 보여주되, 강제 시청처럼 보이지 않게 한다.
- 알림 권한은 앱 진입 즉시보다 보상 맥락에서 요청한다.
- 포인트 정책은 앱별로 다르게 두지 말고 공통값으로 시작한다.
- 게임앱은 별도 루프가 있으므로 이 템플릿을 그대로 적용하지 않는다.
- 실제 구현 시 광고 SDK, 알림 권한, 딥링크 정책은 플랫폼 심사 기준을 우선한다.
