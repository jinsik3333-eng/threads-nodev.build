## Problem Statement

1~3편(설치 → 첫 프로젝트)을 마친 비개발자 직장인이 "이제 어떻게 더 잘 쓰냐"에서 막힌다.
설치는 됐고, 첫 도구도 만들었는데 — 매번 세션이 끊기고, 프롬프트가 왜 안 먹히는지 모르고,
돈이 얼마나 나가는지도 모른다. cc101(axwith.com/ko) 콘텐츠를 비개발자 직장인 기준으로
필터링하고, 레벨별로 올라가는 구조로 4~6편을 설계한다.

---

## What Makes This Cool

- 1~3편 완주자 전용 — "이미 써본 사람을 위한 다음 단계" 포지셔닝
- cc101 콘텐츠를 비개발자 언어로 재번역 (MCP·Hooks·Sub-agents 제외)
- 레벨 배지 개념: 입문(1~3편) → 기초(4편) → 중급(5편) → 심화(6편)
- 각 편 완주 시 "다음 레벨 예고"로 자연스러운 팔로우 유도

---

## 타깃

**1~3편 완주자** — Claude Code 설치 완료, 첫 프로젝트(크롤링/분석 도구) 경험 있음.
설치 방법은 알지만 "잘 쓰는 법"에서 막힌 비개발자 직장인.

---

## Constraints

- HTML 파일: 외부 의존성 없는 자기완결형 (1~3편과 동일 템플릿)
- 한국어, 반말 톤 (가이드 내부는 설명체 유지)
- cc101 내용 참고하되 직접 인용 금지 — 비개발자 언어로 재작성
- 스크린샷 없이 코드 블록 + 기대 출력 텍스트만
- 개발자 전용 섹션 완전 제외: MCP, Hooks, Sub-agents, GitHub Actions, Headless

---

## Premises

1. 1~3편 완주자가 막히는 지점은 "설치"가 아니라 "잘 쓰는 법"이다 ✅
2. CLAUDE.md·프롬프트·명령어를 따로 설명하는 것보다
   "업무 시나리오 속에서 언제 뭘 쓰는지" 보여주는 게 더 효과적이다 ✅
3. 4~6편도 여전히 비개발자 눈높이 — 개발자 용어 없이 ✅

## 4편 단계별 콘텐츠 명세

**파일명:** `guide-04-better-prompts.html`
**제목:** Claude 잘 쓰는 법 — 프롬프트·CLAUDE.md·명령어 완전 정복
**예상 소요:** 30분
**성공 기준:** 세션이 끊겨도 CLAUDE.md로 복구, /compact로 대화 정리, @ 파일 지정으로 정확한 작업 지시

```
Step 1: 소개
  - 1~3편 다 했는데 왜 잘 안 되는 사람들의 공통 실수 3가지
  - 이 가이드를 마치면 달라지는 것 (before/after)

Step 2: 프롬프트 제대로 짜기
  - "막연한 요청 vs 구체적 요청" 비교 예시 5개
  - @ 기호로 파일 직접 지정하는 법
  - Plan Mode (Shift+Tab 2번) — 언제 쓰는가
  - 기대 결과: Claude가 "파일을 읽겠습니다" 하고 시작하는 것

Step 3: CLAUDE.md 심화
  - 글로벌 CLAUDE.md vs 프로젝트 CLAUDE.md 차이
  - 업무 유형별 템플릿 4종 (리서치 / 문서처리 / 데이터 분석 / 콘텐츠)
  - CLAUDE.md에 넣으면 안 되는 것 (민감정보)
  - Claude에게 "CLAUDE.md 개선해줘" 부탁하는 법

Step 4: 명령어 실전
  - /compact — 언제 쓰는가 (토큰 80% 넘을 때)
  - /clear — 완전 초기화 (새 작업 시작)
  - /cost — 지금까지 얼마 썼는지 확인
  - --continue / --resume — 세션 이어서 시작하기
  - 키보드: Esc(취소), Shift+Tab 2회(Plan Mode)

Step 5: 세션 관리 실전
  - 컨텍스트가 뭔지 비유로 설명 (단기기억 vs CLAUDE.md 장기기억)
  - 긴 작업을 쪼개는 법 (파일 분리 → 세션 분리)
  - 체크포인트 (Esc 2회) — 실수 전으로 되돌리기

Step 6: 막혔을 때 FAQ
  - Claude가 엉뚱한 파일을 건드려요 → @ 지정
  - 대화가 갑자기 느려졌어요 → /compact
  - 이전에 했던 작업 방식을 잊어버려요 → CLAUDE.md 업데이트
  - 비용이 갑자기 많이 나와요 → /cost + 모델 확인
```

## 5편 단계별 콘텐츠 명세

**파일명:** `guide-05-workspace.html`
**제목:** 나만의 워크스페이스 — 업무별 세팅 + 비용 관리
**예상 소요:** 25분
**성공 기준:** 업무별 폴더 3개 이상 세팅, 월 사용 예산 파악

```
Step 1: 소개
  - "도구가 있는데 왜 매번 설명해야 하나" — 워크스페이스 개념
  - 이 가이드 마치면: 폴더 켜면 Claude가 바로 맥락 파악

Step 2: 업무별 폴더 구조 설계
  - 폴더 = 업무 단위 (meeting-summary / sales-analysis / weekly-report)
  - 각 폴더마다 CLAUDE.md + input/ + output/ 구조
  - 실제 폴더 만들기 실습 (터미널 명령어 + 대화로 만들기 두 가지)

Step 3: 워크스페이스 CLAUDE.md 고도화
  - 리서치 전용 워크스페이스 예시
  - 데이터 분석 전용 워크스페이스 예시
  - 콘텐츠 제작 전용 워크스페이스 예시
  - "Claude에게 이 워크스페이스 더 좋게 만들어달라고 부탁하기"

Step 4: 유용한 기능 모음
  - 이미지 입력 — 캡처 화면 붙여넣기로 작업 지시
  - 음성 입력 — 받아쓰기로 긴 지시 편하게
  - 체크포인트 (Esc 2회) 활용법 심화
  - 자동 압축 — Claude가 알아서 처리하는 것

Step 5: 비용 관리
  참고: https://www.youtube.com/watch?v=afr2jTsRSMc ("평생 써먹는 클로드코드 토큰 절약 팁 52가지")
       + 공식 문서(code.claude.com/docs/ko/costs)

  핵심 개념: Claude는 메시지 수가 아니라 토큰 수로 과금된다.
  매 메시지마다 대화 처음부터 끝까지 다시 읽는다 → 대화가 길수록 기하급수적으로 비쌈

  [초급 — 바로 적용 가능]
  1. 모델 고정: settings.json에서 기본 모델 Sonnet으로 설정
  2. 파일 지정: "코드 개선해줘" ❌ → "auth.ts 파일의 login 함수만 봐줘" ✅
  3. 메시지 편집: 새 메시지 보내는 대신 이전 메시지 편집 버튼으로 수정 (컨텍스트 누적 방지)
  4. 질문 묶기: 질문 3개를 따로 보내지 말고 한 번에 → 컨텍스트 로딩 1번으로 줄임
  5. /clear 타이밍: 새 작업 시작할 때마다 — 오래된 컨텍스트가 토큰 낭비
  6. /compact: 대화가 길어지면 압축 — "/compact 코드 변경사항 위주로 요약해줘"
  7. Extended Thinking 끄기: 단순 작업엔 불필요 → /config에서 비활성화 또는 낮추기
  8. CLAUDE.md 500줄 이하 유지

  [중급 — 습관 들이면 효과 큼]
  9. 하이브리드 모드: 계획 수립은 Opus, 실제 실행은 Sonnet으로 전환
  10. Esc로 조기 차단: Claude가 엉뚱한 방향 가면 즉시 Esc → 불필요한 토큰 차단
  11. Plan Mode 먼저: Shift+Tab 2회 → 계획 보고 승인 후 실행 (삽질 방지)
  12. 세션 15~20개 메시지마다 새로 시작: 100번째 메시지는 1번째보다 훨씬 비쌈
  13. 피크 타임 피하기: 미국 업무 시간(한국 밤 10시~새벽 4시)에 속도 저하

  [비용 추적]
  - /cost — API 사용자용 (토큰·금액 확인)
  - /stats — Pro/Max 구독자용 (사용 패턴 확인)
  - 월 예산: Console에서 워크스페이스 지출 한도 설정 (API 사용자)

Step 6: 막혔을 때 FAQ
  - 비용이 예상보다 많이 나와요 → 토큰 절약 체크리스트
  - 이미지를 붙여넣었는데 인식을 못해요 → 파일 형식 확인
  - 워크스페이스 CLAUDE.md가 너무 길어졌어요 → 핵심만 남기기
```

## 6편 단계별 콘텐츠 명세

**파일명:** `guide-06-mcp.html`
**제목:** Claude를 더 강하게 — 공식 플러그인 & 서비스 연동
**예상 소요:** 40분
**성공 기준:** 공식 플러그인 or MCP 1개 이상 연결, Notion/Gmail/Google Drive 중 1개와 Claude가 직접 대화하는 것 경험

```
Step 1: 소개 — Claude가 내 앱들과 직접 대화하면?
  - "Claude에게 내 Notion을 읽게 하면 어떻게 될까?"
  - 플러그인 vs MCP 차이 간단 비유
    - 플러그인: Claude.ai 웹에서 쓰는 앱 연동 (클릭 몇 번)
    - MCP: Claude Code에서 쓰는 더 강력한 연동 (설정 파일)
  - 비개발자 추천 경로: 플러그인 먼저 → 익숙해지면 MCP

Step 2: Claude 공식 플러그인 소개 (claude.ai 웹)
  - claude.ai → Settings → Integrations에서 연결 가능
  - 공식 플러그인 목록:
    1. Notion — 노션 페이지/DB 읽기·쓰기
    2. Gmail — 이메일 읽기·초안 작성
    3. Google Drive — 드라이브 파일 읽기·검색
    4. Google Calendar — 일정 조회·생성
  - 연결 방법: OAuth 인증 (구글/노션 로그인) → 권한 허용
  - 기대 결과: "내 Notion 2025 전략 문서 요약해줘" → Claude가 직접 읽어서 답변

Step 3: 실전 연결 — Notion 연동
  - claude.ai에서 Notion 플러그인 설치 순서 (스크린샷 없이 텍스트 설명)
  - 첫 번째 테스트 프롬프트:
    "내 Notion에서 [페이지 이름] 찾아서 3줄 요약해줘"
  - 활용 예시:
    - 회의록 페이지 → Claude가 읽고 실행 항목 정리
    - 프로젝트 DB → Claude가 필터링해서 이번 주 할 일 추출

Step 4: 실전 연결 — Gmail 연동
  - Gmail 플러그인 설치 및 Gmail 계정 연결
  - 첫 번째 테스트 프롬프트:
    "오늘 받은 이메일 중 답장이 필요한 것 골라줘"
  - 활용 예시:
    - "지난주 [고객사] 관련 이메일 전부 찾아서 타임라인 정리해줘"
    - "이 이메일에 정중한 거절 답장 초안 써줘"
  - 주의: 이메일 발송은 Claude가 직접 하지 않음 — 초안만 작성

Step 5: 실전 연결 — Google Drive 연동
  - Google Drive 플러그인 설치 및 Google 계정 연결
  - 첫 번째 테스트 프롬프트:
    "내 드라이브에서 [파일명] 찾아서 핵심 내용 요약해줘"
  - 활용 예시:
    - 드라이브 속 엑셀 → Claude가 읽고 분석
    - 여러 문서 → Claude가 비교·정리
  - Filesystem MCP와 차이: Drive는 클라우드, Filesystem은 로컬 PC

Step 6: Claude Code에서 MCP로 더 강하게 (선택 심화)
  - MCP란: Claude Code 터미널에서 쓰는 공식 연동 프로토콜
  - 공식 MCP 목록 (Anthropic 제공):
    1. Filesystem — 내 PC 폴더 직접 접근
    2. Web Search (Brave) — 실시간 웹 검색
    3. Google Drive MCP — 드라이브 더 깊은 연동
  - 설정 방법: claude_desktop_config.json 수정
    (파일 위치 안내 + "Claude에게 설정 도움 요청"하기)
  - 비개발자 팁: "MCP 설정 파일 어떻게 고쳐?" → Claude에게 물어보면 됨

Step 7: 막혔을 때 FAQ
  - 플러그인 메뉴가 어디 있는지 모르겠어요
  - 연결했는데 Claude가 파일을 못 찾아요
  - 어떤 걸 먼저 연결해야 할지 모르겠어요
  - 내 데이터가 Anthropic에 저장되나요? (보안 걱정)
  - Claude Code MCP와 claude.ai 플러그인 어떻게 달라요?
```

## Approaches Considered

### Approach A: 4편에 전부 몰기
4~6편 내용을 단일 "심화 가이드" 1편으로 통합.
- 빠르게 배포 가능, 하지만 너무 길고 소화 어려움
- 팔로우 유인 없음 (1편으로 끝나면 다음 게시물이 없음)

### Approach B: 3편 분리 (선택) ✅
4편(프롬프트·명령어) → 5편(워크스페이스) → 6편(플러그인)으로 분리.
- 각 편이 30분 이내로 소화 가능
- "다음 편 예고"로 팔로우 & 재방문 유도
- 1~3편과 동일한 템플릿 재사용 가능

### Approach C: 레벨 배지 시스템 앱
완주 시 레벨 배지 표시, 진행률 저장.
- 강한 차별화지만 현재 오버스펙
- 4~6편 반응 보고 7편 이후에 고려

## Recommended Approach
