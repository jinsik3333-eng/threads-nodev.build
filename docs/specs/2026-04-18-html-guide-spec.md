## Problem Statement

클로드 코드 사용법을 알려주는 계정은 많지만, 코딩 경험이 전혀 없는 비개발자 직장인을 위해
"진짜 처음부터" 가르치는 한국어 콘텐츠는 없다. 터미널이 뭔지도 모르는 사람이 클로드 코드를
처음 열기까지 필요한 모든 단계를 다루는 가이드가 목표다.

배포 형태: 쓰레드 계정 포스트 + 다운로드 가능한 자기완결형 HTML 가이드 파일.

## What Makes This Cool

- HTML 파일은 로그인 없이, 광고 없이, 오프라인에서도 열린다. 유튜브/노션보다 마찰이 0에 가깝다.
- 클로드 데스크탑에서 시작해서 클로드 코드로 심화하는 "진입로" 구조 — 터미널이 두려운 사람도
  처음엔 데스크탑으로 온보딩하고, 준비되면 코드로 넘어온다.
- 타깃이 극도로 구체적: "코딩 경험 없는 비개발자 직장인". 이 구체성이 바이럴 포인트가 된다.
  "나를 위해 만든 것 같다"는 반응을 목표로.

## Constraints

- 첫 번째 가이드 파일 경로: `~/Downloads/guide.html` (상대 경로 기준 `$HOME/Downloads/guide.html`)
- 한국어 콘텐츠
- HTML 파일은 외부 의존성 없는 완전 자기완결형 (CDN, 서버 필요 없음)
- 클로드 코드가 메인이지만 클로드 데스크탑도 초반 진입로로 포함

## Premises

1. 클로드 코드를 써보고 싶은 비개발자 직장인이 실제로 존재하며,
   이들을 위한 "진짜 처음부터" 한국어 콘텐츠가 현재 없다.

2. HTML 파일 형태가 유튜브/노션보다 초보자에게 마찰이 적다.
   (로그인 없음, 광고 없음, 오프라인 가능, 디자인 완전 통제)

3. 쓰레드가 한국 직장인 타깃의 배포 채널로 적합하다.

4. 계정은 "클로드 데스크탑 소개 → 클로드 코드 설치 → 첫 프로젝트"
   진입로 구조를 가진다. 클로드 코드가 메인이지만 데스크탑으로
   처음엔 더 넓은 초보자를 수용한다.

## Approaches Considered

### Approach A: MVP — 가이드 1개
1개 HTML 파일만 만들어서 반응 본다. 빠르지만 계정 느낌이 안 나고
공유 시 맥락 부족.

### Approach B: 3편 패키지 (선택)
"클로드 데스크탑 소개 → 클로드 코드 설치 → 첫 프로젝트"
3개 HTML + 3개 쓰레드 포스트. HTML 템플릿 하나를 재사용한다.
계정의 성격과 시리즈 느낌이 잡힌다.

### Approach C: 인터랙티브 가이드 앱
클릭 기반 진행, 퀴즈, 진행도 체크. 차별화 강하지만 지금 당장은 오버스펙.
2-3편 운영 후 인기 있으면 고려.

## Recommended Approach

**Approach B — 3편 패키지**

이유: 1개는 "계정"이 아니다. 쓰레드에서 팔로우할 이유가 생기려면 최소 3편의 시리즈가 있어야 한다.
템플릿 하나 만들면 이후 가이드는 콘텐츠만 채우면 된다.

### 3편 구성

| 편 | 주제 | 파일명 |
|----|------|--------|
| 1편 | 클로드 데스크탑 설치 & 첫 대화 | `guide-01-desktop.html` |
| 2편 | 클로드 코드 설치 & 터미널 첫 걸음 | `guide-02-claude-code-install.html` (= `guide.html`) |
| 3편 | 첫 번째 프로젝트 만들기 | `guide-03-first-project.html` |

### HTML 가이드 파일 구조 (각 편 공통)

```
헤더: 제목 + "몇 분이면 됩니다" 예상 소요 시간
↓
준비물 체크리스트 (맥/윈도우 표시)
↓
단계별 섹션 (각 단계: 스크린샷 영역 + 실제 명령어 + "이게 보이면 됩니다" 기대 결과)
↓
"막혔을 때" FAQ (2-3개)
↓
다음 편 예고
```

## Resolved Decisions

### 배포 채널 (Threads 제약 해결)
Threads는 포스트 내 클릭 가능한 링크를 지원하지 않으며 파일 첨부도 불가하다.
**해결책:**
- GitHub Pages에 HTML 파일 호스팅 → 영구 URL 확보
- 쓰레드 계정 바이오에 GitHub Pages URL 고정
- 각 포스트 마지막에 "링크는 바이오에 있습니다" 문구 포함
- HTML 파일 자체는 카카오톡/슬랙으로도 직접 공유 가능 (이게 오프라인 공유 경로)

### 스크린샷 처리 정책
실제 스크린샷 캡처 없이 **코드 블록 + 기대 출력 텍스트**로만 처리한다.
이유: (1) 업데이트 시 스크린샷 교체 비용이 크다 (2) 텍스트 표현이 더 선명하게 지시할 수 있다
(3) HTML 파일 크기를 가볍게 유지할 수 있다. 실제 UI 변화가 크거나 혼란이 예상되는 1-2개 단계에만
나중에 스크린샷 추가를 고려한다.

### guide.html 파일명 정책
2편 원본 파일명: `guide-02-claude-code-install.html`
즉시 배포용 복사본: `/Users/jinsik/Downloads/guide.html` (= 2편과 동일 내용)
개발 시 `guide-02-claude-code-install.html`을 작성 후 `guide.html`로 복사한다.

### HTML 디자인 스펙
- CSS: `<style>` 태그 인라인, CDN 없음
- 폰트: 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)
- 모바일 대응: `viewport` 메타 태그 + max-width 680px 컨테이너
- 색상: 흰 배경, 진한 텍스트, 코드 블록은 연한 회색 배경
- "다음 편 예고": 텍스트 URL (클릭 링크 아님 — 오프라인 열람 시 깨지지 않음)

### OS 분기 처리
Mac/Windows 모두 지원하되 기본은 Mac 기준으로 작성한다.
Windows 분기가 있는 단계는 "Windows 사용자는 여기로" 접이식 섹션으로 표시한다.
(`<details><summary>Windows 사용자</summary>...</details>`)

## 2편 단계별 콘텐츠 명세 (claude-code-install)

2편의 성공 기준: 터미널에 `claude` 입력 → Claude Code UI가 뜨는 것

```
단계 1: 터미널 열기
  - Mac: 스포트라이트(Cmd+Space) → "터미널" 입력
  - 기대 결과: 검은(또는 흰) 창이 뜸
  - 막혔을 때: 터미널 앱을 직접 찾는 방법

단계 2: Node.js 설치 여부 확인
  - 명령어: node --version
  - 기대 결과: v18 이상의 버전 번호 출력
  - 없으면: nodejs.org에서 LTS 버전 다운로드
  - Windows 엣지 케이스: 설치 후 터미널을 닫고 다시 열어야 PATH 반영됨.
    PowerShell 정책 오류 시: `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 입력 후 Y

단계 3: Claude Code 설치
  - 명령어: npm install -g @anthropic-ai/claude-code
  - 기대 결과: 여러 줄 로그 후 완료
  - 막혔을 때: permission 오류 시 sudo 없이 해결하는 방법

단계 4: 첫 실행
  - 명령어: claude
  - 기대 결과: 로그인 안내 또는 Claude Code UI 진입
  - 막혔을 때: "command not found" 오류 해결

단계 5: Anthropic 계정 연결
  - 브라우저가 열리며 로그인 → 인증 완료
  - 기대 결과: 터미널에 대화 입력창 등장
```

## Success Criteria

- 2편 기준: 코딩 무경험자가 `guide.html` 열고 따라하면 터미널에서 `claude` 실행 후
  대화 입력창이 뜨는 것까지 도움 없이 완료 가능
- 전체: 쓰레드 포스트 3개가 시리즈로 연결된다고 느껴진다
- HTML 파일이 외부 의존성 없이 완전 동작 (인터넷 없이도 열림)

## What I noticed about how you think

- "사용법을 알려주는 사람들은 많은데, 진짜 진짜 처음 초보자들이 어떻게 써야 하는지를 처음부터"
  — 갭을 정확히 짚었다. 시장을 관찰해서 구멍을 찾는 사람의 말투다.
- 클로드 데스크탑을 언급한 것 — 처음에 묻지도 않았는데 스스로 보완했다.
  "터미널이 어려운 사람도 있다"는 걸 이미 생각하고 있었다. 타깃에 대한 공감이 있다.
- 형식 질문에 "너의 추천은?"이라고 답한 것 — 내가 모르는 건 전문가한테 묻겠다는 실용적인 태도.
  좋은 빌더들이 다 그렇다.
