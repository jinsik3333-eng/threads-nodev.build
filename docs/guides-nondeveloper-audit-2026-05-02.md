# 비개발자용 가이드 수정 제안 — 2026-05-02

대상: https://jinsik3333-eng.github.io/threads-nodev.build/guides/

## 한 줄 평가

전체 구조와 톤은 좋지만, 2편 이후부터 비개발자에게는 “따라 하기”보다 “공부해야 할 것”처럼 느껴질 위험이 있다. 특히 Claude Code 설치, MCP, Skills, Hooks는 용어와 명령어가 빠르게 늘어나므로 완충 장치가 필요하다.

## 가장 큰 수정 방향

현재 가이드 흐름:

```text
도구 소개 → 설치 → 프로젝트 → 프롬프트/설정 → MCP/Skills/Agents/Hooks
```

추천 흐름:

```text
불안 제거 → 최소 성공 경험 → 반복 가능한 작업 패턴 → 고급 기능은 선택 사항
```

즉, 비개발자에게는 “이걸 다 배워야 합니다”가 아니라 “오늘은 이것 하나만 성공하면 됩니다”로 느끼게 해야 한다.

## 우선 수정해야 할 5가지

### 1. 각 편 첫 화면에 “오늘 끝나면 되는 것” 추가

각 가이드 상단에 아래 3개를 고정한다.

- 오늘의 목표
- 끝났다는 증거
- 안 해도 되는 것

예시 — Claude Code 설치 편:

```text
오늘의 목표: Claude Code를 열고 “안녕”까지 입력해보기
끝났다는 증거: 터미널에 Claude 답변이 보인다
오늘 안 해도 되는 것: MCP, GitHub, 자동화, 설정 파일
```

### 2. “처음이면 여기까지만” 경계선 추가

비개발자는 전체 페이지를 보면 부담을 느낀다. 각 편 중간에 명확히 끊어줘야 한다.

```text
처음이면 여기까지만 해도 성공입니다.
아래 내용은 익숙해진 뒤 다시 보세요.
```

특히 2, 4, 5, 6, 7, 9편에 필요하다.

### 3. 용어 사전/비유 박스 강화

현재 비유가 있긴 하지만, 고급 편으로 갈수록 용어 밀도가 높다.

필수 용어는 처음 나올 때 항상 이렇게 처리한다.

```text
MCP = Claude가 다른 앱을 만질 수 있게 해주는 연결잭
CLI = 마우스 대신 글자로 명령하는 창
Hook = 위험한 행동을 하기 전에 멈추는 안전벨트
Skill = 매번 반복하는 설명을 저장해둔 레시피
Agent = 일을 나눠 맡기는 AI 팀원
```

### 4. “복붙용 명령어”와 “설명”을 분리

비개발자는 코드블록을 보면 겁먹는다. 명령어 앞에 항상 이 문장을 붙인다.

```text
아래 줄은 이해하려고 하지 말고 그대로 복사해서 붙여넣으면 됩니다.
```

그리고 명령어 아래에는:

```text
이 명령어가 하는 일: meeting이라는 스킬 폴더를 만듭니다.
```

### 5. 고급 기능은 ‘필수 코스’가 아니라 ‘잠금 해제’로 표현

6~9편은 비개발자에게 어려울 수 있다. 제목과 설명을 조금 더 낮춰야 한다.

현재:
- MCP 공식 플러그인 연동
- Skills — 반복 작업을 /명령어로
- Sub-agents — AI가 AI팀을 만들다
- Hooks — 내 작업 지키는 안전장치

추천:
- MCP: Claude가 Notion/Gmail을 직접 보게 하기
- Skills: 자주 하는 일을 버튼처럼 만들기
- Agents: 일이 많을 때 AI에게 나눠 맡기기
- Hooks: AI가 위험한 실수를 못 하게 막기

## 편별 수정 제안

### 1편 Claude Desktop
- 가장 쉬운 입문편이라 좋다.
- 수정: “Claude Desktop으로 먼저 충분하다”는 안심 문장 추가.
- 목적: 처음부터 Claude Code로 넘어가지 않아도 된다는 안정감.

### 2편 Claude Code 설치
- 가장 이탈 위험이 높다.
- 이유: 길이 13,000자 이상, 코드/명령어 40개 수준, 터미널/Node/npm 등 용어가 많다.
- 수정:
  - Mac/Windows 경로를 분리해서 한 번에 보이는 양 줄이기
  - “5분 버전”과 “자세한 버전” 분리
  - 설치 성공 기준을 아주 작게 만들기: `claude` 실행 후 첫 질문
  - Cursor/VS Code는 처음 설치 편에서 뒤로 빼도 됨

### 3편 첫 프로젝트
- 좋은 핵심 편이다. 콘텐츠로도 가장 많이 재활용 가능.
- 수정:
  - 예제 3개 중 하나만 고르게 하기
  - “회의록 요약기”를 대표 경로로 삼고 나머지는 선택지 처리
  - 성공 화면/결과 파일 예시를 더 크게 보여주기

### 4편 Claude 잘 쓰는 법
- 매우 중요하지만 제목이 조금 넓다.
- 수정:
  - “프롬프트 잘 쓰기”보다 “AI에게 일을 맡기는 법”으로 재프레이밍
  - CLAUDE.md는 뒤쪽으로 빼고, 먼저 목표/자료/조건/검수 기준 4요소 강조
  - 카드뉴스 핵심 콘텐츠로 전환하기 좋음

### 5편 워크스페이스
- 비개발자에게 꼭 필요한 편.
- 수정:
  - “폴더가 곧 작업방”이라는 비유를 더 강하게
  - 비용 관리는 별도 보너스 섹션으로 분리
  - 업무별 폴더 예시: 콘텐츠, 회의록, 세금, 고객관리, 여행계획

### 6편 MCP
- 난이도 높음. “공식 플러그인”보다 “앱 연결”로 낮춰야 함.
- 수정:
  - API/CLI/MCP 구분은 좋은데, 너무 이론처럼 보이면 이탈 가능
  - 처음엔 Filesystem/Notion/Google Drive 중 하나만 따라 하게 하기
  - “MCP는 지금 몰라도 됩니다” 안전문구 필요

### 7편 Skills
- 매우 좋지만, 비개발자가 파일을 직접 만들기 시작하면 부담이 큼.
- 수정:
  - “Claude에게 스킬 만들어달라고 시키는 방식”을 먼저 제시
  - 수동 파일 생성은 고급 옵션으로 이동
  - 첫 예시는 회의록/콘텐츠 캘린더처럼 바로 쓰는 업무형 추천

### 8편 Sub-agents
- 컨셉은 매력적이지만 초보에겐 고급.
- 수정:
  - “처음 2주 동안은 안 봐도 됩니다” 문구 추가 가능
  - 사용 사례를 리서치/검수/요약처럼 비개발자 업무로 낮추기

### 9편 Hooks
- 안전장치라는 메시지는 좋지만 JSON/스크립트가 부담.
- 수정:
  - “위험한 삭제 막기” 하나만 먼저
  - 복붙 전용으로 만들고 설명은 뒤로
  - 설정 실패 시 복구법을 더 앞에 둔다

## 가이드 메인 페이지 수정 제안

현재 9편이 한 번에 보이기 때문에 “다 해야 하나?” 느낌이 날 수 있다.

추천 섹션 분리:

### 1단계: 오늘 시작
1. Claude Desktop
2. Claude Code 설치
3. 첫 프로젝트

### 2단계: 매일 쓰기
4. AI에게 일을 맡기는 법
5. 워크스페이스 만들기

### 3단계: 자동화 확장
6. 앱 연결
7. 반복 작업 버튼화

### 4단계: 고급/안전
8. AI 팀 만들기
9. 안전장치

메인 문구 추천:

```text
9편을 다 볼 필요 없습니다.
처음이라면 1~3편만 끝내도 충분합니다.
그다음 내 업무에 필요할 때 하나씩 열어보세요.
```

## 콘텐츠 연계 방식

가이드를 바로 홍보하지 말고, 인스타 콘텐츠에서 문제를 먼저 만든다.

예:

- 카드뉴스: `Claude Code 시작 전, 이 3개만 준비하세요`
  - 연결: 2편/3편
- 카드뉴스: `AI가 못하는 게 아니라 시키는 방식이 문제입니다`
  - 연결: 4편/5편
- 카드뉴스: `비개발자가 처음 만들기 좋은 도구 5개`
  - 연결: 3편
- 카드뉴스: `AI가 파일을 날릴까봐 무섭다면`
  - 연결: 9편 Hooks

## 최우선 수정 3개

1. 메인 페이지에 “처음이면 1~3편만” 문구 추가
2. 2편 설치 가이드를 5분 버전/자세한 버전으로 분리
3. 각 편 상단에 “오늘의 목표 / 성공 증거 / 안 해도 되는 것” 추가
