# 카드뉴스 디자인 재학습 루프

## 목표
비개발dev 인스타 카드뉴스를 만들 때 폰트, 배경 이미지 후킹, 레이아웃을 즉흥 감각이 아니라 레퍼런스 기반 규칙으로 설계한다.

## 문제 인식
현재 결과물은 방향성은 맞지만 다음 약점이 있다.

1. 한글 폰트 선택과 자간/행간/굵기 감각이 약하다.
2. 배경 이미지가 예쁘지만 후킹 장면으로 작동하는지 판단 기준이 부족하다.
3. 레이아웃이 정보 박스/카드 UI로 쉽게 흘러 프리미엄 매거진 감도가 떨어진다.
4. 레퍼런스 계정의 표면 스타일만 흉내 내고, 왜 그 구도가 클릭을 만드는지 분석이 부족하다.

## 학습 방식

### 1. 레퍼런스 해부
대상:
- @adrianabubori: 프리미엄 매거진/브랜드 광고형
- @halfdoneclub.ai: 다크 테크/실무 썸네일형
- Apple / Airbnb / Notion / Claude 디자인 시스템: 여백, 타이포, 사진 사용법 참고

분석 항목:
- 첫 1초에 보이는 후킹 요소
- 배경 이미지의 역할: 인물, 손, 제품, 공간, 대비, 시선 방향
- 제목 폰트 종류: serif/sans, 굵기, 자간, 행간
- 제목 위치와 크기
- 보조 문구의 양과 배치
- CTA가 버튼인지 캡션인지
- 여백 비율
- 사진과 글자의 대비 방식

### 2. 폰트 실험
한글 기준 최소 5개 조합을 실험한다.

- 프리미엄형: Pretendard/Noto Sans KR + serif 느낌 영문 보조
- 매거진형: Apple SD Gothic Neo 굵기 제한 + Playfair Display 영문
- 정보형: SUIT/Pretendard 계열 산세리프
- 감성형: Noto Serif KR 일부 제목 테스트
- 다크 테크형: Pretendard ExtraBold + JetBrains Mono 라벨

검수 기준:
- 모바일 1초 가독성
- 촌스러움 여부
- 배경과 충돌 여부
- 한글 줄바꿈 자연스러움
- 브랜드 톤 적합성

### 3. 배경 이미지 후킹 기준 만들기
Adriana형 배경 이미지 조건:
- 예쁜 공간만으로는 부족하다.
- 결과를 상상하게 하는 장면이어야 한다.
- 예: 노트북+콘텐츠 보드, 촬영 세팅, 브랜드 패키지, 편집 데스크, 고급 사무 공간, 크리에이터 루틴.
- 피해야 할 것: 의미 없는 감성 소품, 가짜 텍스트가 많은 종이, 너무 평면적인 책상, 시선이 분산되는 오브제 과다.

배경 이미지 평가표:
- 후킹성 0~10
- 텍스트 안전영역 0~10
- 고급감 0~10
- AI 티 0~10
- 주제 적합성 0~10

### 4. 레이아웃 문법 정리
Adriana형 카드 기본 레이아웃 후보:

A. 매거진 커버형
- 상단 작은 라벨
- 대형 제목 2~3줄
- 하단 작은 캡션 CTA
- 박스 최소화

B. 광고 포스터형
- 사진 주인공 60~70%
- 제목이 사진을 일부 덮음
- CTA는 작은 캡슐

C. 에디토리얼 인용형
- 여백 50%
- 한 문장 중심
- 하단 얇은 라인/캡션

D. 워크플로우형
- 박스보다 얇은 라인, 번호, 컬럼 구조
- 정보량은 줄이고 rhythm으로 읽게 함

### 5. 실험 산출물
한 번에 완성본을 만들지 않는다.

1. 같은 카피로 커버 3시안 제작
   - 폰트만 다른 버전
   - 이미지 후킹만 다른 버전
   - 레이아웃만 다른 버전
2. contact sheet 생성
3. vision 검수
4. 사용자에게 3안 중 선택 요청
5. 선택된 안의 디자인 토큰으로 6장 확장
6. 확장 후 다시 vision 검수

## 다음 실행 계획
현재 카드뉴스를 바로 수정하기보다, 먼저 1장 커버만 다시 만든다.

주제: 혼자 일하지만 팀처럼 보이는 법

3개 방향:
1. Adriana 프리미엄 매거진형
2. Adriana 브랜드 광고형
3. Halfdoneclub 다크 테크형 비교안

각 방향마다:
- 배경 이미지 기준
- 폰트 조합
- 레이아웃 그리드
- 제목 줄바꿈
- CTA 위치
을 명시하고 1080×1350 PNG로 뽑는다.

## 성공 기준
- 사용자가 “이제 좀 배운 것 같다”고 느끼는 수준
- 예쁜 배경이 아니라 클릭하고 싶은 커버
- 한글 제목이 촌스럽지 않고 모바일에서 바로 읽힘
- 박스가 아니라 타이포/사진/여백으로 위계를 만듦
- 카드 6장 확장 전에 커버 방향이 먼저 검증됨
