## 지금 문제: v8 05번 실패 진단

v8 05번은 `워크플로우`를 말하려 했지만 구조가 약하다.

### 실패 이유

1. 강조 대상이 불명확하다.
   - 도구를 보여주려는 건지, 작업 순서를 보여주려는 건지 애매하다.
   - 중앙은 `Claude / Notion / Canva / Instagram` 도구명 중심.
   - 하단은 `초안 → 캘린더 → 디자인 → 업로드` 단계 중심.
   - 두 레이어가 같은 말을 반복하지만 기준이 다르다.

2. 시선 이동이 약하다.
   - 2×2 그리드가 흐름도처럼 보이지만, 사용자가 어느 방향으로 읽어야 할지 즉시 알기 어렵다.
   - 연결선이 관계를 강하게 만들지 못한다.

3. 템플릿화 기준이 없다.
   - 카드, 하단 요약 박스, CTA가 모두 존재하지만 역할 위계가 약하다.
   - 다른 주제로 바꿀 때 무엇을 유지하고 무엇을 바꿔야 할지 명확하지 않다.

### 결론

05번은 `도구 네트워크`가 아니라 `프로세스 전환` 페이지로 다시 정의해야 한다.

## 레이아웃 템플릿 원칙

각 페이지는 제작 전에 아래를 먼저 확정한다.

1. 이 장의 역할
2. 독자가 기억해야 할 한 문장
3. 시선 이동 방향
4. 메인 강조 요소 1개
5. 보조 요소 최대 2개
6. 삭제할 장식 요소

보조 요소는 메인 메시지를 돕지 않으면 삭제한다.

---

## 6장 기본 스토리 템플릿

### 01 Cover / 결과 후킹

- 역할: 이 콘텐츠를 저장하거나 넘겨보게 만드는 첫 장.
- 메인 메시지: `무엇을 얻는가`를 즉시 보여준다.
- 시선 이동: 인물/오브젝트 → 큰 제목 → CTA.
- 메인 요소: 강한 캠페인 이미지 + 큰 한글 제목.
- 보조 요소:
  - 작은 프롬프트 칩
  - CTA pill
- 금지:
  - 설명형 문장 길게 쓰기
  - 얼굴을 제목으로 가리기
  - 읽히지 않는 작은 프롬프트 박스

### 02 Campaign Bridge / 아이디어가 캠페인이 되는 장면

- 역할: 커버의 약속을 실제 콘텐츠 단위로 연결.
- 메인 메시지: `아이디어 하나가 여러 콘텐츠 재료로 쪼개진다.`
- 시선 이동: 큰 제목 → 3개 output 카드.
- 메인 요소: 무드보드/에셋 플랫레이/작업물 이미지.
- 보조 요소:
  - Hook / Asset / Caption 3카드
  - 작은 방향 CTA
- 금지:
  - 같은 커버 이미지 반복
  - 카드 3개의 역할이 서로 겹치는 것

### 03 Prompt Setup / 입력 구조

- 역할: Claude에게 무엇을 어떻게 입력하는지 보여준다.
- 메인 메시지: `프롬프트보다 방향 설정이 먼저다.`
- 시선 이동: 제목 → Claude 입력창 → send/copy CTA.
- 메인 요소: 큰 Claude/browser 입력창.
- 보조 요소:
  - HOOKS / VISUALS / CALENDAR 같은 output chip
  - Copy setup CTA
- 금지:
  - 코드처럼 보이지만 읽히지 않는 작은 텍스트
  - 입력창 주변 장식 과다

### 04 Generated Assets / 산출물 카드

- 역할: Claude가 실제로 뽑아주는 결과물을 구조화.
- 메인 메시지: `바로 쓸 수 있는 조각들이 나온다.`
- 시선 이동: 제목 → 4개 산출물 카드.
- 메인 요소: 2×2 output card grid.
- 보조 요소:
  - 아주 약한 배경 워터마크
- 금지:
  - 4개 카드의 위계가 제각각인 것
  - 보조 설명문이 안 보이는 크기

### 05 Process Conversion / 초안이 발행물로 바뀌는 구조

- 역할: 산출물이 실제 발행 프로세스로 전환되는 장면.
- 메인 메시지: `아이디어가 4단계를 거쳐 업로드 가능한 콘텐츠가 된다.`
- 시선 이동: 좌→우 단일 라인 권장.
- 메인 요소: 번호형 4단계 프로세스.
- 권장 구조:
  - 01 Draft
  - 02 Calendar
  - 03 Design
  - 04 Publish
- 보조 요소:
  - 각 단계 아래 도구명 작게 표기: Claude / Notion / Canva / Instagram
  - 하단에는 한 줄 요약만 사용
- 금지:
  - 2×2 도구 네트워크처럼 보여 흐름이 모호한 구조
  - 중앙 카드와 하단 요약의 용어 불일치
  - 연결선이 관계를 명확히 만들지 못하는 경우

### 06 Final CTA / 행동 유도

- 역할: 댓글, DM, 팔로우 행동을 만든다.
- 메인 메시지: `어떤 키워드를 댓글에 남기면 무엇을 받는지`.
- 시선 이동: Free label → 댓글 키워드 → DM 버튼 → 제공항목 → 팔로우.
- 메인 요소: 댓글 키워드 CTA.
- 보조 요소:
  - 제공 자료 3개
  - 하단 팔로우 바
- 금지:
  - 의미 없는 밝은 박스
  - 자료 미리보기인지 장식인지 모호한 박스
  - 작은 라벨이 모바일에서 읽히지 않는 크기

## 폰트 가독성 규칙

1080×1350 인스타 카드 기준:

- 메인 제목: 80px 이상 가능.
- 중간 설명/서브카피: 30~40px 권장.
- 카드 보조문: 24px 이상 권장.
- 읽혀야 하는 라벨/번호/프롬프트 텍스트: 22px 근처를 실질 최소값으로 본다.
- 18~21px는 비필수 라벨에만 제한적으로 허용.
- contact sheet에서 `살짝 작다`는 말이 나오면 발행 전 패치한다.

### 05 재제작안

- 제목: `초안이 발행물로 바뀌는 4단계`
- 구조: 좌→우 단일 라인 또는 상→하 명확한 번호형 프로세스
- 단계:
  1. Draft / Claude / 초안 만들기
  2. Calendar / Notion / 날짜에 배치
  3. Design / Canva 또는 Figma / 카드로 변환
  4. Publish / Instagram / 업로드
- 하단 요약: `초안 하나를 발행 가능한 콘텐츠 흐름으로 바꾼다.`
- CTA: 작게 `SEE THE FLOW →`

이렇게 해야 05번의 역할이 `툴 나열`이 아니라 `프로세스 전환`으로 보인다.
