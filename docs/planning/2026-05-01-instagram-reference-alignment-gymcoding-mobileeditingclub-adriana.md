# 인스타 파이프라인 레퍼런스 재정렬

작성일: 2026-05-01

## 사용자 정정

사용자는 인스타 콘텐츠 파이프라인을 다음처럼 구분하기로 했다.

1. **설명형**
   - 참고 계정: `@gymcoding`
   - 방향: AI/Claude/개발/업무 도구를 실무 문제 중심으로 풀어주는 설명형 카드뉴스
   - 현재 정리 완료한 Warm Minimal / Claude Editorial 디자인은 유지하되, **토픽과 내용 기획은 @gymcoding에 더 가깝게 잡는다.**

2. **트렌디형**
   - 참고 계정: `@mobileeditingclub`, `@adrianabubori`
   - 방향: AI/Claude/콘텐츠 자동화/브랜드 워크플로우를 고급 인물·제품 이미지, 큰 타이포, 짧은 카피로 포장하는 매거진/캠페인형
   - 이전에 참고했던 `@photo_tvoritel`은 포토그래퍼/영상 중심이라 사용자 방향과 다르므로 메인 레퍼런스에서 제외한다.

## 1. 설명형 레퍼런스: @gymcoding

### 계정 포지션

- `짐코딩 | AI 네이티브 개발`
- Claude Code, AI 개발, MCP, 개발 워크플로우, AI 시대 개발자 생존 전략 중심
- 코딩 교육/강의 전환 구조가 명확함

### 주요 토픽

- Claude / Claude Code 활용법
- AI 개발 워크플로우
- MCP, Subagent, Memory, Context 같은 실무 개념
- AI 뉴스/공식 발표 해설
- 개발자의 불안/오해/실수 교정
- 비개발자/입문자의 진입 장벽 해소

### 후킹 패턴

1. **위기감 조성형**
   - `개발자가 코드 안 짜는 시대`
   - `AI가 해킹을 자동화하기 시작했다`

2. **오해 교정형**
   - `AI 바꾸지 마세요 환경 바꾸세요`
   - `Claude Code 혼자 일 시키지 마세요`

3. **질문형**
   - `코딩 1도 모르는데 진짜 되나요?`
   - `클로드 발목 잡는 건 혹시 내 코드?`

4. **숫자/시간 압축형**
   - `MCP 5분 정리`
   - `3가지 이유`

5. **권위 차용형**
   - `Claude 공식 발표`
   - `Anthropic News`

6. **말풍선/밈형**
   - 인물 이미지 위 상황극 말풍선으로 기술 주제를 덜 딱딱하게 만듦

### 설명형에 적용할 기획 기준

우리 설명형은 디자인은 Warm Minimal / Pretendard 기반으로 가되, 토픽과 제목은 다음 방향을 따른다.

- 개념 설명보다 독자의 문제/오해/불안을 먼저 잡는다.
- `하지 마세요`, `문제는 OO입니다`, `5분 정리`, `OO 때문에 실패합니다` 같은 제목 구조를 쓴다.
- 공식 발표/출처/뉴스 해설 라벨을 활용해 신뢰도를 만든다.
- Claude/AI 도구를 단순 기능이 아니라 실무 워크플로우와 연결한다.
- 내부 슬라이드는 문제 → 오해 → 핵심 개념 → 실무 적용 → 체크리스트/CTA 구조로 간다.

### 우리 설명형 토픽 예시

- `Claude Code 혼자 일 시키지 마세요`
- `AI가 멍청한 게 아니라 컨텍스트가 문제입니다`
- `MCP를 모르면 에이전트를 제대로 못 씁니다`
- `프롬프트만 고치지 마세요. 작업 환경이 문제입니다`
- `Claude를 팀처럼 나눠 쓰는 5분 정리`
- `AI에게 맡기기 전에 역할부터 쪼개세요`

## 2. 트렌디형 레퍼런스: @mobileeditingclub

### 계정 포지션

- AI, editing tips, news & trends
- creative branded content
- Claude, Figma, GPT Image, Nano Banana 등 최신 툴/모델 활용
- AI 교육 콘텐츠를 패션/뷰티/브랜드 캠페인처럼 포장

### 주요 토픽

- AI 이미지 생성/제품 촬영
- AI 제품/브랜드 캠페인 비주얼
- Figma & Claude for Brands
- Claude Skills / Claude 워크플로우
- AI 모델 비교
- 콘텐츠 파이프라인 자동화
- 뷰티/패션/주얼리/라이프스타일 소재

### 비주얼 스타일

- 고급 패션 매거진/브랜드 캠페인 톤
- 인물 클로즈업, 제품 착용/사용 장면
- 자연광, 깨끗한 피부, 브랜드 화보 같은 이미지
- 큰 흰색 타이포그래피
- 굵은 산세리프 + 이탤릭 세리프 조합
- 작은 로고 반복
- 하단 알약형 CTA 버튼

### 카피 구조

- `This is AI`
- `Stop Generating AI Images`
- `Figma & Claude for Brands`
- `AI Product Shoots`
- `Claude Skills`
- `SEE THE WORKFLOW →`
- `LIKE THIS →`
- `OUR WORKFLOWS →`

### 트렌디형에 적용할 기준

- 첫 장은 정보 카드가 아니라 **광고 캠페인 컷**처럼 만든다.
- AI 결과물을 숨기지 않고 `이게 AI라고?` 반응을 유도한다.
- 사람 얼굴, 손, 제품, 작업 장면을 적극적으로 쓴다.
- 설명은 첫 장에 많이 넣지 않고 내부 슬라이드/캡션으로 넘긴다.
- 짧고 큰 카피 + 작은 CTA + 작은 로고로 구성한다.

## 3. 트렌디형 레퍼런스: @adrianabubori

### 계정 포지션

- 개인 브랜드 + `@mobileeditingclub` 빌딩
- AI/Claude/콘텐츠 자동화/비즈니스 성장 도구를 라이프스타일처럼 포장
- Claude를 단순 챗봇이 아니라 창업자/크리에이터의 업무 동료로 보여줌

### 주요 토픽

- Claude cowork
- Claude Skills
- AI agent
- 콘텐츠 파이프라인 자동화
- 브랜드가 더 많은 콘텐츠를 만드는 방식
- 폰트/디자인 큐레이션
- 업무 시간을 줄이는 Claude 활용법

### 인물 이미지 사용 방식

- 본인 얼굴/창업자 이미지로 신뢰와 개인 브랜드 강화
- 패션/광고 모델 같은 AI 휴먼 이미지로 감도 형성
- 초현실적 AI 이미지로 AI의 가능성 표현
- 인물은 장식이 아니라 `이런 사람이 이 도구를 쓴다`는 라이프스타일 신호

### 카피 구조

- `I save 4 hours a day with Claude`
- `Claude cowork`
- `Prompt less. Edit more.`
- `Try this AI agent`
- `This is AI`
- `I hired a full creative team with Claude`

### 우리에게 적용할 기준

- 트렌디형은 설명형의 반대가 아니라, 설명할 내용을 더 매력적으로 포장하는 방식이다.
- 도구 사용법보다 결과/욕망을 먼저 보여준다.
- 인물 이미지는 우리 타깃의 상황과 정체성을 보여줄 때 사용한다.
- Claude 같은 도구명은 필요하면 쓰되, 계정 정체성을 특정 툴에 종속시키지 않는다.

이전에는 어두운 시네마틱 작업자 이미지 참고로 `@photo_tvoritel`을 봤지만, 사용자가 지적한 대로 이 계정은 포토그래퍼/영상 중심이다.

- 메인 트렌디형 레퍼런스로 쓰지 않는다.
- 필요할 때만 조명/무드/시네마틱 촬영 감각 참고용으로 제한한다.
- 우리 트렌디형의 핵심은 사진작가/영상 제작자 무드가 아니라, `@mobileeditingclub` + `@adrianabubori`처럼 **AI/브랜드/콘텐츠 자동화가 세련된 캠페인처럼 보이는 것**이다.

## 5. 최종 구분

### 설명형

참고: `@gymcoding`

- 토픽: AI/Claude/개발/업무 도구 실무 해설
- 목적: 이해, 저장, 신뢰, 교육 전환
- 카피: 오해 교정, 질문, 위기감, 5분 정리
- 구조: 문제 → 오해 → 개념 → 적용 → 체크리스트
- 디자인: Warm Minimal / Pretendard 유지 가능

### 트렌디형

참고: `@mobileeditingclub`, `@adrianabubori`

- 토픽: AI/Claude/콘텐츠 자동화/브랜드 워크플로우
- 목적: 욕망, 클릭, 브랜드 감도, `이게 AI라고?` 반응
- 카피: 짧은 결과형/선언형/툴 조합형
- 구조: 캠페인 컷 → 결과 → 워크플로우 → 프롬프트/팁 → CTA
- 디자인: 고급 인물/제품 이미지 + 대형 화이트 타이포 + 작은 로고/CTA

## 주의

- 트렌디형에서 과한 박스/도식/긴 설명을 넣으면 설명형으로 돌아간다.
- 설명형에서 너무 화보형 이미지에 의존하면 내용 신뢰가 흐려질 수 있다.
- 두 파이프라인은 섞지 말고, 포스트 목적에 따라 선택한다.
