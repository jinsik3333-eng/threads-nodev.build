# 트렌디형 인스타 카드뉴스 템플릿 v1

Date: 2026-05-01
Reference: @mobileeditingclub + @adrianabubori

## 핵심 교정

마지막 페이지의 밝은 박스는 "자료 미리보기" 의도로 넣었지만, 현재 형태는 역할이 애매하다.

- 리소스 카드인지
- 프롬프트 미리보기인지
- 이벤트 배너인지
- DM 안내 카드인지

한눈에 구분되지 않으면 삭제하거나 명확한 컴포넌트로 바꿔야 한다.

트렌디형 카드뉴스는 각 페이지마다 들어가는 요소의 역할을 고정해야 한다. 예쁜 요소를 추가하는 방식이 아니라, 각 요소가 아래 중 하나의 기능을 해야 한다.

- 후킹
- 맥락 전환
- 프롬프트 입력
- 결과물 미리보기
- 워크플로우 연결
- 행동 유도

---

## 전체 톤 규칙

### 기본 톤

1~5장은 같은 톤으로 유지한다.

- 밝은 크림/베이지/스카이블루 기반
- 프리미엄 캠페인 / 에디토리얼 / 브랜드 키트 무드
- SUIT 기반 굵은 한글 타이틀
- 일부 영문 단어만 세리프 이탤릭 포인트
- 둥근 흰색 카드, 얇은 그림자, 넓은 여백
- 라임 포인트는 핵심 단어 또는 CTA에만 사용

### 예외 톤

6장만 어둡게 가능하다.

사용 조건:

- 팔로우 유도
- 댓글 키워드 유도
- DM 자료 제공
- 이벤트 안내
- 계정 소개

어두운 톤은 마무리/행동 유도용이지, 중간 설명 페이지용이 아니다.

---

## 공통 고정 요소

### 계정명

- 모든 페이지 상단 중앙 고정
- 텍스트: `NODEV BUILDER`
- 위치: top 48~58px, center align
- 좌측 정렬 금지
- 페이지마다 위치 변경 금지

### 페이지 번호

- 우상단 고정
- 예: `02 / 06`
- 계정명보다 시각 위계 낮게

### CTA 버튼

- 각 장마다 넣어도 되지만 역할이 명확해야 한다.
- 장식용 CTA 금지.
- 예:
  - `COPY THE PROMPT →`
  - `FROM IDEA TO CAMPAIGN →`
  - `COPY THE SETUP →`
  - `SEE THE WORKFLOW →`
  - `COMMENT “30일치” →`

## 6장 구조 템플릿

## 01. Cover / 후킹

### 역할

스크롤을 멈추게 하고, 결과를 한 문장으로 약속한다.

### 들어갈 내용

- 결과 중심 카피
- Claude/AI 도구명
- 짧은 CTA
- 프롬프트/자료 제공 암시

### 들어갈 요소

- 고급 인물/제품/캠페인 이미지 1장
- 대형 한글 타이포
- 핵심 단어 색상 강조
- 작은 prompt chip 가능
- 하단 CTA pill

### 현재 통과 예시

```text
콘텐츠 30일치
Claude가 짜줌
```

### 금지

- 설명형 제목
- 긴 부제
- 의미 없는 라벨
- 얼굴 가리는 텍스트

---

## 02. Campaign Bridge / 아이디어가 캠페인이 되는 장면

### 역할

커버의 약속을 "브랜드 캠페인/콘텐츠 키트" 맥락으로 전환한다.

### 들어갈 내용

- 한 아이디어가 여러 콘텐츠 결과물로 바뀐다는 메시지
- Hook / Asset / Caption 같은 결과 카테고리

### 들어갈 요소

- 캠페인 무드보드 이미지 또는 제품/콘텐츠 에셋 플랫레이
- 큰 타이틀
- 3개 이하의 결과 카드
- 작은 CTA

### 카피 예시

```text
한 아이디어가
campaign이 되는 과정
```

### 카드 예시

- Hook
- Asset
- Caption

### 금지

- 같은 인물 이미지 재사용
- 설명문 박스 과다
- 하단 카드 4개 이상
- 너무 작은 보조 텍스트

---

## 03. Prompt Setup / 프롬프트 입력

### 역할

사용자가 따라 할 수 있는 첫 입력 구조를 보여준다.

### 들어갈 내용

- 캠페인 방향 설정 프롬프트
- 입력값 placeholder
- output 항목

### 들어갈 요소

- 밝은 톤의 Claude/browser input window
- mono/UI 폰트
- 2~3개의 작은 output chip
- 하단 또는 우하단 CTA

### 카피 예시

```text
감 말고
direction부터
```

### 프롬프트 예시

```text
/campaign-builder
Idea: {주제}
Make it feel like: premium brand campaign
Output: hooks · visuals · captions · calendar
```

### 금지

- 다크 UI로 갑자기 전환
- 코드/프롬프트 내용 과밀
- 프롬프트 창이 장식처럼만 보이는 것

---

## 04. Generated Assets / 결과물 카드

### 역할

Claude가 실제로 어떤 재료를 만들어주는지 보여준다.

### 들어갈 내용

- hooks
- frames/slides
- captions
- calendar

### 들어갈 요소

- 밝은 배경
- 4개 output card
- 카드 제목 크게
- 보조 설명은 짧게
- 배경 워터마크는 선택

### 카피 예시

```text
Claude가 뽑은
콘텐츠 재료들
```

### 서브카피 예시

```text
설명 말고, 바로 쓸 조각.
```

### 카드 예시

- `10 angles`
- `6 frames`
- `3 tones`
- `30 days`

### 금지

- 설명형 표처럼 보이는 과한 정리
- 카드마다 긴 설명
- 작은 텍스트 다수

---

## 05. Workflow Map / 툴 연결

### 역할

결과물이 실제 발행 흐름으로 이어지는 것을 보여준다.

### 들어갈 내용

- Claude → Notion → Canva/Figma → Instagram
- 초안 → 캘린더 → 디자인 → 업로드

### 들어갈 요소

- 밝은 톤 유지
- 2x2 툴 카드 또는 가로 플로우
- 연결선은 굵고 명확하게
- 하단 campaign flow 카드
- CTA

### 카피 예시

```text
초안이
workflow로 퍼지는 법
```

### 금지

- 다크 SaaS UI 톤으로 전환
- 얼굴/인물 이미지 재사용
- 연결선이 너무 얇아서 안 보이는 것
- 툴 카드가 모바일에서 너무 작은 것

---

## 06. Final CTA / 마무리 행동 유도

### 역할

카드뉴스를 저장/댓글/팔로우/DM 행동으로 닫는다.

### 톤

- 유일하게 다크 톤 가능
- 하지만 1번의 라임 포인트와 연결해야 함
- 계정명은 여전히 상단 중앙

### 선택 가능한 마무리 구조

#### A. 댓글 키워드 + DM 자료 제공형

가장 추천.

```text
FREE TEMPLATE
댓글에 30일치 남기기
콘텐츠 캘린더 프롬프트 템플릿을 DM으로 보내드릴게요.
```

요소:

- 큰 댓글 키워드 CTA
- DM 자료 설명
- 자료 미리보기는 작고 명확하게
- 하단 팔로우 바

#### B. 팔로우 유도형

```text
AI 워크플로우를
계속 보고 싶다면
@nodev.builder 팔로우
```

요소:

- 계정명 크게
- 어떤 콘텐츠를 계속 주는지 명시
- 하단 CTA pill

#### C. 저장 유도형

```text
이 프롬프트
저장해두기
```

요소:

- 복사용 prompt card
- 하단 save CTA
- DM/댓글 유도 없음

### 자료 미리보기 박스 규칙

밝은 박스를 넣을 경우 반드시 역할이 명확해야 한다.

허용:

- DM으로 보낼 자료의 mockup
- prompt kit preview
- 템플릿 표지
- checklist preview

금지:

- 의미 없는 흰색 박스
- 읽기 어려운 긴 prompt 전문
- 리소스인지 장식인지 모호한 박스

### 현재 v7 문제

마지막 페이지의 밝은 박스는 `30-day content prompt kit` 미리보기 의도였지만, 역할이 즉시 명확하지 않다.

수정 방향:

- 박스를 없애고 CTA만 강하게 가거나
- 박스 안에 `DM RESOURCE PREVIEW` / `Prompt Kit` / `Calendar Template`처럼 역할을 명확히 넣는다.

## 이미지 사용 규칙

- 같은 이미지는 한 카드뉴스 안에서 1회만 사용한다.
- 같은 인물 이미지를 반복하면 템플릿처럼 보인다.
- 1번 커버 이미지 재사용 금지.
- 2번은 무드보드/에셋 이미지 권장.
- 3~5번은 UI/카드/워크플로우 중심으로 가도 된다.
- 6번은 이미지보다 CTA 구조가 우선이다.

## QA 체크리스트

### 톤 일관성

- [ ] 1~5장이 같은 밝은 캠페인 톤인가?
- [ ] 6장만 어둡게 전환되는 이유가 명확한가?
- [ ] 중간에 갑자기 설명형/SaaS 튜토리얼 톤으로 무너지지 않는가?

### 요소 역할

- [ ] 모든 박스가 역할을 갖는가?
- [ ] prompt window는 실제 입력 구조인가?
- [ ] output cards는 실제 산출물을 보여주는가?
- [ ] workflow map은 발행 흐름을 보여주는가?
- [ ] 마지막 CTA는 사용자가 무엇을 해야 하는지 즉시 보이는가?

### 모바일

- [ ] 메인 타이틀은 축소해도 읽히는가?
- [ ] 보조 텍스트는 너무 작지 않은가?
- [ ] 카드/박스/CTA가 1px도 겹치지 않는가?
- [ ] 계정명은 모든 페이지 상단 중앙인가?
- [ ] 하단 CTA가 안전영역 안에 있는가?

## 최종 원칙

트렌디형 카드뉴스는 "예쁜 설명 카드"가 아니라 "브랜드 캠페인처럼 포장된 실행 자료"여야 한다.

각 페이지는 다음 문장을 만족해야 한다.

1. Cover: 이걸 보면 갖고 싶다.
2. Campaign Bridge: 이게 어떤 결과로 바뀌는지 보인다.
3. Prompt Setup: 어떻게 시작하는지 보인다.
4. Generated Assets: 무엇이 나오는지 보인다.
5. Workflow Map: 어디로 이어지는지 보인다.
6. Final CTA: 지금 무엇을 해야 하는지 보인다.
