# @nodev.builder 첫 게시글 v2 — 채널 정체성 선언

## 방향

첫 게시글은 가이드 홍보가 아니라 채널 선언이다.

핵심 메시지:
> 비개발자가 AI/Claude Code로 자기 일을 직접 도구화하는 과정을 보여주는 채널

전체 가이드는 공개하되, 처음 보는 사람에게는 `입문 3단계`만 먼저 안내한다.
`1~3편`이라는 내부 번호는 카드에서는 쓰지 않는다.

---

## 6장 구조

### 01 — Hook / 정체성

큰 제목:
```text
코딩 몰라도
내 일은
직접 만들 수 있습니다
```

보조:
```text
비개발자를 위한 AI 작업 시스템
```

하단:
```text
@nodev.builder
```

### 02 — 공감 / 기존 문제

큰 제목:
```text
아이디어는 있는데
구현에서
멈춘 적 있나요
```

보조:
```text
회의록 정리기, 파일 정리 자동화,
콘텐츠 캘린더 같은 것들.
```

하단:
```text
문제는 아이디어가 아니라 실행 순서였습니다
```

### 03 — 프레임 전환

큰 제목:
```text
개발자가 되는 게
목표는
아닙니다
```

보조:
```text
목표, 자료, 조건, 확인 기준을 정리해
AI에게 일을 맡기는 법을 익히는 것.
```

하단:
```text
코딩 강의보다 작업 지시서에 가깝습니다
```

### 04 — 이 채널이 보여줄 것

큰 제목:
```text
이 계정에서는
만드는 과정을
그대로 보여드립니다
```

보조:
```text
처음 막힌 화면, 잘못 나온 결과,
수정하는 프롬프트까지 기록합니다.
```

하단 리스트:
```text
Claude Code 시작하기
첫 프로젝트 만들기
업무별 워크스페이스
반복 업무 자동화
```

### 05 — 공개 가이드 + 입문 경로

큰 제목:
```text
전체 가이드는
공개해두었습니다
```

보조:
```text
다만 처음이라면 전부 보지 말고,
입문 3단계만 먼저 따라오세요.
```

하단:
```text
화면 감각 → 설치 → 첫 프로젝트
```

### 06 — CTA

큰 제목:
```text
처음 시작하는
순서가 필요하면
“시작”
```

보조:
```text
댓글에 남겨주세요.
비개발자용 입문 3단계를 DM으로 보내드릴게요.
```

하단:
```text
전체 목차는 프로필 링크에도 열어두었습니다
```

## 캡션

```text
코딩을 몰라도 이제는 직접 만들 수 있는 것들이 많아졌습니다.

회의록 정리기,
파일 정리 자동화,
콘텐츠 캘린더,
반복 업무 체크리스트 같은 것들.

중요한 건 개발자가 되는 게 아니라,
AI에게 내 일을 맡길 수 있게 설명하는 순서를 익히는 것입니다.

이 계정에서는 비개발자가 Claude Code와 AI 도구로
자기 일을 직접 도구화하는 과정을 하나씩 정리합니다.

전체 가이드는 무료로 공개해두었고,
처음 시작하는 분들을 위해 입문 3단계부터 볼 순서를 안내드릴게요.

필요하시면 댓글에 `시작`이라고 남겨주세요.

#비개발자 #ClaudeCode #AI활용 #업무자동화 #생성형AI #직장인AI
```
