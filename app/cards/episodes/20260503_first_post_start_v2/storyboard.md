# 2026-05-03 첫 게시글 v2 — 논리 구조 재설계

## 왜 v1이 약했는가

v1 문제:
- 첫 장의 약속이 `처음 켤 때`인데, 뒤에서는 갑자기 `1~3편`을 보라고 한다.
- `1~3편`이 무엇인지 처음 보는 사람에게 선행 설명이 없다.
- 채널 정체성이 아니라 가이드 홍보처럼 시작한다.
- 각 장의 역할이 `정보 나열`에 가깝고, 다음 장을 넘길 이유가 약하다.

따라서 첫 게시글은 `가이드 소개`가 아니라 **채널 선언 + 문제 재정의 + 앞으로 볼 이유**가 되어야 한다.

---

## 새 핵심 메시지

> 비개발자에게 필요한 건 코딩 강의가 아니라, AI에게 내 일을 맡기는 순서다.

이 계정은 Claude Code 기능 소개 계정이 아니라:

> 비개발자가 AI로 자기 일을 직접 도구화하는 과정을 보여주는 채널

---

## 새 6장 구조

구조: Before → Tension → Reframe → Method → Starting Point → CTA

### 1장 — 채널 정체성 / Hook

역할: 처음 보는 사람에게 “이 계정이 뭔지” 즉시 알림.

문구:
```text
코딩 몰라도,
내 일은
직접 만들 수 있습니다
```

보조:
```text
비개발자를 위한 AI 작업 시스템
```

의도:
- `Claude Code 처음 켤 때`보다 훨씬 넓고 채널 선언에 적합.
- 첫 게시글로서 정체성 명확.

---

### 2장 — 기존 세계 / 공감

역할: 타깃의 현재 상태를 말해준다.

문구:
```text
전에는 뭔가 만들려면
개발자에게 부탁해야 했습니다
```

보조:
```text
회의록 정리기, 견적서 자동화,
파일 정리, 콘텐츠 캘린더 같은 것들.
```

의도:
- 비개발자가 느끼는 의존감/막막함을 잡음.
- 채널이 다룰 결과물 범위를 자연스럽게 보여줌.

---

### 3장 — 진짜 문제 / 긴장

역할: “내가 못해서가 아니다”라는 안도와 재정의.

문구:
```text
문제는 아이디어가 아니라
만드는 순서를
몰랐던 것입니다
```

보조:
```text
무엇을 시킬지, 어떤 자료를 줄지,
끝났는지 어떻게 확인할지 몰랐던 것.
```

의도:
- 코딩 부족이 아니라 작업 설계 부족으로 프레임 전환.
- 다음 장의 해결 원리로 연결.

---

### 4장 — 새 프레임 / 핵심 관점

역할: 이 채널의 철학을 한 문장으로 제시.

문구:
```text
이제는 코딩보다
일을 설명하는 능력이
먼저입니다
```

보조:
```text
목표, 자료, 조건, 확인 기준을 정리하면
Claude가 만들 수 있는 일이 많아집니다.
```

의도:
- “코딩 강의 아님”을 명확히 함.
- 사용자 포지셔닝과 맞음.

---

### 5장 — 앞으로 다룰 것 / 보상

역할: 팔로우/저장할 이유 제공.

문구:
```text
이 계정에서
하나씩 보여드릴게요
```

보조 항목:
```text
Claude Code 시작하기
첫 프로젝트 만들기
업무별 워크스페이스
반복 업무 자동화
실패하지 않는 안전장치
```

의도:
- 1~3편을 바로 들이밀지 않고, 채널의 콘텐츠 약속을 먼저 줌.
- 첫 게시글로 자연스러움.

---

### 6장 — CTA / 댓글 키워드

역할: 작은 행동 하나만 요청.

문구:
```text
처음 시작하는 순서가 필요하면
댓글에 “시작”이라고 남겨주세요
```

보조:
```text
비개발자용 Claude Code 입문 3단계를
DM으로 보내드릴게요.
```

의도:
- 여기서 처음으로 `입문 3단계`가 등장하므로 맥락이 생김.
- `1~3편`이 아니라 `입문 3단계`라고 말해 처음 보는 사람도 이해 가능.
- DM에서는 그 3단계가 실제 가이드 1~3편이라고 설명하면 됨.

---
