### @adrianabubori
- 퍼스널 브랜딩+본인 워크플로우 공개
- AI를 팀원/동료로 은유
- 댓글 키워드 DM 리드 생성이 명확

### @sebia.ai
- 한국어 기술 카드뉴스, 실수/손해/망했습니다 등 강한 훅
- 공식 문서·수치·한계점으로 신뢰 보강
- 다크 브라운+형광 강조는 참고하되 과하게 복제 금지

### @gymcoding
- 개념 설명형, 한 장 정리, 변화/역할 전환 프레이밍

- nodev에서는 개발자 용어를 비개발자 화면 확인/행동으로 번역 필요

## @nodev.builder용 템플릿 후보

### 1. 숫자형 리소스 리스트
- 적용 예: 30 Claude 프롬프트 / 비개발자 앱 제작 체크리스트
- 기본 흐름: 숫자 훅 → 카테고리별 목록 → 저장 CTA
- 참조 post_id: mobileeditingclub-001

### 2. A vs B 비교
- 적용 예: Claude vs ChatGPT / Cursor vs Claude Code / 붙여넣기식 vs 프로젝트식
- 기본 흐름: 비교 기준 → 결과물/코드/흐름 대조 → 상황별 추천
- 참조 post_id: mobileeditingclub-002, gymcoding-002

### 3. N대 실수
- 적용 예: 비개발자 앱 만들기 5대 실수 / Claude Code 세팅 5대 실수
- 기본 흐름: 실수 선언 → 실수별 이유+교정 → CTA
- 참조 post_id: sebia-ai-001, mobileeditingclub-003

### 4. 손실 회피 팁
- 적용 예: Claude Code 이렇게 쓰면 토큰만 샙니다
- 기본 흐름: 손해 훅 → 수치 비교 → 대안 설정법 → CTA
- 참조 post_id: sebia-ai-002

### 5. AI 팀/역할 은유
- 적용 예: Claude로 혼자 쓰는 작은 제작팀
- 기본 흐름: 팀 은유 → 역할별 결과물 → 키워드 DM
- 참조 post_id: adrianabubori-003

### 6. 파이프라인/오토파일럿
- 적용 예: 아이디어 1개가 일주일 콘텐츠가 되는 흐름
- 기본 흐름: What if/10x 훅 → 단계별 자동화 → 실제 화면 증거 → CTA
- 참조 post_id: adrianabubori-002, adrianabubori-005

### 7. 한 장 정리/패러다임 전환
- 적용 예: 채팅창에서 끝나지 않는 앱 제작 흐름 한 장 정리
- 기본 흐름: 기존 방식 구식화 → 새 구조 다이어그램 → 기존 vs 신방식
- 참조 post_id: gymcoding-001

### 8. 실패 고백 스토리
- 적용 예: 비개발자로 앱 만들다 망한 지점
- 기본 흐름: 실패 → 원인 → 극복 → 결과/수치 → 배운 점
- 참조 post_id: sebia-ai-004

### 9. 기능 확장 뉴스
- 적용 예: Claude가 이제 앱 제작 어디까지 대신하나?
- 기본 흐름: 뉴스 훅 → 기능 설명 → 활용 시나리오 → 한계점
- 참조 post_id: gymcoding-003

### 10. 정체성/역할 변화
- 적용 예: 개발자가 코드 안 짜는 시대, 비개발자는 뭘 해야 하나
- 기본 흐름: 불안 훅 → 변화 배경 → 반론 → 새 스킬셋
- 참조 post_id: gymcoding-004

## 운영 적용 원칙
- 설명형 70% + 트렌디/캠페인형 30% 유지
- 모든 CTA는 댓글 키워드→ManyChat DM 또는 프로필 링크 중 하나로 명확히 연결
- 모바일에서 작은 글씨 금지: 코드/프롬프트 캡처는 읽을 수 있을 만큼 크게
- 세비아식 강한 훅은 가져오되 형광/다크를 과하게 복제하지 않기
- MEC/Adriana식 인간/캠페인 이미지는 커버나 증거 슬라이드에 제한적으로 사용하고, 내부 설명 슬라이드는 가독성 우선
