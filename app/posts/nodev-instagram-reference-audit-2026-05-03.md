# @nodev.builder 인스타그램 레퍼런스 감사 노트

작성일: 2026-05-03

## 왜 다시 하는가

이번 `콘텐츠 30일치 Claude가 짜줌` 카드뉴스는 레퍼런스 체계 없이 감각으로 이동하다가 실패했다.

실패 원인:
- image2 결과물을 그대로 믿고 진행함
- 가독성 보정 과정에서 얼굴/주요 피사체를 가림
- 모든 장이 이미지 주도라 메시지 집중이 깨짐
- 반대로 readable_v2는 정보 위계만 남고 인스타 감도가 사라짐
- `@nodev.builder`용 정형 레퍼런스 시스템이 없었음

따라서 다음 카드 제작 전에는 반드시 아래 레퍼런스 계정의 게시글 10개 이상을 기준으로 구조를 먼저 만든다.

---

## 분석 대상

1. `@mobileeditingclub`
2. `@adrianabubori`
3. `@sebia.ai`
4. `@gymcoding`

비로그인 Instagram 웹에서 보이는 최근/고정 포함 약 10~12개 썸네일 기준으로 1차 분석했다. 실제 개별 캐러셀 내부 슬라이드까지 보려면 사용자가 링크/스크린샷을 주거나 로그인 가능한 환경에서 추가 분석이 필요하다.

## 1. @mobileeditingclub

### 핵심 포지션

AI/브랜드 콘텐츠 제작을 패션·뷰티 매거진 커버처럼 보이게 만드는 계정.

### 최근 썸네일에서 보인 패턴

- 인물/제품 중심의 고퀄리티 비주얼
- 큰 흰색 타이포
- 툴명 + 결과물 제목
- `Claude`, `GPT Image 2`, `Midjourney`, `AI Product Shoots` 등 키워드 중심
- 상단 중앙 또는 상단부 브랜드명 고정
- 하단 작은 알약형 CTA
- 텍스트 박스보다 이미지 위 직접 타이포
- 이미지 80~90%, 텍스트 10~20%

### 커버 카피 공식

- `숫자 + 툴/자료`: `45 Claude Prompts`
- `A vs B`: `Midjourney VS GPT Image 2`
- `Stop + 행동`: `Stop Generating AI Images`
- `Try this + 도구`: `Try this AI agent`
- `How to use A and B`
- `This is AI`
- `Generate X with AI`

### @nodev.builder에 가져올 것

- 툴 설명보다 결과물 먼저 보여주기
- 제목은 짧고 강하게
- 하단 알약 CTA 고정
- 툴 로고는 작은 스티커처럼만 사용
- 그리드 전체 타이포 시스템 통일

### 그대로 가져오면 안 되는 것

- 패션/뷰티 모델컷 남발
- AI 이미지 계정처럼 보이는 비주얼 반복
- 실제 노코드/자동화 맥락 없는 감성 이미지
- 주제와 무관한 모델 사진

## 2. @adrianabubori

### 핵심 포지션

AI/Claude/콘텐츠 자동화를 크리에이티브 디렉터·브랜드 오너의 라이프스타일로 포장하는 계정.

### 최근 썸네일에서 보인 패턴

- 럭셔리 에디토리얼 + 생산성 욕망
- 인물/공간/오브젝트 중심
- Claude를 `cowork`, `creative team`, `autopilot`처럼 의인화
- 앱 아이콘은 스티커처럼 2~3개만 배치
- 실제 툴 UI보다 결과/상상 장면 중심
- 큰 산세리프 + 세리프/이탤릭/스크립트 조합
- 하단 작은 CTA: `my favorite workflows →`, `OUR WORKFLOWS →`

### 커버 카피 공식

- `I save [시간] with [툴/시스템]`
- `I hired a [팀/역할] with [툴]`
- `How [타깃] produce [배수] more [결과물] with [툴]`
- `What if [툴] could [결과] on autopilot?`
- `[기존 행동] less / [새 행동] more`
- `[리소스] of the week / [카테고리] Edition`

### @nodev.builder에 가져올 것

- “AI를 쓰는 법”보다 “AI로 내 일이 어떻게 바뀌는지” 표현
- `Code less / Ship more` 같은 반전형 카피
- `Claude를 팀원처럼 쓰는 구조` 같은 의인화
- 인물은 개발자가 아니라 창업자/오퍼레이터/크리에이터처럼 보이게
- 툴 아이콘은 보조 스티커로 제한

### 그대로 가져오면 안 되는 것

- 여성 패션/라이프스타일 무드 과다 복제
- AI 생성 인물이 너무 주인공이 되는 구조
- UI/워크플로우 실체 없이 분위기만 있는 커버
- 스크립트/세리프 과다 혼용

## 3. @sebia.ai

### 핵심 포지션

한국어 Claude/AI 툴 정보계정. 특정 키워드를 반복 점유하고, 손실회피형 카피로 클릭을 만든다.

### 최근 썸네일에서 보인 패턴

- Claude/Claude Code 키워드 집중 반복
- 손실회피/실수/금지/공식/무조건 계열 카피
- 브라운·오렌지·블랙 배경 + 청록 포인트
- `AI TOOLS | GENERAL` 같은 카테고리 라벨
- 인물/툴 화면/일러스트 배경 위 큰 제목
- 댓글 키워드/DM 유도는 캡션 중심, 썸네일에는 약하게 보임

### 커버 카피 공식

- `[툴] 세팅 5대 실수`
- `[툴] 이렇게 쓰면 손해`
- `[공식/제작자]가 공개한 사용법`
- `[툴] 나 대신 일한다`
- `[행동]하면 지는 겁니다`
- `[기능] 그냥 쓰면 99% 손해`

### @nodev.builder에 가져올 것

- 한동안 하나의 핵심 키워드 집중 점유
- 손실회피형 카피 테스트
- 카테고리 라벨 시스템
- 브랜드 포인트 컬러 고정
- 댓글 키워드/DM/템플릿 퍼널 연결

### 그대로 가져오면 안 되는 것

- 특정 툴 하나에 과도하게 종속
- 모든 게시글을 자극형 카피로만 구성
- 썸네일 텍스트 과밀
- 스톡/AI 배경 이미지 과의존

## 4. @gymcoding

### 핵심 포지션

Claude/AI 개발 뉴스를 개발자·비개발자에게 쉽게 번역하는 교육/미디어형 계정.

### 최근 썸네일에서 보인 패턴

- `Claude 공식 발표`, `Anthropic News` 등 신뢰 라벨
- 실제 인물/업계 인물 사진 + 말풍선
- 하단 검정 그라데이션 + 큰 제목
- 흰색 제목 + 주황색 핵심 단어 강조
- `한 장 정리`, `5분 정리`, `3가지 패턴` 같은 저장형 후킹
- 기술 키워드를 생활형 질문으로 번역

### 커버 카피 공식

- `[기술 키워드] 한 장 정리`
- `지금 쓰는 [툴], 옛날 방식일 수도?`
- `[행동] 혼자 시키지 마세요`
- `[툴] 발목 잡는 게 혹시 내 [문제]?`
- `[키워드] 5분 정리`
- `[기능] 100% 뽑아내는 3가지 패턴`

### @nodev.builder에 가져올 것

- 기술을 사용자 고민으로 번역
- 한 썸네일에 메시지 하나만
- 포인트 컬러는 핵심 단어에만
- 뉴스보다 “그래서 비개발자에게 무슨 의미인가” 해석
- 저장형 표현 사용

### 그대로 가져오면 안 되는 것

- 유명 인물 사진에 과도하게 의존
- 모든 게시글을 뉴스형으로 구성
- `AI가 다 해준다` 식 과장
- 밈/말풍선이 계정 신뢰감을 해치는 수준으로 과해지는 것

## @nodev.builder용 1차 디자인 원칙

### 포지셔닝

`개발 튜토리얼 계정`이 아니라 `비개발자가 AI와 노코드로 실제 업무/제품 시스템을 만드는 빌더 브랜드`.

### 시각 방향

- MEC/Adriana에서 가져올 것: 큰 제목, 매거진형 커버, 작은 CTA, 결과물 중심 이미지
- Sebia/Gymcoding에서 가져올 것: 한국어 후킹, 정보 위계, 저장형 카피, 기술 번역
- 버릴 것: 모델컷 남발, 정체불명 AI 이미지, SaaS 랜딩페이지식 박스 반복, 정보만 남은 밋밋한 템플릿

### 커버 구성 권장

```text
상단: @nodev.builder 또는 시리즈 라벨
중앙/하단: 초대형 핵심 카피 2줄
보조: 결과를 암시하는 한 줄
배경: 실제 앱/워크플로우/노트북/작업 장면/제품 결과물
스티커: Claude, Notion, Make 등 툴 아이콘 2~3개 이하
하단: 작은 알약 CTA
```

### 이미지 생성 전 필수 조건

1. 레퍼런스 기반 슬라이드 역할표 작성
2. 커버 3시안 먼저 제작
3. 인물 사용 여부를 슬라이드별로 제한
4. 이미지 생성 프롬프트에 반드시 `clear empty typography area` 포함
5. 얼굴/손/주요 오브젝트 위에 텍스트 박스 금지
6. 이미지가 설명을 대신하지 못하면 폐기
7. 6장 전체 렌더 전에 사용자에게 direction board 먼저 확인
