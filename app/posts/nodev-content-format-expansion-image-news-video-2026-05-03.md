# @nodev.builder 포맷 확장 전략 — 이미지·뉴스·영상 — 2026-05-03

## 배경

기존 계획은 인스타 카드뉴스와 HTML 리드 마그넷 중심이었다. 여기에 다음 세 가지 포맷을 추가한다.

1. ChatGPT image2 기반 트렌디 이미지 제작
2. 최근 인기 AI/Claude/업무 자동화 이슈를 뉴스처럼 큐레이션
3. 일부 콘텐츠의 짧은 영상화

핵심은 포맷을 늘리더라도 채널 정체성을 잃지 않는 것이다.

> 비개발자가 AI로 자기 일을 직접 도구화하는 과정을 보여주는 채널

---

## 1. 이미지 제작 전략

### 결론

트렌디형 카드뉴스의 이미지 베이스는 Claude Code가 아니라 ChatGPT image2 계열을 적극 활용한다.

Claude Code는 다음 역할에 더 적합하다.

- 파일 구조 생성
- HTML/CSS/PIL 렌더 스크립트 작성
- 카드뉴스 레이아웃 자동화
- 리소스 HTML 페이지 제작
- 반복 렌더/ZIP 생성

ChatGPT image2는 다음 역할에 더 적합하다.

- 프리미엄 캠페인 컷
- 인물/제품/공간 이미지
- AI 작업실 무드 컷
- 인스타 첫 장용 트렌디 비주얼
- 영상 썸네일/커버용 이미지

### 사용 기준

설명형 카드뉴스:
- 실제 UI, 스크린샷, 도형, 텍스트 중심
- Claude Code/PIL/HTML 렌더 중심

트렌디형 카드뉴스:
- ChatGPT image2로 3~5개 비주얼 후보 생성
- 얼굴/손/텍스트 가림 여부 vision QA
- 선택된 이미지 위에 Hermes/Claude Code로 타이포/카드 UI 합성

### 커버 제작 룰

- 얼굴 위에는 박스/텍스트 금지
- AI 이미지에 글자를 직접 넣지 않음
- 이미지 생성은 무텍스트로 뽑고, 텍스트는 후처리로 얹음
- 4:5 안전영역 기준으로 타이포 위치를 먼저 정함
- 인물/오브젝트/하늘/여백의 역할을 분명히 함

## 2. 뉴스형 콘텐츠 전략

### 방향

최근 인기 있는 AI/Claude/업무 자동화 내용을 그대로 전달하는 뉴스 계정이 아니라, 비개발자 관점으로 번역하는 큐레이션 포맷으로 운영한다.

단순 뉴스:
> Claude에 새 기능이 나왔습니다.

우리식 뉴스:
> 비개발자에게 중요한 건 새 기능 이름이 아니라, 이제 어떤 일을 맡길 수 있느냐입니다.

### 뉴스형 콘텐츠 기준

다룰 수 있는 주제:
- Claude / ChatGPT / Gemini / Perplexity 등 주요 AI 업데이트
- Claude Code / MCP / Desktop Connector / Skills / Agents 관련 변화
- Meta / Instagram / Threads 크리에이터 기능 변화
- AI로 콘텐츠 제작, 업무 자동화, 소규모 앱 제작에 영향을 주는 변화
- 생산성 도구 업데이트: Notion, Canva, Figma, Google Workspace 등

다루지 않을 주제:
- 단순 모델 점수 비교
- 투자/주가 중심 AI 뉴스
- 너무 개발자 전용인 인프라 업데이트
- 공식 문서 내용을 그대로 요약한 PDF식 자료

### 뉴스형 게시글 구조

#### 5분 정리형
1. 무슨 일이 있었나
2. 비개발자에게 왜 중요한가
3. 바로 써볼 수 있는 일
4. 조심할 점
5. 관련 프롬프트/워크플로우
6. 저장/프로필 링크 CTA

#### 오해 교정형
1. 다들 새 기능이라고만 봅니다
2. 사실 중요한 건 이 부분입니다
3. 비개발자는 이렇게 써야 합니다
4. 예시 업무 3개
5. 복붙 프롬프트
6. 리소스 CTA

#### 뉴스 + 리소스형
1. 업데이트 후킹
2. 쓸모 있는 상황
3. 예시 입력
4. 예시 출력
5. 체크리스트
6. HTML 템플릿 CTA

### 빈도

초기에는 주 1회 이하.

추천 비율:
- 설명형/가이드형: 50%
- 트렌디형/캠페인형: 25%
- 케이스형: 15%
- 뉴스형: 10%

뉴스형은 도달에는 좋지만, 너무 많아지면 계정이 뉴스 계정처럼 보일 수 있다.

## 3. 영상 콘텐츠 전략

### 방향

모든 콘텐츠를 영상화하지 않는다. 영상으로 만들었을 때 더 잘 전달되는 것만 짧게 만든다.

영상에 적합한 콘텐츠:
- 전후 비교가 있는 것
- 화면 변화가 있는 것
- Claude에게 입력 → 결과 출력 과정이 있는 것
- 3단계 흐름이 있는 것
- 실제 HTML 리소스 사용 장면

영상에 부적합한 콘텐츠:
- 긴 설명
- 체크리스트만 있는 게시글
- 화면 변화 없이 텍스트만 읽는 내용
- 억지로 만든 릴스형 자막 영상

### 영상 포맷 후보

#### A. 15초 결과 먼저 보여주기
구조:
1. 0~2초: 결과물 먼저
2. 2~5초: Claude에 넣은 요청
3. 5~10초: 결과가 생성되는 화면
4. 10~13초: HTML 프롬프트 키트
5. 13~15초: 저장/프로필 링크 CTA

적합 주제:
- 콘텐츠 30일치
- 회의록 정리
- 캡션 10개
- 캐러셀 구조 생성

#### B. 30초 미니 튜토리얼
구조:
1. 문제 한 줄
2. 복붙 프롬프트 보여주기
3. Claude 결과 보여주기
4. 검수 기준 한 가지
5. 리소스 안내

적합 주제:
- 업무 지시서 템플릿
- Claude Code 시작 전 체크리스트
- 프롬프트 디버깅

#### C. 뉴스 숏폼
구조:
1. 업데이트 한 줄
2. 사람들이 놓치는 포인트
3. 비개발자 활용 예시
4. 바로 써볼 프롬프트
5. 저장 CTA

적합 주제:
- Claude/ChatGPT 기능 업데이트
- Instagram/Threads 운영 기능 변화
- AI 업무 도구 업데이트

### 제작 방식

1. 카드뉴스용 스토리보드 작성
2. 영상화 가능한지 판단
3. 가능하면 세로 영상 스크립트 작성
4. 필요한 화면:
   - Claude 입력 화면
   - 결과 화면
   - HTML 리소스 화면
   - 최종 카드/캘린더 화면
5. 15~30초 버전으로 편집

초기에는 완성도 높은 영상보다, 화면 녹화 + 자막 + 커버 이미지 조합으로 테스트한다.

## 4. 포맷별 역할 정리

### 카드뉴스
- 저장/신뢰/설명
- 리소스 연결
- 계정 아카이브 구축

### 트렌디 이미지 카드
- 첫인상/도달/브랜드 감도
- 결과 중심 후킹
- 프롬프트 키트 유도

### 뉴스형 카드
- 최신성/공유성
- “이 계정은 흐름을 따라간다”는 인식
- 단, 실용 해석을 붙여야 함

### 영상/릴스
- 도달/체류/실제 사용 장면 증명
- 프롬프트가 결과로 바뀌는 순간을 보여줌
- 같은 리소스를 다른 포맷으로 재활용

## 결론

이미지는 ChatGPT image2를 적극 활용한다. Claude Code는 제작 자동화와 HTML/렌더링에 쓴다.

뉴스형 콘텐츠는 일부 섞되, 단순 요약이 아니라 `비개발자에게 그래서 뭘 할 수 있나`로 번역한다.

영상은 모든 게시글에 붙이지 말고, 프롬프트 입력 → 결과 출력 → HTML 리소스 사용이 보이는 콘텐츠부터 짧게 만든다.
