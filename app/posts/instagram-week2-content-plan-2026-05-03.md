# Instagram/Threads 콘텐츠 기획 — 2026-05-03 시작

## 채널 포지션

**비개발자를 위한 AI 작업 시스템 채널**

- 개발을 가르치는 채널이 아니라, 비개발자가 AI로 직접 결과물을 만드는 과정을 보여주는 채널.
- 설명형은 신뢰/저장, 트렌디형은 욕망/도달/첫인상 담당.
- 한 채널 안에서 두 포맷을 운영한다.

## 이번 주 운영 목표

- 계정 정체성 고정: “비개발자도 AI로 직접 만든다”
- 가이드를 바로 홍보하지 말고, 콘텐츠 안에서 문제를 먼저 만들고 자연스럽게 연결
- 설명형 70%, 트렌디형 30%
- 링크는 남발하지 않음. Instagram은 프로필/댓글 유도 중심, Threads는 링크 제한 규칙 유지.

## 1주차 콘텐츠 캘린더

### 05/03 일 — 런칭/선언형
- 포맷: 설명형 + 선언
- 스토리 구조: Before → After → Bridge
- 제목 후보: `코딩 몰라도, 이제 만들 수 있습니다`
- 핵심 메시지: 이 채널은 비개발자가 AI로 직접 업무 도구와 콘텐츠 시스템을 만드는 과정을 다룬다.
- 역할: 채널 정체성 고정
- CTA: “이번 주부터 하나씩 따라오세요.”

### 05/04 월 — 첫 문제 제기
- 포맷: 설명형 카드뉴스
- 스토리 구조: Misbelief → Reframe
- 제목 후보: `AI가 못하는 게 아니라, 시키는 방식이 문제입니다`
- 핵심 메시지: 프롬프트 한 줄보다 작업 환경/자료/순서가 중요하다.
- 가이드 연결: 4편 Claude 잘 쓰는 법, 5편 워크스페이스
- CTA: “저장해두고 다음 요청 전에 확인하세요.”

### 05/05 화 — 트렌디형 결과물 중심
- 포맷: 트렌디형 캠페인 카드
- 스토리 구조: Prerequisite → Result Chain
- 제목 후보: `콘텐츠 30일치, Claude가 짜줌`
- 핵심 메시지: 비개발자도 콘텐츠 캘린더/훅/CTA를 시스템으로 만들 수 있다.
- 시각 방향: premium campaign + 작업 UI + 결과물 캘린더
- CTA: “댓글에 ‘콘텐츠’라고 남기면 구조를 정리해볼게요.”

### 05/06 수 — 가이드 맛보기
- 포맷: 설명형 체크리스트
- 스토리 구조: Checklist
- 제목 후보: `Claude Code 시작 전, 이 3개만 준비하세요`
- 핵심 메시지: 터미널 공포를 줄이고, 계정/폴더/첫 목표만 준비하면 된다.
- 가이드 연결: 2편 설치, 3편 첫 프로젝트
- CTA: “설치보다 먼저 목표 하나를 정하세요.”

### 05/07 목 — 실패담/공감형
- 포맷: 공감 스토리
- 스토리 구조: Mini Drama
- 제목 후보: `비개발자가 Claude Code 쓰면 처음엔 이렇게 망합니다`
- 핵심 메시지: 실패는 정상이고, 잘못된 지시/맥락 부족/검수 부재가 문제다.
- 가이드 연결: 4편 프롬프트, 9편 Hooks
- CTA: “망한 기록도 작업 자산입니다.”

### 05/08 금 — 사례 스택형
- 포맷: 설명형/저장형
- 스토리 구조: Case Stack
- 제목 후보: `비개발자가 처음 만들기 좋은 도구 5개`
- 사례: 회의록 요약기, CSV 분석기, 파일 이름 정리기, 견적서 자동화, 콘텐츠 캘린더
- 가이드 연결: 3편 첫 프로젝트
- CTA: “하나만 골라서 이번 주말에 만들어보세요.”

### 05/09 토 — 주간 요약/전환
- 포맷: 요약 + 다음 주 예고
- 스토리 구조: Reason → Evidence → Action
- 제목 후보: `비개발자에게 필요한 건 코딩 공부가 아닙니다`
- 핵심 메시지: 필요한 건 개발 지식 전체가 아니라, AI에게 일을 맡기는 순서와 검수법.
- CTA: “다음 주는 실제 제작 과정을 보여줄게요.”

## 첫 게시물 초안 — 05/03 일

### Instagram 카드뉴스 6장 스토리보드

1. Hook
   - `코딩 몰라도, 이제 만들 수 있습니다`
   - 보조: `비개발자를 위한 AI 작업 시스템`

2. Before
   - `전에는 뭔가 만들려면 개발자에게 부탁해야 했습니다`
   - 예: 견적서 자동화, 회의록 정리, 파일 정리, 콘텐츠 캘린더

3. Tension
   - `문제는 아이디어가 없어서가 아니라, 만드는 방법을 몰랐던 것`
   - “코딩을 배워야 하나?”라는 부담 강조

4. Turn
   - `이제는 코딩보다 작업을 설명하는 능력이 먼저입니다`
   - AI에게 목표/자료/조건/검수 기준을 주는 방식

5. Reward
   - `이번 주부터 다룰 것`
   - Claude Desktop / Claude Code / 첫 프로젝트 / 워크스페이스 / 자동화 / 안전장치

6. CTA
   - `혼자 만들고 싶은 게 있다면, 이번 주부터 같이 해봅시다`
   - `@nodev.builder`

### Threads 텍스트 버전

```text
비개발자인데 뭔가 직접 만들고 싶다는 생각을 계속 했다.

회의록 정리기, 견적서 자동화, 콘텐츠 캘린더 같은 거.
예전엔 개발자한테 부탁해야 한다고 생각했는데,
요즘은 Claude한테 제대로 설명하면 꽤 많은 걸 직접 만들 수 있다.

이 계정에서는 코딩 강의가 아니라,
비개발자가 AI로 자기 일을 직접 도구화하는 과정을 기록해보려고 한다.

이번 주부터 하나씩 해보자.
```

## 제작 규칙

- 첫 주는 너무 판매/가이드 홍보처럼 보이지 않게 한다.
- 가이드는 “필요하면 더 볼 수 있는 자료”로만 연결한다.
- 트렌디형도 항상 비개발자의 결과/시간 절약/혼자 만든 시스템과 연결한다.
- 설명형 카드뉴스는 Warm Minimal/Pretendard.
- 트렌디형은 premium campaign이지만 “멋진 AI 이미지”로 끝나면 실패.
