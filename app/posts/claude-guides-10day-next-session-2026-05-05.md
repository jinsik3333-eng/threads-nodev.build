## 3. 리드 마그넷 1 — 클로드 가이드

### 목적

Claude Desktop 앱을 처음 쓰는 비개발자를 위한 초입 가이드.

이 사람을 대상으로 한다:
- Claude가 뭔지는 들어봤지만 제대로 써본 적 없음.
- ChatGPT처럼 질문만 던지는 수준.
- Desktop 앱을 왜 써야 하는지 모름.
- 프로젝트/파일/작업 맥락 개념이 없음.
- 설치 화면, 로그인 화면, 새 대화 화면부터 낯설다.

### 임시 제목 후보

- 비개발자를 위한 Claude 첫 시작 가이드
- Claude 처음 켜면 이 순서대로 보세요
- Claude Desktop, 설치보다 먼저 알아야 할 5가지
- Claude를 검색창처럼 쓰지 않는 법

### URL 후보

- `/guides/resources/claude-beginner-guide.html`
- `/guides/resources/claude-desktop-start.html`

### 구성 초안

1. Claude는 검색창이 아니라 같이 일하는 작업 상대입니다
2. Claude Desktop 앱을 쓰는 이유
3. 처음 켜면 봐야 할 화면
4. 새 대화에서 바로 하면 안 되는 요청
5. 좋은 첫 요청의 구조
6. 파일/자료를 넣을 때 조심할 것
7. 결과가 마음에 안 들 때 다시 시키는 법
8. 다음 단계: Claude Code가 필요한 순간

### 핵심 메시지

“처음부터 잘 쓰려고 하지 말고, Claude에게 일을 맡기는 기본 순서부터 익히면 됩니다.”

### 콘텐츠에서 연결할 CTA

- “처음이면 프로필 링크의 Claude 가이드부터 보세요.”
- “댓글에 `시작`을 남기면 입문 순서부터 보내드릴게요.”

## 4. 리드 마그넷 2 — 클로드 코워크 가이드

### 목적

Claude를 단순 채팅 도구가 아니라 “같이 일하는 파트너”로 쓰는 법을 알려준다.
Claude Desktop → 업무 지시서 → 작업 폴더 → Claude Code로 자연스럽게 넘어가는 다리 역할.

### 대상

- Claude에게 매번 “해줘”라고만 말하는 사람.
- 답변이 마음에 안 들어도 무엇을 고쳐야 할지 모르는 사람.
- 업무 지시서, 검수 기준, 작업 맥락이라는 개념이 없는 사람.
- Claude Code까지 가기 전에 Claude와 일하는 기본 습관이 필요한 사람.

### 임시 제목 후보

- Claude와 같이 일하는 법
- Claude에게 일을 맡기는 순서
- 비개발자를 위한 Claude 코워크 가이드
- Claude 답변이 별로일 때 고치는 법

### URL 후보

- `/guides/resources/claude-cowork-guide.html`
- `/guides/resources/claude-workflow-for-non-devs.html`

### 구성 초안

1. Claude에게 “질문”하지 말고 “업무”를 맡기기
2. 업무 지시서 6요소: 역할, 목표, 자료, 제약, 출력형식, 검수기준
3. 나쁜 요청 vs 좋은 요청
4. 결과가 별로일 때 고치는 질문 5개
5. Claude와 같이 일하는 3단계: 초안 → 검수 → 수정
6. 프로젝트 폴더/워크스페이스가 필요한 이유
7. Claude Code는 언제 필요한가
8. 다음 단계: 첫 프로젝트 만들기

### 핵심 메시지

“Claude를 잘 쓰는 사람은 질문을 잘하는 사람이 아니라, 일을 맡기는 순서를 아는 사람입니다.”

### 콘텐츠에서 연결할 CTA

- “설치가 끝났다면, 다음은 Claude에게 일을 맡기는 법입니다.”
- “프로필 링크에 Claude 코워크 가이드를 열어둘게요.”

## 7. 10일 콘텐츠별 상세 메모

### Day 1 — Claude 처음 켜면 어디부터 봐야 하나요?

포맷: 카드뉴스 6장

슬라이드:
1. Claude 처음 켜면 어디부터 봐야 할까요?
2. 먼저 새 대화 화면부터 익숙해지면 됩니다
3. 긴 질문보다 작은 업무 하나를 맡겨보세요
4. 예: 이 메모를 회의록처럼 정리해줘
5. 결과가 별로여도 실패가 아닙니다
6. 처음이면 Claude 가이드부터 보세요

### Day 2 — Claude Desktop 앱, 웹이랑 뭐가 다른가요?

포맷: 20~30초 화면녹화 릴스

구성:
1. 웹 Claude 화면
2. Desktop 앱 화면
3. 앱을 Dock/작업표시줄에서 여는 장면
4. 새 대화 시작
5. “작업을 이어가는 공간”이라는 자막

### Day 3 — Claude에게 처음 시키면 좋은 일 3가지

포맷: 카드뉴스

3가지:
1. 긴 글 요약
2. 회의 메모 정리
3. 초안 만들기

하지 말 것:
- 처음부터 앱 만들어줘
- 전부 알아서 해줘
- 내 업무 자동화해줘

### Day 4 — Claude에게 하면 안 되는 첫 요청

포맷: 비교 카드

나쁜 요청:
- “이거 알아서 정리해줘”
- “좋은 콘텐츠 만들어줘”

좋은 요청:
- “아래 메모를 보고, 1) 요약 2) 결정사항 3) 할 일을 표로 정리해줘. 모르는 내용은 추측하지 말고 비워둬.”

### Day 5 — 좋은 요청은 질문이 아니라 업무 지시서입니다

포맷: 카드뉴스/릴스

구성:
1. 역할
2. 목표
3. 자료
4. 제약
5. 출력 형식
6. 검수 기준

CTA:
- Claude 코워크 가이드 안내

### Day 6 — Windows 사용자는 여기서 가장 많이 막힙니다

포맷: 묶음 화면녹화 릴스

구성:
1. 다운로드 파일 찾기
2. PowerShell/터미널 찾기
3. 명령어 붙여넣기
4. 권한/보안 경고
5. 성공 화면 기준

### Day 7 — Claude 답변이 별로일 때 바로 고치는 법

포맷: 화면녹화 릴스

구성:
1. 별로인 답변 보여주기
2. “더 잘해줘” 대신 기준 추가
3. 예: 더 짧게, 표로, 초보자 말투로, 예시 포함
4. 수정된 결과 보여주기

### Day 8 — Claude와 일할 때 파일을 어떻게 넣어야 하나요?

포맷: 튜토리얼 카드

핵심:
- 파일을 넣기 전에 개인정보 확인
- Claude에게 파일을 읽고 무엇을 해야 하는지 말하기
- 결과 형식을 지정하기
- 원본을 지우거나 덮어쓰게 하지 않기

### Day 9 — Claude Code로 넘어가기 전에 알아야 할 것

포맷: 가이드 스크롤 릴스

핵심:
- Claude Code는 코딩 공부 앱이 아니라 작업 폴더를 다루는 도구
- 넘어가기 전 필요한 것: 작업 목표, 폴더, 결과물 기준
- 바로 앱 만들기보다 README 하나부터

### Day 10 — 처음이면 이 순서로만 보세요

포맷: 로드맵 카드뉴스

순서:
1. Claude 가이드
2. Claude 코워크 가이드
3. Claude Code 설치 가이드
4. 첫 프로젝트 만들기
5. 잘 시키는 법/워크스페이스

- 너무 개발자스럽게 설명하지 않는다.
- “터미널”, “PowerShell”, “워크스페이스”, “Claude Code” 같은 단어는 처음 등장할 때 화면 기준으로 설명한다.
- Windows 사용자가 “내 화면이 다르다”고 느끼지 않게 한다.
- 리드 마그넷은 PDF가 아니라 HTML 페이지 우선.
- 모든 게시글에 DM 키워드를 붙이지 않는다.
- 현재 키워드 `시작`은 유지하고, 새 키워드는 반응 확인 후 추가한다.
- 영상/이미지 산출물은 프로젝트 안에 오래 보관하지 않는다.
