채널: Instagram 메인, Threads 보조
현재 상황 반영:

## 1. 이번 한 달의 핵심 방향

### 한 줄 전략

비개발자가 AI로 직접 만드는 과정을 “설치부터, 막히는 지점, 가이드 4~9편 활용, 작은 결과물 제작까지” 단계별로 보여준다.

### 기존 30일 캘린더와 달라진 점

이전 캘린더는 프롬프트/업무 자동화 주제가 많았다.
이번 v2는 초반 난이도를 더 낮추고, 운영체제별 설치 차이를 분리한다.

- 설치 자체를 못 하는 사람을 위한 콘텐츠를 늘린다.
- Mac뿐 아니라 Windows 사용자의 설치/터미널/권한/경로 문제를 별도 콘텐츠로 다룬다.
- 다만 너무 쉬운 주제는 단독 게시글로 쪼개지 않고, 3~5개를 묶어서 하나의 짧은 영상으로 만든다.
- “Claude Code 잘 쓰는 법”보다 “처음 화면에서 어디를 눌러야 하는지”부터 다룬다.
- 가이드 4~6편은 “설치 다음 단계”로, 7~9편은 “반복 작업과 안전장치”로 중간중간 소개한다.
- 계정 운영 과정보다는, 실제 가이드가 어떤 문제를 해결하는지 보여준다.
- 완성형 카드뉴스보다 실제 촬영/화면녹화/편집 전후를 더 많이 쓴다.
- 비개발자에게 중요한 감정: 무서움, 설치 실패, 터미널 공포, 내 화면이 가이드와 다를 때의 불안.

## 2. 콘텐츠 기둥 6개

### A. 설치/입문 튜토리얼 — 35%

역할: 저장, 신뢰, 실질 도움.

다루는 것:
- Claude Desktop 설치
- Claude Code 설치
- 터미널 열기
- 작업 폴더 만들기
- 첫 프로젝트 열기
- 흔한 오류 화면
- 성공 화면 기준

대표 문장:
- “설치부터 막히는 분들이 생각보다 많아서, 여기부터 다시 갑니다.”
- “코딩보다 먼저, 이 화면이 무서운 게 정상입니다.”

추천 포맷:
- Instagram: 20~40초 화면녹화 릴스 + 카드뉴스 요약
- Threads: 짧은 체크리스트 + 한 장면 스크린샷/가이드 링크

### B. Windows 설치/입문 보강 — 20%

역할: 실제 초보자가 가장 많이 막히는 Windows 환경을 별도로 챙기기.

다루는 것:
- Windows에서 터미널/PowerShell 여는 법
- 다운로드 파일을 어디서 찾는지
- 설치 후 명령어가 안 먹힐 때 확인할 것
- 경로/권한/복붙 문제
- Mac 설명과 Windows 설명을 헷갈리지 않게 분리하기

대표 문장:
- “Windows는 화면 이름부터 달라서 더 헷갈립니다.”
- “내 화면이 Mac 가이드와 다르다고 실패한 게 아닙니다.”

추천 포맷:
- 초저난도 화면녹화 릴스
- Mac vs Windows 비교 카드
- Threads 체크리스트

### C. 가이드 4~9편 활용 소개 — 25%

역할: 이미 만든 가이드를 죽은 링크가 아니라 “따라갈 순서”로 보이게 만들기.

다루는 것:
- 4편: Claude 잘 쓰는 법 — 질문이 아니라 업무 지시서로 시키기
- 5편: 나만의 워크스페이스 — 매번 설명하지 않게 작업 맥락 쌓기
- 6편: 공식 플러그인/서비스 연동 — 연결은 나중에, 개념부터 이해하기
- 7편: Skills — 반복 작업을 명령어처럼 다시 쓰기
- 8편: Sub-agents — AI에게 역할을 나눠 맡기는 감각
- 9편: Hooks — 실수/위험한 작업을 막는 안전장치

대표 문장:
- “설치가 끝나면, 그다음부터는 ‘잘 시키는 법’이 중요합니다.”
- “처음엔 1~3편만 보면 되고, 익숙해지면 4~9편으로 넘어가면 됩니다.”

추천 포맷:
- Instagram: 실제 가이드 스크롤 릴스 + 5장 요약 카드
- Threads: ‘오늘은 4편만 보면 됩니다’ 식의 짧은 안내글

### D. 작은 업무 도구 사례 — 10%

역할: 욕망, 결과, “나도 만들 수 있겠다”.

다루는 것:
- 회의록 정리기
- 파일명 정리기
- 콘텐츠 캘린더 생성
- 캡션 10개 뽑기
- 고객문의 분류

추천 포맷:
- Before/After 영상
- 6장 카드뉴스
- Threads 짧은 사례글

### E. 프롬프트/업무 지시서 — 5%

역할: 비개발자의 핵심 역량 교육.

다루는 것:
- 질문이 아니라 업무 지시서로 시키기
- 역할/목표/자료/제약/검수 기준
- 결과가 별로일 때 고치는 법

### F. 안전/운영/자동화 — 5%

역할: 신뢰, 계정 장기 운영.

다루는 것:
- 자동화는 천천히
- 민감정보 넣지 않기
- 최종 게시/발송은 사람이 확인
- Meta 토큰은 계정 안정 후 발급

## 3. 주차별 흐름

### 1주차: “설치부터 막히는 사람을 위한 시작”

목표:
- 난이도를 과감히 낮춘다.
- 설치/첫 화면/첫 성공 기준을 보여준다.
- Mac/Windows를 분리해서 “내 화면과 다르다”는 불안을 줄인다.

### 2주차: “설치 다음은 잘 시키는 법”

목표:
- Claude Code를 코딩 도구가 아니라 작업실로 이해시킨다.
- 폴더/파일/요청/결과 확인을 다룬다.
- 가이드 4~6편을 “설치 다음 단계”로 자연스럽게 소개한다.

### 3주차: “반복 작업과 안전장치로 넘어간다”

목표:
- 비개발자도 만들 수 있는 작은 자동화 사례를 보여준다.
- 가이드 7~9편을 “반복 작업/역할 분담/안전장치”로 묶어 소개한다.

### 4주차: “Mac/Windows 입문자를 모두 받을 수 있는 시스템으로 묶는다”

목표:
- 프롬프트, 폴더, 가이드, 콘텐츠 제작 과정을 하나의 시스템으로 보여준다.
- Windows 설치 문제를 다시 한 번 정리한다.
- 다음 달 미니 도구 제작 시리즈로 연결한다.

- 인스타 편집 앱에서 자막/음악을 올리는 장면

### 촬영 가이드

- 세로 9:16으로 찍기. 가능하면 1080x1920 또는 4K 세로.
- 한 클립은 5~12초면 충분.
- 마우스 움직임을 너무 빠르게 하지 않기.
- 개인정보, 이메일, 토큰, 결제정보, DM 내용은 가리기.
- 화면을 찍을 때는 “성공 화면”과 “막히는 화면”을 둘 다 남기기.
- Windows 화면은 Mac 화면과 섞지 말고 별도 클립으로 찍기.
- 음성은 없어도 된다. 나중에 자막/음악으로 편집 가능.

### 내가 편집할 때 필요한 것

사용자가 영상 원본을 주면 다음 구조로 편집한다.

1. 0~2초: 문제 후킹
   - 예: “Windows에서 터미널부터 막힌다면”
2. 2~8초: 실제 화면
   - 클릭, 스크롤, 입력, 오류/성공 화면
3. 8~15초: 기준 한 줄
   - 예: “이 화면까지 오면 설치는 된 겁니다”
4. 15~20초: 다음 행동
   - 예: “처음이면 댓글에 `시작`”

## 8. Instagram / Threads 운영 차이

## 10. 이번 주 바로 제작할 5개 상세안

### 1) 설치부터 막히는 게 정상입니다

포맷: 카드뉴스 6장

슬라이드:
1. 설치부터 막히는 게 정상입니다
2. 문제는 의지가 아니라 화면이 낯선 것
3. 다운로드, 로그인, 터미널, 권한 — 여기서 다 멈춥니다
4. Mac 화면과 Windows 화면이 달라서 더 헷갈립니다
5. 그래서 성공 기준을 먼저 보여드릴게요
6. 처음이면 댓글에 `시작`

### 2) Windows 설치 첫 화면에서 막히는 5가지

포맷: 25~35초 묶음 화면녹화 릴스

의도:
- “터미널 여는 법”, “다운로드 폴더 찾기”, “붙여넣는 위치”를 각각 단독 콘텐츠로 만들면 너무 얇다.
- 하나의 영상 안에서 초보자가 실제로 막히는 첫 5초 지점을 빠르게 보여준다.

구성:
1. 시작 메뉴에서 Terminal/PowerShell 찾기
2. 다운로드 폴더에서 설치 파일 찾기
3. 브라우저와 앱 화면 구분하기
4. 명령어를 붙여넣는 위치 보여주기
5. 성공 화면 기준 보여주기

자막 톤:
- “여기서 막히는 게 정상입니다.”
- “중요한 건 이름을 외우는 게 아니라, 명령어가 들어가는 위치를 아는 겁니다.”

### 3) Claude Desktop 설치 전 확인 3가지

포맷: 화면녹화 릴스

후킹:
- “Claude 설치 전에 이 3개만 확인하세요.”

구성:
1. 계정 로그인 가능?
2. 앱 설치 위치 확인?
3. 첫 화면에서 새 대화 열림?

CTA:
- “이 화면까지 오면 1단계는 끝입니다.”

### 4) Mac과 Windows 설치 화면 한 번에 비교

포맷: 20~30초 비교형 묶음 릴스

핵심:
- 같은 설치라도 Mac은 Terminal, Windows는 PowerShell/터미널 화면을 보게 된다.
- 내 화면이 가이드와 다르다고 해서 실패한 것이 아니다.
- 운영체제별로 “정상 화면”을 따로 보여주는 콘텐츠가 필요하다.

구성:
1. Mac: Spotlight → Terminal
2. Windows: 시작 메뉴 → PowerShell/Terminal
3. Mac: 다운로드/앱 실행 흐름
4. Windows: 다운로드 폴더/실행 흐름
5. 공통 성공 기준

### 5) 가이드 4편: Claude 잘 쓰는 법

포맷: 가이드 스크롤 릴스 + 요약 카드

후킹:
- “설치가 끝났다면, 이제부터는 질문이 아니라 지시서를 써야 합니다.”

구성:
1. 가이드 4편 제목 화면
2. 나쁜 요청 예시
3. 좋은 요청 예시
4. 역할/목표/자료/제약/검수 기준
5. “처음이면 1~3편, 설치 끝났다면 4편”

## 11. 제작 원칙

- 초반에는 “멋있게”보다 “안 막히게”가 우선.
- 화면녹화가 카드뉴스보다 더 신뢰를 줄 때는 영상 우선.
- 카드뉴스는 요약/저장용으로 사용.
- Threads는 완성본 홍보보다 짧은 안내/생각 기록에 가깝게 쓴다.
- 실패와 오류 화면도 콘텐츠로 쓰되, “반응 없음” 빌드로그는 이번 캘린더에서는 제외한다.
- 가이드 4~6편, 7~9편은 한 번에 홍보하지 말고 중간중간 “지금 필요한 다음 단계”로 소개한다.
- Windows 사용자는 별도 초저난도 콘텐츠로 챙긴다.
- 영상 원본은 프로젝트 안에 오래 두지 않고, 편집 후 아카이브한다.
