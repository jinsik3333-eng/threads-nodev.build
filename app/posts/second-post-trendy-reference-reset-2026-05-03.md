# 두 번째 트렌디 게시글 리셋 — 레퍼런스 재정렬

## 사용자 피드백

- 기존 2~5페이지가 모두 이상함.
- 글 내용과 트렌디 이미지가 맞지 않는 느낌이 강함.
- HTML로 만들다 보니 오히려 안 좋게 뽑히는 것 같음.
- @mobileeditingclub / @adrianabubori 레퍼런스를 다시 보고 이미지가 얼마나/어떻게 다른지 확인해야 함.
- image2로 바로 뽑는 방식 검토.

## 확인한 레퍼런스 차이

### @mobileeditingclub

- 패션/뷰티/브랜드 캠페인 같은 AI 이미지 중심.
- 큰 흰색 산세리프 타이포 + 세리프/이탤릭 포인트.
- 인물/제품/소품이 주인공이고, 텍스트는 짧고 강함.
- 작은 pill CTA, 반복 로고, 캐러셀 아이콘, 포스터형 썸네일.
- 정보 카드가 아니라 “AI로 만든 광고/캠페인 비주얼”처럼 보임.

### @adrianabubori

- 개인 브랜딩 + Claude/AI workflow 콘텐츠.
- “I save 4 hours”, “Claude cowork”, “Prompt less Edit more”처럼 경험/결과 중심.
- 인물, Claude 로고/아이콘, 노트북/작업 장면, AI agent 이미지가 섞임.
- 브랜드 계정보다 더 창업자/크리에이터 얼굴과 라이프스타일이 강함.

## 기존 시안이 틀어진 이유

1. 2~5페이지가 웹/HTML/SaaS 랜딩페이지처럼 보임.
   - 라운드 카드, 그리드, 버튼, 브라우저 UI가 반복됨.
   - 인스타 트렌디 이미지가 아니라 웹 컴포넌트처럼 보임.

2. 글 내용과 이미지가 직접 맞물리지 않음.
   - “콘텐츠가 되는 과정”인데 그냥 예쁜 책상.
   - “콘텐츠 재료”인데 실제 훅/캡션/캘린더 조각은 안 보임.
   - “발행물로 바꾸는 법”인데 툴 이름 나열 차트.

3. 실제 레퍼런스는 설명 카드가 아니라 결과/무드/캡처감/콜라주가 강함.
   - 실제 Claude 출력 캡처처럼 보이는 조각
   - 캘린더/인스타 카드/폰 화면/노트/손/커서/하이라이트
   - 하나의 아이디어가 여러 발행물로 분열되는 장면
   - 완성된 브랜드 캠페인처럼 보이는 이미지

## 새 방향

HTML로 슬라이드 전체를 조립하지 말고, image2로 각 장의 “완성된 시각 장면”을 먼저 만든다.

단, image2에 한국어 텍스트를 맡기지 않는다.

- image2: 이미지/장면/콜라주/무드/오브젝트/프레임 생성
- 후처리: 큰 한국어 타이틀 1~2줄만 얹기
- HTML/PIL: 최종 타이포 얹기와 PNG/ZIP만 담당

## 새 6장 구조 후보

1. Cover
   - 프리미엄 인물/브랜드 캠페인 이미지
   - `콘텐츠 30일치 / Claude가 짜줌`

2. Problem
   - 빈 콘텐츠 캘린더, 메모, 폰 초안, 지친 크리에이터/작업 책상
   - `매일 새로 생각하니까 / 계정이 멈춥니다`

3. Transformation
   - 하나의 메모/아이디어가 카드, 릴스, 캡션, 캘린더로 분열되는 콜라주
   - `아이디어를 더 짜는 게 아니라 / 쪼개야 합니다`

4. Claude Output
   - Claude 답변 화면처럼 보이는 프롬프트/결과 조각 + 하이라이트
   - `Claude에게 / 결과물 형태를 먼저 정해줍니다`

5. Content Asset Board
   - 30일 캘린더 일부, 훅 문장, 카드 썸네일, 캡션 조각, 업로드 체크리스트가 보이는 asset board
   - `결과는 아이디어가 아니라 / 발행 가능한 재료여야 합니다`

6. CTA
   - 프롬프트 키트/캘린더/워크플로우가 묶인 프리뷰 이미지
   - `프로필 링크에서 / 무료 키트 받기`

## 제작 원칙

- image2에서 텍스트를 많이 넣지 말 것. 가짜 글자는 흐릿한 UI 조각 정도만 허용.
- 한국어 타이틀은 후처리로 얹는다.
- 2~5페이지는 같은 라운드 카드 시스템 반복 금지.
- 각 장은 “정보 카드”가 아니라 “장면”이어야 함.
- 실제 리소스의 가치가 보이게 훅/캘린더/캡션/CTA 조각을 이미지에 넣는다.
- 얼굴/손/주요 오브젝트 위에 박스 금지.
