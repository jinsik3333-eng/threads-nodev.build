# @nodev.builder 30일 리드 마그넷 맵 — 2026-05-03

## 원칙

리드 마그넷은 “자료를 많이 주는 것”이 아니라 “바로 실행하게 만드는 것”이어야 한다.

따라서 기본 포맷은 PDF가 아니라 HTML이다.

- 복붙 가능한 프롬프트
- 변수 바꿔 넣는 칸
- 예시 입력값
- Claude 결과물 예시
- 검수 체크리스트
- 다음 행동

초기 계정에서는 모든 게시글에 DM 자동화를 붙이지 않는다. 핵심 리소스 몇 개만 댓글 키워드/DM으로 연결하고, 나머지는 프로필 링크/리소스 허브에서 공개한다.

## 리드 마그넷 레벨

### L0 — 공개 리소스 허브
- 목적: 프로필 링크에서 항상 열려 있는 기본 자료실
- URL 후보: `/guides/resources/`
- 역할: 모든 HTML 키트를 모아두는 허브
- CTA 문구: `프로필 링크에서 열어두었습니다.`

### L1 — 작은 프롬프트 키트
- 목적: 게시글 하나와 바로 연결되는 복붙용 프롬프트
- 형식: HTML 1페이지
- 예: 콘텐츠 30일치 프롬프트, 회의록 정리 프롬프트, 브랜드 톤 프롬프트

### L2 — 워크플로우 템플릿
- 목적: 프롬프트 하나보다 큰 업무 흐름 제공
- 형식: HTML + 체크리스트 + 예시 결과물
- 예: 콘텐츠 제작 워크플로우, Claude 업무 지시서 템플릿, 첫 프로젝트 준비 체크리스트

### L3 — 미니 시스템
- 목적: 실제로 자기 업무에 적용 가능한 작은 시스템
- 형식: HTML 가이드 + 샘플 파일/템플릿
- 예: 30일 콘텐츠 캘린더 생성기, 고객문의 분류기, 파일명 정리 워크플로우
