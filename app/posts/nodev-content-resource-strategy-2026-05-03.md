# @nodev.builder 콘텐츠/리소스 전략 메모 — 2026-05-03

## 출발점

첫 게시글은 채널 정체성 선언과 무료 가이드 안내 역할을 했다. 두 번째 이후 게시글은 단순히 “자료 받으세요”가 아니라, 실제로 복붙해서 써볼 수 있는 프롬프트/워크플로우/템플릿을 중심으로 설계한다.

## 피해야 할 방향

- Claude 공식 문서나 공식 계정 안내를 PDF 수십 페이지로 재가공해서 팔로워 모집용으로 쓰는 방식
- 실제로 쓰지 않을 거대한 자료집을 미끼처럼 주는 방식
- “팔로우해야 무료자료”처럼 강제성이 강한 CTA
- 사용자가 열어봐도 바로 행동으로 이어지지 않는 추상적인 가이드
- 링크/DM/자동화가 너무 많아 새 계정 스팸 신호가 커지는 흐름

## 우리가 가져갈 방향

> 적게 주더라도 바로 쓸 수 있게 준다.

핵심 리소스는 PDF보다 HTML 페이지가 적합하다.

- 모바일에서 바로 열림
- 각 프롬프트를 버튼으로 복사 가능
- 예시 입력값과 결과물 구조를 같이 보여줄 수 있음
- 업데이트가 쉬움
- 인스타 DM에서는 링크 1개만 보내도 충분함

## 리소스 포맷

### 1. Prompt HTML Kit

예시: `콘텐츠 30일치 Claude 프롬프트 키트`

구성:
- 이 프롬프트가 필요한 상황
- 복붙용 프롬프트
- 내 상황에 맞게 바꿀 변수
- Claude에 넣기 전 준비물
- 결과물을 검수하는 기준
- 다음에 할 행동

### 2. Workflow HTML

예시: `아이디어 → Claude → 콘텐츠 캘린더 → 인스타 업로드 흐름`

구성:
- 전체 흐름 한 장 요약
- 각 단계별 입력/출력
- 실패했을 때 고치는 질문
- 실제 예시

### 3. Tiny Template

예시:
- 콘텐츠 캘린더 표
- 캐러셀 6장 구조 템플릿
- 캡션/CTA 생성 프롬프트
- 브랜드 톤 정리 프롬프트

## 두 번째 게시글 후보

### 주제

`콘텐츠 30일치 / Claude가 짜줌`

### 메시지

콘텐츠를 매일 새로 생각하는 게 아니라, 하나의 아이디어를 Claude에게 구조화시키면 30일치 재료로 쪼갤 수 있다.

### CTA

초기 계정 안정성을 고려하면 두 번째 게시글에 바로 ManyChat 자동화를 추가할지는 신중히 결정한다.

가능한 CTA:

1. 안전형
   - `저장해두고, 필요할 때 복붙해서 써보세요.`
   - 프로필 링크의 리소스 허브로 유도

2. DM형
   - 댓글 키워드: `30일치`
   - DM: `콘텐츠 캘린더 프롬프트 키트 링크입니다.`
   - 단, 새 계정 경고가 있었으므로 링크 1개 + 짧은 문구만 사용

## 트렌디 커버 수정 원칙

기존 `carousel-v8-kr/card-01.png`의 형식과 이미지 톤은 유지한다.

단, 우상단 프롬프트 박스가 얼굴을 가리는 문제는 수정한다.

수정 규칙:
- 얼굴 위에는 박스/라벨/CTA를 올리지 않는다.
- 프롬프트 박스는 얼굴 반대쪽 하늘 여백으로 이동한다.
- 인물 얼굴은 캠페인 컷의 핵심 오브젝트로 보존한다.
- 메인 한글 타이틀은 기존처럼 하단 대형 타이포를 유지한다.
- `30일치` 같은 핵심 단어만 라임 컬러로 강조한다.

## 두 번째 게시글 6장 구조 초안

1. Cover
   - `콘텐츠 30일치 / Claude가 짜줌`
   - 기존 v8 트렌디 커버 톤 유지
   - 얼굴을 가리지 않는 프롬프트 박스 배치

2. Problem
   - `매일 올릴 게 없어서 계정이 멈춥니다`
   - 아이디어 부족이 아니라 구조 부족이라는 문제 제기

3. Reframe
   - `아이디어를 더 짜는 게 아니라, 쪼개는 방식을 정해야 합니다`
   - 하나의 주제를 hook / angle / format / CTA로 분해

4. Prompt
   - 실제 복붙 가능한 Claude 프롬프트 일부 공개
   - HTML 리소스에서 전체 프롬프트 제공

5. Output
   - Claude가 만들어주는 결과 예시
   - hooks, carousel outline, captions, CTA, calendar

6. CTA
   - 안전형: `프로필 링크에서 프롬프트 키트 열기`
   - 또는 DM형: `댓글에 30일치 남기면 링크 보내드릴게요`
   - 초기엔 첫 게시글 자동화만 유지하는 선택도 가능
