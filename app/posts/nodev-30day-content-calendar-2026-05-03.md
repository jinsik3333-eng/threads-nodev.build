# @nodev.builder 30일 콘텐츠 캘린더 초안 — 2026-05-03

## 운영 원칙

- 초반 1~2주는 하루 1개 수동 업로드.
- 첫 게시글 `시작` 자동화는 유지하되, 모든 게시글에 자동 DM을 붙이지 않는다.
- 설명형 70%, 트렌디형 30% 비율.
- 자료 미끼형 PDF보다 실제 복붙 가능한 HTML 프롬프트/워크플로우 키트로 연결한다.
- 각 게시글은 `비개발자가 AI로 자기 일을 직접 도구화한다`는 채널 정체성으로 돌아와야 한다.

## 포맷 구분

- 설명형: Warm Minimal / Claude Editorial / 저장 유도 / 실무 오해 교정
- 트렌디형: premium campaign / human + Claude UI / 짧은 결과 중심 카피
- 케이스형: 실제로 만들 수 있는 작은 도구 사례
- 리소스형: 복붙 가능한 HTML 프롬프트/템플릿 연결

- 형식: 설명형 / 채널 선언
- 제목: `코딩 몰라도, 내 일은 직접 만들 수 있습니다`
- 역할: 계정 정체성 선언, 무료 가이드 안내
- CTA: 댓글 `시작` → 입문 3단계 DM
- 상태: 업로드 완료 / ManyChat 작동 확인

### Day 2
- 형식: 트렌디형 / 리소스형
- 제목: `콘텐츠 30일치, Claude가 짜줌`
- 핵심 메시지: 콘텐츠를 매일 새로 생각하는 게 아니라, 하나의 아이디어를 30일치 구조로 쪼개는 것.
- 리소스: `콘텐츠 30일치 Claude 프롬프트 키트` HTML
- CTA: 프로필 링크에서 프롬프트 키트 열기 또는 저장 유도
- 자동화: 당장은 추가 DM 자동화 없이 안전 운영 권장

### Day 3
- 형식: 설명형

- 핵심 메시지: 프롬프트는 질문이 아니라 업무 지시서다.
- 연결 가이드: 4편 Claude 잘 쓰는 법
- CTA: 저장 / 프로필 링크

### Day 4

- 핵심 메시지: 계정, 작업 폴더, 성공 화면 기준이 먼저다.
- 연결 가이드: 1~2편
- CTA: 댓글 `설치`는 추후 자동화 후보

### Day 5

- CTA: 저장 유도

### Day 6

### Day 7

- CTA: 다음 주 예고

### Day 8
