# @nodev.builder 첫 게시글 캐릭터형 6장 스토리보드 — 2026-05-04

## 방향
- 최종 방향: B Premium Campaign 기반 캐릭터형.
- 캐릭터 기준: 둥근 검정 뿔테, 검은 비니/후드, 태블릿/워크플로우를 든 friendly builder.
- 톤: 귀엽지만 유치하지 않게, 프리미엄 앱 마스코트처럼 정돈.
- 핵심 메시지: 코딩을 배우는 계정이 아니라, 비개발자가 AI에게 일을 맡기는 구조를 직접 만드는 계정.

## 6장 구조

### 01 Cover / 정체성 선언
- 제목: `코딩 몰라도 / 내 업무 시스템은 / 직접 만듭니다`
- 보조: `비개발자를 위한 AI 작업 시스템`
- 후킹: `전체 가이드 무료 공개`
- 시각: 둥근 뿔테 캐릭터를 오른쪽에 크게 배치. 왼쪽은 강한 SUIT 타이포.
- 이유: 첫 장에서 계정 정체성과 캐릭터를 동시에 각인.

### 02 Recognition / 공감
- 제목: `아이디어는 있는데 / 구현에서 / 멈춘 적 있나요`
- 예시: `회의록 정리기`, `파일 정리 자동화`, `콘텐츠 캘린더`
- 보조: `문제는 의지가 아니라, 시작 구조가 없다는 것.`
- 시각: 생각은 카드로 떠 있는데 아래쪽에서 막히는 병목 구조.
- 이유: 비개발자 독자가 “내 얘기”라고 느끼게 함.

### 03 Reframe / 프레임 전환
- 제목: `개발자가 되는 게 / 목표는 / 아닙니다`
- 핵심: `목표 · 자료 · 조건 · 확인 기준을 정리해 AI에게 맡기는 법`
- 시각: “코딩 공부”를 작은 회색 항목으로 낮추고, “업무 시스템 설계”를 크게 보여줌.
- 이유: 계정의 관점을 분명히 전환.

### 04 Promise / 이 계정이 보여줄 것
- 제목: `만드는 과정을 / 그대로 / 보여드립니다`
- 항목: `처음 막힌 화면`, `잘못 나온 결과`, `수정 프롬프트`, `완성된 흐름`
- 시각: 제작 로그 타임라인/빌드 보드.
- 이유: 단순 팁 계정이 아니라 실제 제작 과정을 공개한다는 신뢰 형성.

### 05 Reward / 입문 경로
- 제목: `처음이라면 / 입문 3단계부터`
- 단계: `화면 감각 잡기`, `Claude Code 설치`, `첫 프로젝트 만들기`
- 보조: `전체 가이드는 공개해두었습니다. 처음엔 볼 순서만 잡으면 됩니다.`
- 시각: 3단계 루트 카드.
- 이유: 실질적인 저장/DM 가치 제공.

### 06 CTA / 행동
- 제목: `처음 시작할 / 순서가 필요하면 / “시작”`
- 보조: `댓글에 시작이라고 남기면, 입문 3단계부터 볼 순서를 DM으로 안내드립니다.`
- 하단: `@nodev.builder · 비개발자를 위한 AI 작업 시스템`
- 시각: 어두운 CTA 카드 + 캐릭터 작은 등장.
- 이유: CTA를 하나로 좁히고 ManyChat 키워드와 자연스럽게 연결.

## QA 기준
- 6장 모두 상단 중앙 `@nodev.builder`, 우상단 페이지 번호만 사용.
- 가장 작은 읽어야 하는 텍스트는 24px 이상.
- `1~3편` 같은 내부 번호는 노출하지 않음. 공개 카피에서는 `입문 3단계`만 사용.
- `무료 가이드는 열어두고` 같은 어색한 표현 금지.
- 마지막 CTA는 `시작` 하나만 강조.
